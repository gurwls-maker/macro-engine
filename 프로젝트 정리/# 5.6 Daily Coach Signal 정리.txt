# 5.6 Daily Coach Signal 기준 정의

## 1. 공통 기준

* 모든 Signal은 Today live 기준으로 계산한다
* 기준값은 “목표 대비 비율(%)”로 판단한다
* severity는 상태 강도를 의미한다

### 상태 기준

* low: 부족 상태
* ok: 정상 범위
* high: 과다 상태

### severity 기준

* 0: 정상
* 1: 경미한 문제
* 2: 명확한 문제

---

## 2. calorieSignal

### 기준 (목표 대비)

* 90% ~ 110% → ok (severity 0)
* 75% ~ 90% → low (severity 1)
* 110% ~ 125% → high (severity 1)
* < 75% → low (severity 2)
* > 125% → high (severity 2)

### reason 예시

* "목표 대비 섭취량이 부족합니다"
* "목표 대비 섭취량이 많습니다"

---

## 3. proteinSignal

### 기준

* 90% ~ 110% → ok
* 70% ~ 90% → low (1)
* 110% ~ 140% → high (1)
* < 70% → low (2)
* > 140% → high (2)

### 특징

* 단백질은 부족 쪽에 더 민감하게 본다

---

## 4. carbSignal

### 기준

* 85% ~ 115% → ok
* 70% ~ 85% → low (1)
* 115% ~ 135% → high (1)
* < 70% → low (2)
* > 135% → high (2)

---

## 5. fatSignal

### 기준

* 85% ~ 115% → ok
* 70% ~ 85% → low (1)
* 115% ~ 135% → high (1)
* < 70% → low (2)
* > 135% → high (2)

### 특징

* 지방은 과다 쪽 리스크를 강조할 수 있음

---

## 6. trainingSignal

### 입력 기준

* 운동 여부
* 운동 시간
* 운동 강도 (있는 경우)

### 상태 기준

* 운동 없음 + 목표 활동 있음 → low (1~2)
* 운동 있음 → ok
* 고강도 + 저섭취 상태 → high (스트레스 상태)

### 목적

* 영양 상태와 함께 해석용 (단독 판단 X)

---

## 7. dataQualitySignal

### 기준

* snapshot 있음 → ok (0)
* snapshot 없음 (manual 상태) → low (1)
* 입력 부족 (식사 미입력 등) → low (2)

### 역할

* Coach 출력 톤 조절용

---

## 8. 우선순위 규칙

여러 Signal이 동시에 문제일 경우:

1순위: calorie
2순위: protein
3순위: carb / fat
4순위: training
5순위: dataQuality

---

## 9. 출력 연결 규칙

* severity 2 → 반드시 상태/이유/힌트 출력
* severity 1 → 간단 요약
* severity 0 → 필요 시 생략 가능

---

## 10. 설계 완료 상태 정의

다음 조건 만족 시 5.6 완료:

* 모든 Signal 기준 정의 완료
* 상태/Severity 기준 명확
* dataQualitySignal 포함됨
* 우선순위 규칙 정의됨



# 5.6 Daily Coach Signal Severity 확장 수정안

## 1. 수정 이유

기존 severity 0~2 구조는 상태를 지나치게 단순화한다.

예를 들어 calorieSignal 기준에서:

* 목표 대비 74%
* 목표 대비 1%

두 값이 모두 low / severity 2로 처리되면,
실제 상태 차이에 비해 Coach 설명이 지나치게 동일해진다.

따라서 Daily Coach의 해석 정확도를 높이기 위해
calorie / protein / carb / fat Signal의 severity를 최소 0~3으로 확장한다.

---

## 2. 공통 severity 기준

### severity 0

* 정상 범위
* 특별한 경고 없음

### severity 1

* 경미한 부족 또는 과다
* 가벼운 조정 힌트 제공

### severity 2

* 명확한 부족 또는 과다
* Coach가 상태와 이유를 분명히 설명

### severity 3

* 심한 부족 또는 과다
* 강한 주의 문구 필요
* 단, 의학적 진단처럼 표현하지 않음

---

## 3. calorieSignal 수정 기준

### 목표 대비 섭취율

* 90% ~ 110%
  → ok / severity 0

* 75% ~ 90%
  → low / severity 1

* 60% ~ 75%
  → low / severity 2

* < 60%
  → low / severity 3

* 110% ~ 125%
  → high / severity 1

* 125% ~ 145%
  → high / severity 2

* > 145%
  > → high / severity 3

---

## 4. proteinSignal 수정 기준

### 목표 대비 섭취율

* 90% ~ 115%
  → ok / severity 0

* 75% ~ 90%
  → low / severity 1

* 55% ~ 75%
  → low / severity 2

* < 55%
  → low / severity 3

* 115% ~ 145%
  → high / severity 1

* 145% ~ 170%
  → high / severity 2

* > 170%
  > → high / severity 3

### 특징

* 단백질은 부족 쪽을 더 중요하게 본다.
* 과다는 경고보다는 “목표 대비 높음” 정도로 완화해서 표현한다.

---

## 5. carbSignal 수정 기준

### 목표 대비 섭취율

* 85% ~ 115%
  → ok / severity 0

* 70% ~ 85%
  → low / severity 1

* 50% ~ 70%
  → low / severity 2

* < 50%
  → low / severity 3

* 115% ~ 135%
  → high / severity 1

* 135% ~ 160%
  → high / severity 2

* > 160%
  > → high / severity 3

---

## 6. fatSignal 수정 기준

### 목표 대비 섭취율

* 85% ~ 115%
  → ok / severity 0

* 70% ~ 85%
  → low / severity 1

* 50% ~ 70%
  → low / severity 2

* < 50%
  → low / severity 3

* 115% ~ 135%
  → high / severity 1

* 135% ~ 160%
  → high / severity 2

* > 160%
  > → high / severity 3

### 특징

* 지방은 과다 쪽을 더 민감하게 볼 수 있다.
* 다만 하루 단위 Coach에서는 단정적 표현을 피한다.

---

## 7. trainingSignal 세분화

trainingSignal도 severity 0~3 구조에 맞춘다.

### severity 0

* 운동 기록 있음
* 입력된 운동량과 섭취 상태가 크게 충돌하지 않음

### severity 1

* 운동량이 적거나 입력이 약함
* 또는 운동 기록은 있으나 해석에 필요한 정보가 부족함

### severity 2

* 목표 활동 대비 운동 기록이 부족함
* 또는 운동량이 있는데 섭취가 명확히 부족함

### severity 3

* 고강도 또는 긴 운동 기록이 있는데 섭취가 심하게 부족함
* 회복/에너지 부족 가능성을 강하게 안내해야 함

### 주의

* trainingSignal은 단독 처방 기준이 아니다.
* calorie / protein / carb / fat Signal과 함께 해석한다.

---

## 8. dataQualitySignal 세분화

dataQualitySignal도 severity 0~3 구조에 맞춘다.

### severity 0

* Today 입력 충분
* auto 평가 가능
* snapshot 또는 기준 데이터가 명확함

### severity 1

* 일부 입력 부족
* 해석은 가능하지만 확신도 낮음

### severity 2

* 식사 기록 부족
* manual/fallback 상태
* Coach는 참고 수준으로만 표현

### severity 3

* 핵심 입력이 거의 없음
* Coach는 상태 판단 대신 기록 보완 안내 중심으로 표시

---

## 9. 출력 톤 연결 규칙

### severity 0

* 정상 요약
* 필요 시 생략 가능

### severity 1

* 가벼운 조정 힌트
* 부드러운 표현 사용

### severity 2

* 명확한 상태 설명
* 이유와 행동 힌트 제공

### severity 3

* 강한 주의 문구
* 단정 대신 “현재 입력 기준” 표현 사용
* 기록 누락 가능성도 함께 고려

---

## 10. 향후 확장 여지

현재는 severity 0~3을 기본으로 한다.

다만 필요하면 이후 다음처럼 확장할 수 있다:

### severity 4

* 입력값이 비정상적으로 극단적인 경우
* 기록 오류 가능성이 높은 경우
* Coach 판단보다 입력 확인을 우선 안내

예:

* 목표 대비 0%
* 목표 대비 250% 이상
* 운동량 대비 섭취량이 극단적으로 낮은 경우

5.6에서는 severity 4를 필수 구현 대상으로 두지 않고,
향후 확장 가능성으로만 남긴다.

# 5.6 Daily Coach 문장 생성 규칙 (Signal → Coach 변환) [개정]

## 1. 문장 템플릿 고정 규칙

[상태]  
{영양소} 섭취가 {강도} {상태}입니다  

[이유]  
목표 대비 {설명}  

[행동]  
{행동 가이드}  

[주의]  
{dataQuality 기반 보정 문장}

---

## 2. 강도 표현 매핑

low  
- s1 → 약간 부족한  
- s2 → 부족한  
- s3 → 크게 부족한  

high  
- s1 → 약간 많은  
- s2 → 많은  
- s3 → 많이 높은  

ok  
- s0 → 적정  

---

## 3. Signal × severity 상태 문장 매핑

calorieSignal  
- low s1 → 총 섭취 열량이 약간 부족한 상태입니다  
- low s2 → 총 섭취 열량이 부족한 상태입니다  
- low s3 → 총 섭취 열량이 크게 부족한 상태입니다  
- high s1 → 총 섭취 열량이 약간 많은 상태입니다  
- high s2 → 총 섭취 열량이 많은 상태입니다  
- high s3 → 총 섭취 열량이 많이 높은 상태입니다  

proteinSignal  
- low s1 → 단백질 섭취가 약간 부족한 상태입니다  
- low s2 → 단백질 섭취가 부족한 상태입니다  
- low s3 → 단백질 섭취가 크게 부족한 상태입니다  
- high s1 → 단백질 섭취가 약간 많은 상태입니다  
- high s2 → 단백질 섭취가 많은 상태입니다  
- high s3 → 단백질 섭취가 많이 높은 상태입니다  

carbSignal  
- low s1 → 탄수화물 섭취가 약간 부족한 상태입니다  
- low s2 → 탄수화물 섭취가 부족한 상태입니다  
- low s3 → 탄수화물 섭취가 크게 부족한 상태입니다  
- high s1 → 탄수화물 섭취가 약간 많은 상태입니다  
- high s2 → 탄수화물 섭취가 많은 상태입니다  
- high s3 → 탄수화물 섭취가 많이 높은 상태입니다  

fatSignal  
- low s1 → 지방 섭취가 약간 부족한 상태입니다  
- low s2 → 지방 섭취가 부족한 상태입니다  
- low s3 → 지방 섭취가 크게 부족한 상태입니다  
- high s1 → 지방 섭취가 약간 많은 상태입니다  
- high s2 → 지방 섭취가 많은 상태입니다  
- high s3 → 지방 섭취가 많이 높은 상태입니다  

trainingSignal  
- s1 → 운동량이 부족하거나 해석 정보가 제한된 상태입니다  
- s2 → 운동량 대비 섭취가 부족한 상태입니다  
- s3 → 운동량 대비 에너지 부족이 큰 상태입니다  

dataQualitySignal  
- s1 → 입력 정보가 일부 부족한 상태입니다  
- s2 → 기록 기반 해석이 제한된 상태입니다  
- s3 → 기록 정보가 부족하여 정확한 판단이 어렵습니다  

---

## 4. 이유 문장 규칙

low → 목표 대비 섭취량이 부족한 수준입니다  
high → 목표 대비 섭취량이 많은 수준입니다  
severity 3 → 목표 대비 차이가 큰 상태입니다  

---

## 5. 행동 문장 규칙

low → 다음 식사에서 해당 영양소를 보충해보세요  
high → 다음 식사에서 해당 영양소를 줄여보세요  

severity 3 → 우선적으로 해당 부분을 조정하는 것이 좋습니다  

---

## 6. 다중 Signal 출력 규칙

severity 3 → 전부 출력  
severity 2 → 최대 2개 출력  
severity 1 → 보조로 1개만 출력  
severity 0 → 생략  

calorie / protein이 동시에 문제면 둘 다 출력  
carb / fat은 보조 Signal로 취급하되 severity 3이면 출력한다  

---

## 7. dataQuality 적용 규칙

s0 → 기본 문장 유지  

s1 → "현재 입력 기준으로는" 추가  

s2 이상 →  
"현재 기록 기준으로는 ~로 보입니다"  
+  
"정확한 분석을 위해 기록을 보완해보세요"  

---

## 8. 최종 출력 순서

1. 주요 상태  
2. 이유  
3. 행동  
4. 주의  

---

## 9. 완료 조건

- 템플릿 고정 완료  
- 매핑 정의 완료  
- 다중 Signal 규칙 반영  
- dataQuality 보정 규칙 반영
