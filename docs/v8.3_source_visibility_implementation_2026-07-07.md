# v8.3 source visibility implementation

작성일: 2026-07-07
변경 성격: narrow implementation
score formula 변경: 없음
anchor / curve / penalty 값 변경: 없음
storage/schema/backup/Recent 변경: 없음
scoreDeltaPreview 변경: 없음
old records migration/recompute/reset: 없음

## 1. 목적

이 구현은 `docs/v8.3_score_source_visibility_decision_2026-07-07.md`에서 닫은 source visibility 정책을 Today live score surface에 반영한다.

핵심 판단:
- complete macro input이 있고 점수 계산이 가능한 날은 missing total burn만으로 null/blocked하지 않는다.
- 다만 total burn 기준이 없으면 TDEE overload 축이 빠진 점수이므로 사용자에게 해석 참고 문구를 보여준다.
- body weight가 실제 체중이 아니라 target protein fallback에서 온 경우에도 점수는 유지하되 사용자에게 해석 참고 문구를 보여준다.
- warning은 score penalty가 아니며 anchor tuning도 아니다.
- warning은 storage/schema field가 아니다.
- stored current-version auto score는 hidden recompute하지 않는다.
- no meals / one meal / insufficient / snapshotless / invalid context는 계속 null/blocked gate다.

## 2. 구현 내용

`index.html`에 다음 helper를 추가했다.

```text
getAdherenceSourceVisibilityFlags(adherenceResult)
getAdherenceSourceVisibilityNotes(adherenceResult)
renderAdherenceSourceVisibilityNotes(adherenceResult)
```

읽는 source:
- `adherenceResult.detail.scoringContext.tdeeAvailable`
- `adherenceResult.detail.scoringContext.totalBurnSource`
- `adherenceResult.detail.scoringContext.weightSource`

warning 조건:
- `status === "available"`
- `isAvailable === true`
- numeric `percent`
- live `detail.scoringContext` 존재
- missing total burn: `tdeeAvailable === false` 또는 `totalBurnSource === "missing"`
- inferred weight: `weightSource === "targetProteinFallback"`

warning이 붙지 않는 경우:
- `stored_auto_snapshot`
- unavailable / pending / blocked result
- `detail.scoringContext`가 없는 result

## 3. 사용자 문구

표시 문구:
- `오늘 총소비량 기준이 없어, 목표 대비 섭취량 중심으로 점수를 해석합니다.`
- `체중 기준이 없어, 단백질 목표에서 추정한 체중 기준으로 일부 항목을 해석합니다.`

사용자 화면에 노출하지 않는 내부 용어:
- `TDEE`
- `scoringContext`
- `penalty`
- `anchor`
- `curve`
- `source visibility`
- `targetProteinFallback`

## 4. UI 위치

`renderTodayAdherencePanel(...)`의 available score detail card 안에서 `renderAdherenceCompactEvidenceList(...)` 바로 아래에 source visibility notes를 렌더링한다.

의도:
- 오늘 점수의 해석 참고를 점수 근거 근처에서 보여준다.
- 새 storage field나 Recent aggregation을 만들지 않는다.
- UI 전체 redesign을 하지 않는다.
- public breakdown panel을 열지 않는다.

## 5. 테스트 보강

`runMacroRangeContinuousScoringTests`에 source visibility regression을 추가했다.

고정한 것:
- missing total burn + inferred weight fallback이어도 complete macro day는 numeric score를 유지한다.
- user-facing source warning copy가 생성된다.
- source warning copy에 내부 용어가 나오지 않는다.
- stored current-version auto snapshot은 live source warning을 붙이지 않는다.
- pending/blocked result는 warning numeric score로 승격하지 않는다.

`runAdherenceUiTests`에 Today panel 렌더링 regression을 추가했다.

고정한 것:
- Today adherence panel에 source notes가 표시된다.
- 렌더링된 사용자 화면 text에 내부 용어가 나오지 않는다.
