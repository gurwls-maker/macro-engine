## 1. 구현 범위

추가한 suite:

```text
runMacroRangeRecentMixedBasisDesignTests
```

추가 위치:

```text
index.html
```

runner 연결:

```text
package.json
  test:macro-policy

tools/render_audit/run_internal_tests.cjs
  core profile
```

이 suite는 production Recent를 range-aware로 바꾸지 않는다.

대신 suite 내부 test-local detector로 raw fixture를 분류하고, production Recent summary/count/window/DailyCoach output은 그대로 유지되는지 확인한다.

## 2. test-local detector

suite 내부 후보:

```text
detectRecentMixedBasisForTest(records)
```

출력 성격:

```text
reportOnly: true
persisted: false
uiVisible: false
productionRecentChanged: false
```

분류:

```text
fixedBasisDays
rangeBasisDays
invalidRangeBasisDays
unsupportedRangeBasisDays
componentIneligibleDays
mixedBasisDetected
recommendedAggregationMode
reasons
```

금지:

```text
detector output에 fatLowDays / fatHighDays / proteinLowDays / carbLowTrainingDays를 만들지 않는다.
detector output에 score / adherenceScore / candidateScorePreview를 만들지 않는다.
detector를 buildRecentContextWindow(), getRecentContextComponentCounts(), DailyCoach에 연결하지 않는다.
```

## 3. fixture

fixture는 다음을 분리한다.

```text
old fixed record:
  macroRangeContract 없음
  현재 Recent production source of truth

range candidate record:
  raw goalSnapshot에 macroRangeContract / macroRangeBasisSignature 후보가 있음
  production normalize 경로에서는 fixed target만 남음
  test-local detector에서만 rangeBasisDays로 분류

invalid range candidate:
  fat/protein/carb order violation 또는 targetCal mismatch 등
  rangeBasisDays로 세지 않음

unsupported range candidate:
  unsupported version
  unsupportedRangeBasisDays로 분리

component-ineligible:
  one-meal pending 또는 snapshotless
  mixed basis count를 만들지 않음
```

## 4. 추가 테스트

`runMacroRangeRecentMixedBasisDesignTests`는 12개 케이스로 구성했다.

```text
1. current summary ignores unknown range fields
2. current component counts unchanged for pruned range fixture
3. detector counts fixed basis days
4. detector counts range candidate without production Recent changes
5. detector flags fixed/range mixed basis
6. invalid range candidate is not included in range counts
7. unsupported range candidate is explicit
8. component-ineligible records do not create basis counts
9. range candidate does not alter production macroAnalysis and fat counts
10. Recent windows remain [1,3,7,14,28]
11. candidateScorePreview does not enter Recent output
12. detector does not change DailyCoach output
```

## 5. production no-impact 확인

이번 suite에서 확인한 production 경계:

```text
getRecentContextRecordSummary() output에 macroRangeContract / macroRangeBasisSignature가 없음
getRecentContextComponentCounts() output이 fixed fixture와 range raw fixture에서 동일함
buildRecentContextWindow()의 macroAnalysisDays / fatLowDays / fatHighDays가 range raw fixture 때문에 바뀌지 않음
RECENT_CONTEXT_WINDOWS는 [1,3,7,14,28] 유지
candidateScorePreview가 Recent summary/context에 들어가지 않음
DailyCoach output이 detector 실행 전후 동일함
```
