## 비개발자용 설명

### 무엇을 바꿨는지

앱 화면과 실제 점수 계산은 바꾸지 않았다. 대신 탄수화물과 지방의 조합 문제를 별도 감점으로 남길지 판단하기 위해, 가능한 탄수·지방 조합 범위에서 얼마나 벗어났는지를 계산하는 Option C를 더 촘촘하게 시뮬레이션했다.

이번 검사는 한 가지 예시만 본 것이 아니다. 실제 앱이 만드는 목표 조건 66개에서 탄수·지방 조합을 정해진 grid로 3,058번 움직여 다음을 함께 확인했다. 이 숫자는 실제 사용자에게 얼마나 자주 나타나는지를 뜻하는 prevalence가 아니라, 두 종류가 모두 존재하는지 반증하기 위한 deterministic geometry sample 수다.

- 조합 범위 안에서는 residual이 0인지
- 범위를 조금 벗어날 때 값이 갑자기 뛰지 않는지
- 더 멀리 벗어날수록 값이 한 방향으로 커지는지
- 같은 칼로리 비율을 다른 수학 표현으로 계산해도 같은 결과인지
- 이 residual이 기존 탄수·지방·총칼로리 감점과 겹치는지

결론은 “Option C로 교체해도 된다”도 아니고 “조합 감점을 없애도 된다”도 아니다. 실제 사용 기록에서 무엇을 별도 문제로 볼지 판단할 근거가 더 필요하다.

### 왜 바꿨는지

현재 조합 감점은 경계에서 갑자기 켜졌다가 꺼질 수 있다. 사용자가 실제 화면에서 재현한 값은 약 `20.6점 -> 0점`이었고, 같은 섭취 전환을 고정 fixture와 현재 곡선으로 다시 계산한 값도 `22.449점 -> 0점`으로 같은 절벽을 보였다. 선행 문서의 `21.392점`은 당시 test-only linear interpolation 값이고, 이번 `22.449점`은 production과 같은 smoothstep interpolation으로 바로잡은 exact harness 값이다. 과거 문서를 소급 변경하지 않고 이 문서에서 차이를 명시한다. 이런 구조는 유지하기 어렵다.

Option C의 거리 자체는 그 구간에서 `0.354111`에서 `0.458886`으로 연속적으로 움직였다. 기존 감점 곡선을 시험 삼아 연결하면 `5.702점`에서 `9.147점`이 된다. 하지만 이 숫자는 승인된 새 벌점이 아니다. 연속적인 거리를 만들 수 있다는 것과 그 거리를 실제 점수로 몇 점 반영해야 한다는 것은 서로 다른 판단이다.

3,058개 deterministic grid 표본 중 residual이 양수인 경우는 477개였다. 그중 237개는 기존 탄수·지방·총칼로리 감점이 모두 0인 `core-unique` 사례였고, 240개는 기존 core 감점과 겹치는 `core-overlap` 사례였다. 이 비율은 실제 사용자 분포가 아니라 두 구조가 production geometry에 모두 존재한다는 증거다. 지금 교체하면 중복감점 위험을 남기고 지금 제거하면 별도 조합 신호를 잃을 수 있다.

### 5. privacy-safe actual-day evidence 계약

다음 gate의 입력은 사용자가 명시적으로 제공한 backup이어야 한다. 현재 sanitizer는 raw meal name, memo, body metrics, date 같은 원본 필드를 결과 JSON/CSV에 내보내지 않고 extraction/count/distribution과 익명 대표 표본까지만 만든다.

- residual-positive / core-unique / core-overlap count
- residual 방향과 크기
- current score와 aggregation probe의 익명 요약
- 표본 부족/범주 누락 표시

actual-day matched-pair grouping은 아직 구현하지 않았다. 다음 gate에서는 다음 계약을 먼저 닫아야 한다.

- joint를 제외한 energy/carb/fat core penalty vector, goal, training context, source-validity 상태를 matching 축으로 사용한다.
- exact match와 여러 tolerance sensitivity를 함께 보고하며, 한 tolerance만 골라 결론을 만들지 않는다.
- unmatched count, coverage, 각 residual 방향의 표본 수를 모두 공개한다.
- rejected current joint가 포함된 현재 final score나 aggregation probe를 독립 정답으로 사용하지 않는다.
- core-matched residual contrast가 tolerance sensitivity에서 안정성을 잃고 actual-day residual-positive가 기존 core 설명으로만 남으면 `DROP_JOINT_AXIS_READY` 후보로 보낸다.
- core-unique가 여러 goal/training 맥락에서 반복되고, current score를 참조하지 않아도 joint-feasibility라는 별도 제품 의미를 설명하며, overlap 사례를 0 또는 deduplicated 처리할 수 있을 때만 `CURRENT_JOINT_REPLACEMENT_READY` 후보로 보낸다. curve coefficient 승인은 그 뒤 별도다.
- 두 구조가 계속 섞이거나 표본/coverage가 부족하면 production을 열지 않고 `MORE_EVIDENCE_REQUIRED`를 유지한다. coefficient만 바꾸는 반복 loop는 허용하지 않는다.

실제 backup이 없으면 `evidence unavailable`로 끝내며 합성 자료로 사용자 결과를 대신하지 않는다.
