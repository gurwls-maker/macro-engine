## Original intent preservation

이번 결정은 세션 원문에서 반복된 아래 의도를 기준으로 한다.

- 사용자는 "회복재료", "성장재료"처럼 AI가 만든 듯한 표현을 싫어한다.
- 테스트가 어색한 문구를 고정하고 있으면 앱 문구를 테스트에 맞추지 말고 test expectation 갱신도 허용한다.
- 다만 의미 보호 테스트까지 취향 문제로 약화하면 안 된다.
- "한 줄씩 읽는" app-wide copy audit이 필요하다. 단, 이번 gate는 실제 구현이 아니라 inventory / decision이다.
- 과잉 설명도 AI스러움의 한 형태다. UI 헤더, 숫자, 단위, 카드 위치로 충분히 알 수 있는 것은 반복 설명하지 않는다.
- 사람이 만든 앱 문구처럼 보이려면 사용자에게 익숙한 말, 짧은 구어체, 화면 맥락을 믿는 생략이 필요하다.
- 전문 설명은 기본 화면에 길게 깔지 않고 tooltip/glossary/spec 위치로 분리한다.
- scoring, UI redesign, storage/schema, scoreDeltaPreview, old records cleanup/reset/fallback은 이번 copy inventory와 섞지 않는다.

## 문제 유형

- AI_SOUNDS: 번역투, 문서체, "재료", "흐름" 남발처럼 사람 앱 문구보다 AI 설명문처럼 들림.
- TOO_TECHNICAL: 내부 산식/데이터/전문 용어가 기본 화면에 직접 노출됨.
- TOO_VAGUE: 그래서 사용자가 뭘 해야 하는지 불분명함.
- MISLEADING: 목표/범위/점수/운동/알코올 의미를 잘못 이해시킬 수 있음.
- TOO_ALARMING: 필요 이상으로 겁줌.
- TOO_SOFT: 실제 위험을 너무 약하게 넘김.
- OVER_EXPLAINED: UI 맥락상 이미 알 수 있는 정보를 문장으로 다시 설명함.
- UI_REDUNDANT: 카드 제목, 단위, chip, progress, 버튼명이 이미 말하는 내용을 본문에서 반복함.
- TEST_LOCKED_BAD_COPY: 테스트가 어색한 문구 자체를 고정하고 있음.
- OK_KEEP: 현재 유지 가능.
- NEEDS_MORE_CONTEXT: 화면 전체 맥락이나 tooltip/spec 없이는 판단 보류.
