## 2. 테스트가 고정한 것

이번 suite는 다음을 테스트한다.

```text
candidateScorePreview null 유지
scoreDeltaPreview null 유지
component pointsPreview null 유지
needsDecision inventory 유지
old fixed Records no-reinterpret gate 유지
one-meal / insufficient / snapshotless data quality gate 유지
forbidden score candidate fields snapshot / backup no-leak
production score / scoring version / snapshot signature / backup / Recent / UI / DailyCoach no-impact
```

## 3. component gate

테스트한 component 상태:

```text
kcal high
protein low
fat below_action_floor
fat below_recommended
fat above_recommended
fat above_flexible
carb below_training_support
carb above_candidate_range
```

각 case는 관련 policy decision이 unresolved로 남아야 하며 numeric preview를 만들지 않는다.
