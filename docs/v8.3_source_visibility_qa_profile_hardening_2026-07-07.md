## 1. 목적

`docs/v8.3_source_visibility_implementation_2026-07-07.md`에서 Today live score surface에 source visibility notes를 구현했다.

그 구현은 `runAdherenceUiTests`에 regression test를 추가했지만, 실제 profile 확인 결과:
- `runAdherenceUiTests`는 `index.html`에 존재한다.
- `window.runAdherenceUiTests` export도 존재한다.
- `full` profile은 all exported run*Tests로 잡는다.
- `smoke` profile 명시 목록에는 없었다.
- `core` profile 명시 목록에도 없었다.

판단:
- source visibility warning은 Today 점수 카드 사용자 표면이다.
- `npm test`는 `test:smoke`를 실행하므로, smoke에서 이 UI regression을 놓치면 기본 검증 루틴이 약해진다.
- `test:core`는 구현 작업 후 자주 실행되는 core regression gate이므로, core에서도 이 UI regression을 잡아야 한다.

## 2. 변경 내용

`tools/render_audit/run_internal_tests.cjs`의 profile에 `runAdherenceUiTests`를 추가했다.

추가 위치:
- `smoke`
- `core`
- `ui`

의도:
- source visibility UI regression을 `npm test`에서 잡는다.
- source visibility UI regression을 `npm run test:core`에서 잡는다.
- source visibility UI regression을 `npm run test:ui`에서도 잡는다.
- full profile 전용 guard로 방치하지 않는다.
