## 5. display sanity audit

raw v8.3 scoring no-penalty anchor를 UI range로 그대로 쓰지 않았다.

특히 high-volume carb context는 raw policy상 `10.5 g/kg`까지 열려 있으므로, 체중 75kg 기준 raw upper는 약 788g이다.
이를 그대로 `오늘 범위`에 표시하면 너무 넓어 보인다.

그래서 display helper는 v8.3 policy source를 참고하되 target-based sanity band를 함께 적용한다.

샘플 결과:

```text
S01 REST, 체중 75kg, target carbs 300g
-> carbs 오늘 범위 240~315g

S02 NORMAL_RESISTANCE, 체중 75kg, target carbs 300g
-> carbs 오늘 범위 264~405g

S03 HIGH_VOLUME_OR_LONG_SESSION, 체중 75kg, target carbs 300g
-> carbs 오늘 범위 300~465g

S04 UNKNOWN, 체중 75kg, target carbs 300g
-> carbs 오늘 범위 246~360g

S05 protein target 150g, 체중 75kg
-> protein 오늘 범위 120~180g

S06 fat target 70g, fat range source min 45g / max 90g
-> fat 오늘 범위 45~90g

S07 missing TDEE, high-volume carb context
-> carbs 오늘 범위 300~465g
-> macro card에 source warning을 반복하지 않음

S08 targetProteinFallback weightSource
-> protein 오늘 범위 120~180g
```

판단:

```text
- raw no-penalty anchor가 너무 넓게 UI에 노출되지 않는다.
- target g와 오늘 범위가 완전히 무관하게 보이지 않는다.
- source visibility warning은 Today score surface에 남고, macro card에는 반복하지 않는다.
```

## 6. copy / UI 정책

```text

```

피한 copy:

```text
오늘 해석 범위
회복재료
회복 재료
하한 / 상한 primary card copy
운동했으니 괜찮음
```

테스트 정책:

```text
문구 테스트가 낡은 단일 target 관성이나 AI스러운 표현을 고정하면 앱 문구를 후퇴시키지 않는다.
필요하면 test expectation을 user-facing copy 기준으로 갱신한다.
```

이번 구현에서는 기존 test expectation에 억지로 맞추는 변경은 하지 않았다.
