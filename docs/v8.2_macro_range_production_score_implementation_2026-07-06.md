# v8.2 macro range production score implementation

## 목적

이 문서는 v8.2 macro range score를 실제 production score path로 전환한 구현 기록이다.

핵심은 `scoreDeltaPreview` 비교 라인이 아니라, 기존 fixed target score를 range-aware adherence score로 교체하는 것이다. 이번 구현은 `getDailyAdherenceScore`와 `ADHERENCE_SCORING_VERSION` bump를 같은 gate에서 열었다.

## 구현 요약

- `ADHERENCE_SCORING_VERSION`을 `v8.2_macro_range_score_v1`로 변경했다.
- `LEGACY_FIXED_ADHERENCE_SCORING_VERSION = "v6.1_alcohol_penalty_v2"`를 분리해 old fixed fixture basis가 새 production version으로 덮이지 않게 했다.
- `getDailyAdherenceScore` production path를 fixed-ratio weighted score에서 range-aware component contribution formula로 교체했다.
- production formula는 `clamp(100 + kcal + protein + fat + carbExchange, 0, 100)`이다.
- `dataQuality`는 score component가 아니라 eligibility/null gate로 유지한다.
- new detailed auto Records는 기존 저장 계약인 `adherencePercent` / `adherenceScoringVersion`만 사용한다.
- `pointsPreview`, `candidateScorePreview`, `scoreDeltaPreview`, `rangeScoreCandidateSummary` 저장/노출 field는 열지 않았다.

## Component Contribution

```text
kcal:
- matched 0
- high / low -10
- severe_high / severe_low -16

protein:
- satisfied 0
- low -16
- severe_low -24

fat:
- inside_recommended 0
- below_recommended / above_recommended -5
- below_action_floor / above_flexible -12

carbExchange:
- exchange_explained / inside_candidate_range 0
- below_training_support -8
- above_candidate_range -6
```

`componentScores`는 fixed-ratio score source로 더 이상 사용하지 않는다. v8.2 detail에는 `componentContributions`, `componentStatuses`, `componentRatios`, `rangeScoreFormula`를 둔다.

## Alcohol Semantics

알코올 kcal는 `scoringKcal`에 들어가 kcal contribution에 반영된다. 그러나 range-aware score 산출 뒤 `alcoholImpactPenalty`를 별도로 한 번 더 빼지 않는다.

DailyCoach의 alcohol severity/action signal은 유지한다. 즉 알코올은 score 후처리 감산이 아니라 총칼로리와 코칭 신호에서 다룬다.

## Records Policy

new detailed auto Record:

- `adherencePercent`: v8.2 range-aware score number
- `adherenceScoringVersion`: `v8.2_macro_range_score_v1`

blocked / pending / no meals:

- `adherencePercent`: `null`
- `adherenceScoringVersion`: `null`
- sentinel `0` 금지

old fixed Record:

- saved `adherencePercent` 유지
- saved `adherenceScoringVersion` 유지
- no silent mutation
- production transition blocker로 확대하지 않음
