## 2. field name decision

예약 이름:

```text
scoreDeltaPreview
```

의미:

```text
fixed production score와 future candidateScorePreview 사이의 transient comparison delta 후보
저장 점수가 아닌 비교 후보
DailyCoach action이 아닌 audit / explainability 후보
Recent trend가 아닌 report-only comparison 후보
```

금지 이름:

```text
scoreDelta
deltaScore
scoreChange
improvementScore
adherenceDelta
productionScoreDelta
savedScoreDelta
```

이유:

```text
Preview는 저장 계약이 아님을 드러낸다.
Delta는 fixed score 자체가 아니라 비교 결과 후보임을 드러낸다.
Score라는 단어가 들어가므로 위치와 nullability를 엄격히 제한해야 한다.
```

## 5. why last gate

scoreDeltaPreview는 마지막 gate다.

이유:

```text
fixed production score와 candidate preview score를 직접 비교한다.
candidateScorePreview가 숫자가 아니면 delta 의미가 없다.
component pointsPreview formula가 안정되지 않으면 candidate total의 의미가 없다.
old fixed Records에서는 saved adherencePercent의 의미를 흔들 수 있다.
사용자는 delta를 production 개선/악화로 읽을 수 있다.
UI / Recent / DailyCoach에 새면 report-only 후보가 행동 권고처럼 보일 수 있다.
```

따라서:

```text
component pointsPreview numeric gate 통과 != scoreDeltaPreview readiness
candidateScorePreview numeric gate 통과 != scoreDeltaPreview readiness
consumer no-leak guard 통과 != scoreDeltaPreview visibility readiness
```

scoreDeltaPreview는 candidate total이 실제 숫자로 안정된 뒤에도 별도 policy / test / visibility gate가 필요하다.

## 7. candidateScorePreview relationship

```text

```

현재 유지:


