## 1. current baseline

직전 shape decision에서 닫힌 것:

```text
pointsPreview field name 예약
allowed location: rangeScoreCandidateSummary.componentPreviews.<component>.pointsPreview
default: null
pointsPreviewStatus 후보: not_computed_policy_unresolved / blocked / computed_candidate
computed_candidate는 별도 numeric policy gate 전까지 미개방
dataQuality.pointsPreview: null-only
top-level / record / goalSnapshot / backup pointsPreview 금지
UI / Recent / DailyCoach visibility: false
scoreDeltaPreview 별도 gate 유지
```

이미 test-only로 열린 것:

```text
testLocalNumericPreview.testLocalCandidateScorePreview
testLocalNumericPreview.testLocalComponentPoints
testLocalNumericPreview.testLocalCandidateScorePreviewVersion
testLocalNumericPreview.testLocalFixedScoreBasisLabel
testLocalNumericPreview.testLocalCandidateScoreBasisLabel
```

중요:

```text
testLocalNumericPreview의 숫자는 production-facing candidateScorePreview / pointsPreview / scoreDeltaPreview가 아니다.
이번 문서는 testLocalNumericPreview를 production field로 승격하지 않는다.
```

## 5. component assertion design

### 5-1. kcal

테스트 설계:

```text
kcal matched -> neutral lane, pointsPreview null
kcal high -> material lane, pointsPreview null
kcal low -> material lane, pointsPreview null
```

검증 목적:

```text
kcal은 anchor component다.
matched는 보너스가 아니다.
high / low는 future penalty candidate일 수 있지만 이번 gate에서 숫자화하지 않는다.
```

아직 미결정:

```text
kcal tolerance
kcal high / low 대칭 여부
kcal deduction amount
```

### 5-2. protein

테스트 설계:

```text
protein satisfied -> neutral lane, pointsPreview null
protein low -> strong lane, pointsPreview null
```

검증 목적:

```text
protein은 anchor component다.
protein satisfied는 보너스가 아니다.
protein low는 fat / carb exchange로 상쇄하지 않는다.
```

아직 미결정:

```text
protein low threshold
goal별 severity 차등
protein deduction amount
```

### 5-3. fat

테스트 설계:

```text
fat below_action_floor -> strong lane, pointsPreview null
fat below_recommended -> observation lane, pointsPreview null
fat inside_recommended -> neutral lane, pointsPreview null
fat above_recommended -> observation lane, pointsPreview null
fat above_flexible -> strong lane, pointsPreview null
```

검증 목적:

```text
inside_recommended는 보너스가 아니다.
below_recommended / above_recommended는 observation일 뿐 actual deduction 확정이 아니다.
below_action_floor / above_flexible은 future strong penalty candidate지만 이번 gate에서 숫자화하지 않는다.
```

아직 미결정:

```text
fat threshold production value
observation lane의 실제 감점 여부
below_action_floor / above_flexible 감점 대칭 여부
```

### 5-4. carbExchange

테스트 설계:

```text
carb exchange_explained -> neutral lane, pointsPreview null
carb inside_candidate_range -> neutral lane, pointsPreview null
carb below_training_support -> material lane, pointsPreview null
carb above_candidate_range -> material lane, pointsPreview null
```

검증 목적:

```text
exchange_explained는 보너스가 아니다.
inside_candidate_range는 보너스가 아니다.
below_training_support / above_candidate_range는 future material penalty candidate지만 이번 gate에서 숫자화하지 않는다.
```

아직 미결정:

```text
training day carb support threshold
goal별 carb high policy
fat / carb exchange 우선순위
```

### 5-5. dataQuality

테스트 설계:

```text
full_day_candidate -> eligibility lane, pointsPreview null
one_meal_pending -> blocked lane, pointsPreview null
insufficient_record -> blocked lane, pointsPreview null
snapshotless_debug_only -> blocked lane, pointsPreview null
blocked_by_impossible_state -> blocked lane, pointsPreview null
```

검증 목적:

```text
dataQuality는 score component가 아니다.
dataQuality.pointsPreview는 null-only다.
blocked 상태를 sentinel 0으로 표현하지 않는다.
```

아직 미결정:

```text
dataQuality sentinel 0 필요 여부
full_day_candidate exact eligibility definition
new range-basis record eligibility
```

## 7. old fixed Records policy

old fixed Records test design:

```text
sufficient detailed old fixed Record -> transient eligibility candidate로 분류 가능
simple old fixed Record -> numeric policy 대상 아님
weight_only old fixed Record -> numeric policy 대상 아님
snapshotless old fixed Record -> blocked / debug-only
insufficient old fixed Record -> blocked
```

계속 금지:

```text
saved adherencePercent overwrite
saved adherenceScoringVersion overwrite
backup payload에 pointsPreview 추가
Records UI에 component points 표시
Recent aggregation에 component points 사용
DailyCoach reason/action에 component points 사용
```

검증 방향:

```text
old fixed Records는 latest-policy preview eligibility 후보가 될 수 있지만, 저장 score basis는 fixed로 유지한다.
```

## 9. relationship with candidateScorePreview

이번 test design으로 열지 않는 것:

```text
candidateScorePreview = sum(componentPreviews.*.pointsPreview)
candidateScorePreview fallback from fixed score
candidateScorePreview persistence
candidateScorePreview UI display
```

관계:

```text
pointsPreview numeric policy test design은 component guard 설계다.
candidateScorePreview total formula는 별도 numeric policy gate다.
```

따라서:

```text
component test design 완료 != candidateScorePreview 숫자화 readiness
component test design 완료 != production score readiness
```

## 11. promoted payload rejection design

```text
{
  pointsPreview: { kcal: 0 },
  candidateScorePreview: 92,
  scoreDeltaPreview: 4,
  rangeScoreCandidateSummary: {
    candidateScorePreview: 92,
    componentPreviews: {
      kcal: { pointsPreview: 0 }
    }
  },
  goalSnapshot: {
    pointsPreview: { kcal: 0 }
  }
}
```

기대 assertion:

```text
top_level:pointsPreview rejected
top_level:candidateScorePreview rejected
top_level:scoreDeltaPreview rejected
rangeScoreCandidateSummary.candidateScorePreview rejected
componentPreviews.kcal.pointsPreview rejected
goalSnapshot.pointsPreview rejected
comparison.scoreDeltaPreview rejected
```

주의:

```text
reject는 production sanitizer 구현 지시가 아니다.

```
