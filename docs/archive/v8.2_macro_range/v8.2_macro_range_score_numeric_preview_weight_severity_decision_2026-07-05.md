## 3. weight lane decision

숫자 weight를 정하지 않고 lane만 정한다.

```text
anchor lane:
- kcal
- protein

range evidence lane:
- fat
- carbExchange

eligibility lane:
- dataQuality
```

원칙:

```text
kcal mismatch는 fat/carb range 설명으로 덮지 않는다.
protein low는 fat/carb exchange로 상쇄하지 않는다.
fat range는 exact target penalty를 완화할 수 있는 설명 후보지만 보너스가 아니다.
carb exchange는 fat range와 함께 해석하는 설명 후보지만 보너스가 아니다.
dataQuality는 점수 component가 아니라 numeric preview eligibility gate다.
```

아직 닫지 않는 것:

```text
anchor lane과 range evidence lane의 실제 비율
fat과 carbExchange의 상대 weight
kcal high / low의 대칭 여부
protein low의 goal별 weight
dataQuality를 score에서 제외할지, blocked 상태로만 둘지
```

## 4. severity lane decision

숫자 감점 폭을 정하지 않고 severity lane만 정한다.

```text
blocked:
- one_meal_pending
- insufficient_record
- snapshotless_debug_only
- blocked_by_impossible_state

strong penalty candidate:
- protein low
- fat below_action_floor
- fat above_flexible

material penalty candidate:
- kcal high
- kcal low
- carb below_training_support
- carb above_candidate_range

observation candidate:
- fat below_recommended
- fat above_recommended

neutral / explanation candidate:
- kcal matched
- protein satisfied
- fat inside_recommended
- carb exchange_explained
- carb inside_candidate_range
- full_day_candidate eligibility
```

주의:

```text
severity lane은 pointsPreview가 아니다.
severity lane은 production DailyCoach severity와 다르다.
severity lane은 ADHERENCE_SCORING_VERSION의 의미를 바꾸지 않는다.
neutral / explanation candidate는 점수 상승 또는 보너스가 아니다.
```

## 5. component decision

### 5-1. kcal

문서상 결정:

```text
targetCal은 authoritative다.
kcal high / low는 material penalty candidate lane이다.
kcal matched만으로 점수 보너스를 만들지 않는다.
```

계속 미결정:

```text
kcal tolerance
kcal high / low 감점 대칭 여부
fixed score tolerance와 같은 값을 쓸지
```

### 5-2. protein

문서상 결정:

```text
protein은 anchor lane이다.
protein low는 strong penalty candidate lane이다.
protein low는 carb/fat exchange로 상쇄하지 않는다.
protein satisfied만으로 점수 보너스를 만들지 않는다.
```

계속 미결정:

```text
protein low threshold
goal별 protein severity 차등
body weight / FFM / adjusted weight basis
```

### 5-3. fat

문서상 결정:

```text
below_action_floor는 strong penalty candidate lane이다.
above_flexible은 strong penalty candidate lane이다.
below_recommended는 observation candidate lane이다.
above_recommended는 observation candidate lane이다.
inside_recommended는 neutral lane이다.
inside_recommended만으로 점수 보너스를 만들지 않는다.
```

계속 미결정:

```text
below_recommended를 실제 감점으로 볼지
above_recommended를 실제 감점으로 볼지
below_action_floor와 above_flexible 감점 대칭 여부
fat range threshold의 production 확정값
```

### 5-4. carbExchange

문서상 결정:

```text
below_training_support는 material penalty candidate lane이다.
above_candidate_range는 material penalty candidate lane이다.
exchange_explained는 neutral / explanation lane이다.
inside_candidate_range는 neutral lane이다.
exchange_explained만으로 점수 보너스를 만들지 않는다.
```

계속 미결정:

```text
training day carb minimum의 실제 감점 여부
goal별 carb high penalty 차등
fat below_action_floor와 carb high가 같이 있을 때 우선순위
```

### 5-5. dataQuality

문서상 결정:

```text
full_day_candidate만 future numeric preview 후보가 될 수 있다.
one_meal_pending은 blocked lane이다.
insufficient_record는 blocked lane이다.
snapshotless_debug_only는 blocked lane이다.
blocked_by_impossible_state는 blocked lane이다.
```

계속 미결정:

```text
full_day_candidate의 exact eligibility definition
snapshot이 있는 Today debug fixture 허용 여부
new range-basis record의 future eligibility
```

## 7. scoreDeltaPreview gate

