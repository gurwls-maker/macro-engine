## 1. 구현 요약

추가:

```text
MACRO_RANGE_SCORE_TEST_LOCAL_NUMERIC_PREVIEW_VERSION
MACRO_RANGE_SCORE_TEST_LOCAL_CANDIDATE_BASIS_LABEL
getMacroRangeScoreReportOnlyTestLocalKcalPenalty()
getMacroRangeScoreReportOnlyTestLocalProteinPenalty()
getMacroRangeScoreReportOnlyTestLocalFatPenalty()
getMacroRangeScoreReportOnlyTestLocalCarbExchangePenalty()
buildMacroRangeScoreReportOnlyTestLocalNumericPreview()
```

연결:

```text
buildMacroRangeScoreCandidateReportOnly()
  -> testLocalNumericPreview
```

새 suite는 추가하지 않았다. 기존 guard suite를 실제 output 기준으로 보강했다.

## 2. 허용 output

`buildMacroRangeScoreCandidateReportOnly()`는 이제 다음 nested object를 포함한다.

```text

  scoreVersionChanged: false,
  snapshotSchemaChanged: false,
```

숫자 후보는 이 object 안에만 존재한다.

```text
rangeScoreCandidateSummary.candidateScorePreview
```

## 4. numeric policy shape

이번 numeric preview는 production formula가 아니다.

성격:

```text
deterministic report-only nested test-local preview
penalty-only
0..100 clamp
no positive component points
inside_recommended no bonus
exchange_explained no bonus
kcal mismatch cannot be offset by fat/carb range
protein low cannot be offset by fat/carb exchange
dataQuality is eligibility gate
```

## 5. blocked record handling

blocked 상태에서는 nested shell만 남기고 숫자는 null이다.

예:

```text
testLocalCandidateScorePreview: null
testLocalComponentPoints: null
testLocalBlockedReason: "one_meal_pending" | "insufficient_record" | "snapshotless_debug_only" | ...
```

full-day candidate일 때만 nested numeric preview를 만들 수 있다.
