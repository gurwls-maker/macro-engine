## 4. version assertions

future implementation acceptance:

```text
ADHERENCE_SCORING_VERSION === "v8.2_macro_range_score_v1"
```

금지:

```text
formula만 바꾸고 ADHERENCE_SCORING_VERSION을 v6.1_alcohol_penalty_v2로 유지
ADHERENCE_SCORING_VERSION만 v8.2로 바꾸고 fixed weighted formula 유지
old fixed fixture의 saved adherenceScoringVersion까지 자동으로 v8.2로 치환
testLocal preview version을 production version으로 재사용
scoreDeltaPreview identity를 production version으로 사용
```

test design 결정:

```text
runtime current version assertion은 v8.2_macro_range_score_v1로 변경한다.
old fixed Records fixture에는 legacy fixed version literal을 별도 보존한다.
```

권장 test helper identity:

```text
EXPECTED_RANGE_AWARE_SCORING_VERSION = "v8.2_macro_range_score_v1"
LEGACY_FIXED_ADHERENCE_SCORING_VERSION = "v6.1_alcohol_penalty_v2"
```

주의:

```text
기존 테스트의 "이 작업은 scoring version을 바꾸지 않는다" literal guard를 무작정 v8.2로 일괄 치환하면 안 된다.
runtime version guard와 old fixed fixture basis를 분리해야 한다.
```

## 5. exact score fixture assertions

future implementation suite는 `getDailyAdherenceScore` 또는 production helper를 통해 아래 exact fixture를 검증해야 한다.

```text
eligible_full_day_neutral -> 100
fat_below_recommended_carb_exchange_explained -> 95
kcal_high_fat_observation -> 85
protein_low_anchor -> 84
fat_below_action_floor_carb_exchange_explained -> 88
severe_kcal_fat_carb -> 73
protein_anchor_carb_high -> 68
all_major_penalties -> 40
lower_clamp_defensive -> 0
```

source formula:

```text
productionScore = clamp(100 + sum(kcal/protein/fat/carbExchange contribution), 0, 100)
```

source components:

```text
kcal
protein
fat
carbExchange
```

excluded:

```text
dataQuality
fixedScoreSummary
saved componentScores
testLocalNumericPreview
scoreDeltaPreview
```

assertion rule:

```text
severity label global ordering으로 검증하지 않는다.
component + status exact contribution table을 source of truth로 둔다.
```


### 17-3. runMacroRangeProductionScoreTransitionDecisionTests

Must update or supersede:

```text

productionImplementationChanged false metadata
```

Options:

```text
A. keep suite as historical guard but mark superseded and remove core/profile registration
B. update suite to assert transition has advanced into implementation-ready state
C. leave formula/no-leak parts and move current-version assertions into new implementation suite
```

Recommended:

```text
B or C, not A.
```

Reason:

```text
Dropping the suite loses useful formula/no-leak coverage.
Keeping old v6.1/future-only assertions will block the intended implementation.
```

### 17-4. generic "scoring version unchanged" tests

Many existing tests assert:

```text

```

Future implementation must classify them:

```text
runtime current version guard -> update to v8.2_macro_range_score_v1
legacy old fixed fixture guard -> keep v6.1 literal via LEGACY_FIXED_ADHERENCE_SCORING_VERSION
unrelated feature no-version-change guard -> update expected current version after planned bump
```

Do not blanket replace every literal.
