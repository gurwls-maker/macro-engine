## 2. entry decision

결정:

```text
test-local numeric helper entry는 다음 단계에서 검토 가능하다.
하지만 첫 numeric helper는 production helper가 아니다.
첫 numeric helper는 saved record helper가 아니다.
첫 numeric helper는 UI helper가 아니다.
첫 numeric helper는 backup / Recent / DailyCoach helper가 아니다.
```

허용되는 첫 범위:

```text
test-local function only
test suite internal or exported internal-test target only
full_day_candidate fixture or sufficient detailed old fixed Record only
old fixed Records are not automatically excluded
snapshotless / one_meal / insufficient / impossible state excluded
simple / weight_only / macro-data-insufficient Records excluded
nonnumeric production helper remains unchanged
```

즉 다음 단계에서 가능해지는 것은 production이 아니라 다음 후보뿐이다.

```text
runMacroRangeScoreTestLocalNumericHelperEntryTests
```

또는 같은 범위의 test-only implementation scope.

## 4. input eligibility

첫 test-local numeric helper는 아래 조건에서만 숫자를 낼 수 있다.

```text

oldGoalSnapshot present
fixedScoreContext present

candidate version = range_score_candidate_preview_test_local_v0
weight / severity lane inventory present
Records basis = test-local candidate, not saved record
old fixed Records allowed only as non-persisted latest-policy preview input
```

숫자 금지:

```text

simple / weight_only saved Records
raw backup / raw import records without normalized detailed data

Recent aggregation path
DailyCoach signal path
UI render path
```

## 5. scoring scope

첫 numeric helper가 다룰 수 있는 것은 **candidate preview 총점 후보**까지다.

아직 열면 안 되는 것:

```text
component pointsPreview field
scoreDeltaPreview
production score replacement
Records saved score basis
Recent mixed basis aggregation
DailyCoach action
UI score display
backup / restore schema
```

이유:

```text
component pointsPreview는 사용자가 총점 산식을 추론하게 만든다.
scoreDeltaPreview는 fixed score와 candidate score를 직접 비교한다.
Records 저장은 old/new basis 의미를 바꾼다.
Recent / DailyCoach / UI 연결은 product contract가 된다.
```

따라서 첫 helper가 숫자를 계산하더라도 다음 값은 계속 null이다.

```text
candidateScorePreview: null
pointsPreview: null
scoreDeltaPreview: null
```

test-local helper의 숫자는 별도 test-local key에만 존재해야 한다.

## 6. minimum formula constraints

다음 구현 전 반드시 지켜야 할 formula constraint:

```text
penalty-only
no bonus from inside_recommended
no bonus from exchange_explained
kcal mismatch cannot be offset by fat/carb range
protein low cannot be offset by fat/carb exchange
dataQuality is eligibility gate, not score component
old fixed Records eligibility depends on detailed data sufficiency
old fixed Records saved value no-mutation
```

아직 열지 않는 실제 정책:

```text
actual numeric weight finalization
actual point deduction finalization
production score version bump
component point display
score delta comparison
```

첫 구현이 가능하다면 test-local formula는 다음 성격이어야 한다.

```text
deterministic fixture helper
guard-heavy
not connected to app state
not read by production code
not serialized
```
