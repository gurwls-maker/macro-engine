```text
v8.2_macro_range_candidate_score_preview_field_decision_2026-07-05.md
```

목적:

```text
score 후보와 storage 후보를 분리한다.
candidateScorePreview가 future production 후보로 논의되더라도, 그것이 곧 record / goalSnapshot / backup 저장 계약이 되지 않도록 결정한다.
```

핵심 결론:

```text
score gate와 storage gate는 분리한다.
macroRangeContract 저장 가능성은 range score production 적용과 별개다.
candidateScorePreview production field 후보는 기본적으로 transient 계산/표시 후보이며, 저장 계약이 아니다.
candidateScorePreview를 저장하려면 별도의 score persistence gate가 필요하다.
```

## 2. separation decision

결정:

```text
range basis storage와 range score production은 별도 gate다.
score preview 계산과 score preview 저장도 별도 gate다.
UI 표시와 storage persistence도 별도 gate다.
```

즉 다음은 서로 같은 뜻이 아니다.

```text
macroRangeContract 저장 허용
candidateScorePreview 계산 허용
candidateScorePreview UI 표시 허용
candidateScorePreview record 저장 허용
candidateScorePreview backup roundtrip 허용
getDailyAdherenceScore range-aware 변경
ADHERENCE_SCORING_VERSION 변경
```

각각 별도 decision을 닫아야 한다.

## 5. storage policy for old fixed Records

old fixed Records 정책:

```text
stored adherencePercent는 저장 당시 fixed score로 유지한다.
stored adherenceScoringVersion은 저장 당시 version으로 유지한다.
latest-policy eligibility 후보와 saved score overwrite는 다르다.
old fixed Records를 range basis score로 조용히 재해석하지 않는다.
```

허용:

```text
test-local eligibility detection
transient preview candidate generation
no-mutation comparison
```

금지:

```text
saved adherencePercent overwrite
saved adherenceScoringVersion overwrite
old fixed Records backup payload에 range score field 추가
Records detail에서 old score를 range score처럼 표시
Recent에서 old fixed score를 range basis score로 집계
```
