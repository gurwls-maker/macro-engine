## 2. Records score basis decision

### 2-1. old fixed Records

정의:

```text
macroRangeContract가 없는 record
range score basis field가 없는 record
현재 allowlist normalizeGoalSnapshot으로 읽히는 fixed target snapshot record
adherenceScoringVersion이 ADHERENCE_SCORING_VERSION 또는 legacy로 해석되는 record
```

결정:

```text
old fixed Records의 stored/raw score basis는 fixed score basis로 유지한다.
old fixed Records를 range-basis storage로 조용히 마이그레이션하지 않는다.
old fixed Records의 saved adherencePercent 의미를 바꾸지 않는다.
old fixed Records의 adherenceScoringVersion 의미를 바꾸지 않는다.
detailed macro data와 fixed goalSnapshot이 충분한 old fixed Record는 latest-policy test-local preview eligibility 후보가 될 수 있다.
simple / weight_only / snapshotless / macro-data-insufficient record는 latest-policy numeric score 대상이 아니다.
old fixed Records에 production-facing candidateScorePreview를 저장하거나 노출하지 않는다.
old fixed Records에 production-facing scoreDeltaPreview를 저장하거나 노출하지 않는다.
old fixed Records에 production-facing pointsPreview를 저장하거나 노출하지 않는다.
```

의미:

```text
snapshotless_debug_only
macroRangeContract absent
rangeScoreCandidate absent
candidate score basis absent
latest-policy preview possible only as non-persisted test-local analysis when data is sufficient
```

위 상태는 오류가 아니라 old fixed stored/raw basis다.

### 2-2. current unknown range fields

정의:

```text
raw backup / raw import / raw snapshot에 unknown macroRangeContract 또는 rangeScoreCandidate가 섞인 상태
```

결정:

```text
현재 production에서는 unknown range field를 보존하지 않는다.
normalizeGoalSnapshot allowlist를 우회하지 않는다.
candidateScorePreview / pointsPreview / scoreDeltaPreview는 prune 대상이다.
unknown range field prune은 현재 단계에서 정상 동작이다.
```

금지:

```text
unknown field pass-through
helper output 전체 저장
raw backup field를 독립 score basis로 취급
old fixed Records에 range field 사후 주입
```

### 2-3. future new range-basis Records

정의 후보:

```text
explicit normalizer / storage schema / signature / backup / Recent policy가 승인된 뒤 생성되는 future record
```

현재 결정:

```text
new range-basis Records는 아직 존재하지 않는다.
new range-basis Records를 현재 코드에서 생성하지 않는다.
new range-basis Records를 현재 backup/restore schema로 기대하지 않는다.
new range-basis Records를 Recent aggregation에 연결하지 않는다.
new range-basis Records를 DailyCoach action에 연결하지 않는다.
```

future 최소 조건:

```text
macroRangeContract known normalizer 승인
macroRangeBasisSignature include / exclude / separate decision 승인
backup / restore roundtrip 테스트 승인
Recent mixed-basis handling 승인
old fixed stored-basis no-mutation 테스트 승인
old detailed Records latest-policy preview eligibility 테스트 승인
score basis label UI 비노출 또는 표시 copy 승인
```

그 전까지:

```text
saved record raw score basis는 fixed score다.
latest-policy preview basis는 저장 마이그레이션이 아니다.
```

## 4. candidate score version naming decision

현재 production version:

```text
ADHERENCE_SCORING_VERSION = v6.1_alcohol_penalty_v2
```

결정:

```text
candidate score version은 ADHERENCE_SCORING_VERSION이 아니다.
candidate score version은 adherenceScoringVersion에 저장하지 않는다.
candidate score version은 production score version bump가 아니다.
candidate score version은 test-local/report-only object 안에서만 의미가 있다.
```

현재 report-only helper identity:

```text
MACRO_RANGE_SCORE_CANDIDATE_REPORT_ONLY_MODEL = macro_range_score_candidate_report_only_v0
rangeScoreCandidateSummary.candidateVersion = range_score_candidate_report_only_v0
```

이 이름은 현재 nonnumeric report-only helper identity로 유지한다.

future numeric preview helper가 열릴 경우 예약 이름:

```text
candidateScorePreviewVersion = range_score_candidate_preview_test_local_v0
candidateScoreBasisLabel = macro_range_candidate_preview_test_local
fixedScoreBasisLabel = fixed_adherence:v6.1_alcohol_penalty_v2
```

단, 이번 문서에서는 위 이름을 코드에 추가하지 않는다.

이름 규칙:

```text
range_score_candidate_report_only_v0 = nonnumeric report-only helper
range_score_candidate_preview_test_local_v0 = future numeric test-local preview helper
fixed_adherence:v6.1_alcohol_penalty_v2 = 현재 production fixed score basis label
```

금지:

```text
candidate preview version을 ADHERENCE_SCORING_VERSION으로 저장
candidate preview version을 adherenceScoringVersion으로 저장
candidate preview version을 production score version처럼 표시
candidate preview version 없이 scoreDeltaPreview 계산
```

## 5. scoreDeltaPreview dependency

`scoreDeltaPreview`는 이번 문서로도 열리지 않는다.

이유:

```text
fixedScoreBasisLabel과 candidateScoreBasisLabel을 직접 비교한다.
old fixed Records의 saved score와 직접 비교하면 저장 당시 점수 의미를 흔드는 것처럼 읽힐 수 있다.
UI / Recent / DailyCoach에서 production 개선/악화처럼 읽힐 수 있다.
```

따라서 `scoreDeltaPreview`가 열리려면 최소한 다음 조건이 필요하다.

```text
candidateScorePreview numeric gate 통과
candidateScorePreviewVersion 확정
fixedScoreBasisLabel 확정
candidateScoreBasisLabel 확정
old fixed Records delta no-persist / no-silent-comparison 테스트 통과
UI 비노출 또는 명시 copy 정책 확정
Recent / DailyCoach no-action guard 통과
backup no-leak guard 통과
```

현재 결론:

```text
scoreDeltaPreview는 계속 null이다.
```
