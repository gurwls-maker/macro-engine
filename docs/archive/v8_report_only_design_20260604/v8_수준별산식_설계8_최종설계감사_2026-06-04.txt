# V8 수준별 산식 설계 8단계 - 최종 설계 감사

작성일: 2026-06-04
관련 stage: `v8.0-BH_user_level_formula_final_design_audit_v0`

## 1. 이번 단계의 판단

8단계는 production 수준별 산식 구현 단계가 아니다.

이번 단계는 1-7단계가 서로 다른 말을 하지 않는지, 그리고 설계 통과를 구현 완료로 오해할 여지가 없는지 최종 감사하는 단계다. 따라서 BH의 통과 의미는 다음 하나로 제한한다.

```text
production 구현 계획 또는 owner-scoped 구현 결정으로 넘어갈 수 있다.
```

BH가 닫지 않는 것:

- production 수준별 산식 구현
- user-facing copy / Score / Coach / Records schema / backup schema 구현
- full 8-2 Cartesian 실행 또는 owner-approved alternative gate
- broad profile/routine/session human UX review
- full V8 completion

## 2. 8개 감사 단계

`userLevelFormulaFinalDesignAudit.steps`는 8개로 고정했다.

```text
truthMapCurrentState
inputOwnershipFlow
profileTrainingModel
levelContext
formulaPrinciple
explanationFlow
scenarioVerification
finalAuditBoundary
```

각 단계의 감사 의미:

- `truthMapCurrentState`: 현재 production 산식과 future precision scope가 분리되어 있는지 본다.
- `inputOwnershipFlow`: Settings, Today, Records, InBody, recent, derived, backup/import ownership이 섞이지 않았는지 본다.
- `profileTrainingModel`: 실제 profile/routine/session 정의가 훈련 context로 전환되었는지 보고, 기본세션식 우회를 막는다.
- `levelContext`: 수행 수준, 기록 품질, 체성분 신뢰도, 최근 추세, target relief가 하나의 shortcut으로 과잉 적용되지 않는지 본다.
- `formulaPrinciple`: target kcal, protein, fat, carbohydrate 순서와 external evidence / app policy 분리가 유지되는지 본다.
- `explanationFlow`: Today summary, Coach, Score, Records, snapshot, backup/import 설명 표면이 숫자 산식과 연결될 준비가 되었는지 본다.
- `scenarioVerification`: 18개 human-eye 케이스, 필수축, pairwise, targeted, full Cartesian gate가 분리되어 있는지 본다.
- `finalAuditBoundary`: 최종감사 통과를 production 구현 완료로 읽지 못하게 막는다.

## 3. 11개 consistency check

BH는 다음 11개 check를 전체 통과해야 한다.

```text
report_only_boundary_preserved
production_formula_unchanged
candidate_approval_not_inferred
profile_session_shortcut_blocked
performance_level_derived_not_selector
recent_target_relief_auto_apply_blocked
formula_order_and_evidence_boundaries_preserved
score_coach_records_backup_explained
human_axis_cartesian_separated
full_cartesian_gate_still_open
next_step_scope_limited_to_production_implementation_plan
```

사용자 지적과 직접 연결되는 항목:

- `profile_session_shortcut_blocked`: 기본운동세션 같은 기준통과용 우회를 다시 도입하지 않는다.
- `performance_level_derived_not_selector`: 수준은 사용자가 직접 고르는 shortcut selector가 아니라 여러 입력에서 파생되는 context다.
- `recent_target_relief_auto_apply_blocked`: 최근 추세나 target relief 압박이 자동으로 kcal/macro를 바꾸지 않는다.
- `human_axis_cartesian_separated`: 18개 human-eye 케이스는 80,621,568,000개 product 대체물이 아니다.
- `full_cartesian_gate_still_open`: full Cartesian은 아직 실행되지 않았고 대체 gate도 승인되지 않았다.
