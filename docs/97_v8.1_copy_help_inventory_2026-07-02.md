# v8.1 설명문 / 도움말 inventory

작성일: 2026-07-02

이 문서는 v8.1 copy/help 작업을 바로 구현하기 전에, 현재 화면에 있는 사용자-facing 문구를 화면별로 분류하기 위한 inventory다.

이번 단계에서는 `index.html`의 문구를 수정하지 않는다. 문구 변경, 도움말 이동, 접힘 이동, 삭제, 테스트 기대값 수정은 다음 별도 구현 단계에서 다룬다.

## 1. 분류 기준

| 분류 | 의미 |
| --- | --- |
| 유지 | 현재 화면 위치와 문구가 기능 이해에 직접 필요하다. |
| 축약 | 의미는 유지하되 기본 화면에서는 더 짧게 만드는 후보. |
| `?` 도움말 이동 | 항상 보이는 본문보다 hover/focus 도움말이 적합한 후보. |
| 접힘 이동 | 계산 과정, 근거, 위험 설명처럼 필요하지만 기본 화면에는 무거운 후보. |
| 삭제 | 반복, 내부 설명, 이미 UI가 말해주는 내용이라 제거 후보. |
| 테스트 보호 필요 | 변경 시 기존 회귀 테스트 또는 새 copy guard가 필요하다. |

우선순위:

1. 기본 화면에서 사용자가 매번 보는 설명을 줄인다.
2. 저장, 삭제, 복원, 기준 갱신처럼 위험한 action 문구는 기본 화면 또는 modal 안에 남긴다.
3. 산식/기준 설명은 `?` 도움말이나 접힘 영역으로 보내되 의미는 보존한다.
4. Coach와 점수 문구는 기존 copy regression test가 많으므로 구현 전 테스트 보호 범위를 먼저 확인한다.
5. 중요한 기준 설명의 source of truth는 `?` 도움말 또는 `details`로 둔다. `title`은 보조/중복 힌트로만 허용한다.

## 2. 전체 판단

현재 앱은 이미 많은 문구가 `?`, `details`, modal, summary로 이동되어 있다. v8.1에서 할 일은 모든 문구를 짧게 만드는 것이 아니라, 다음 세 부류를 분리하는 것이다.

- 기본 화면에서 반복 노출되는 안내.
- 계산/산식/기준을 설명하는 참고 문구.
- 저장/복원/삭제/기준 갱신처럼 실수 방지에 필요한 문구.

이번 inventory 기준으로 첫 구현 후보는 `Today 기본 화면/빠른 설정`, `Settings 기본 프로필/생활 패턴`, `Records 선택 날짜 설명`, `InBody 상단 설명`, `Recent 흐름 설명` 순서가 안전하다.

| `runRecentFlowCopyPolishTests` | Recent flow reference copy |
| `runSettingsMobileCopyPolishTests` | Settings mobile copy/wrap |

| `runRecordDetailMealDeleteConfirmTests` | Records 상세 수정 식사 삭제 confirm |
| `runMealEntryConsistencyTests` | Today/Records meal surface consistency |
| `runInbodyCopySourceSeparationTests` | InBody source/copy separation |
| `runChoiceUiConsistencyTests` | choice modal copy |
| `runGoalAwareCopyRegressionTests` | 목표별 Coach copy |
| `runCoachAdherenceCopySeparationTests` | Coach/action과 score/evidence copy 분리 |

다음 구현에서 새 테스트 또는 보강이 필요한 후보:

- Today KPI note를 `?` 도움말로 옮길 경우 targetCal/totalBurn 오해 방지 copy test.
- Today 빠른 설정의 “오늘 수정값은 이 날짜에 유지됩니다.”를 줄일 경우 date-scoped override copy test.
- Records 상세 basis edit 설명을 접힘으로 옮길 경우 basis edit warning visibility test.
- Settings weekday/default precedence copy를 줄일 경우 Settings/Today precedence copy test.
- InBody hero copy를 줄일 경우 source/apply separation copy test.
- Recent 7일/28일 reference copy를 줄일 경우 setting auto-change 방지 test.
- Backup warning copy는 기존 테스트를 유지하고, 문구 변경 시 replace/smart restore/conflict 의미가 빠지지 않는지 확인.
