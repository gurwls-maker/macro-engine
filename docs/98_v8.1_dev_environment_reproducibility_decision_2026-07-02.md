# v8.1 개발환경 재현성 결정

작성일: 2026-07-02

이 문서는 v8.1 인수인계 감사와 docs correction 이후, 개발환경 재현성 문제를 어떻게 닫았는지 남기는 운영 문서다.

이번 단계는 앱 기능 변경이 아니다. `index.html`, 앱 산식, UI, 저장 schema, 테스트 기대값은 변경하지 않는다. 변경 범위는 repo-local package 기준과 QA 진입점이다.

## 1. 현재 결정

v8.1 구현이 여러 커밋으로 이어질 가능성이 있고, 다른 컴퓨터에서 테스트를 재현해야 하므로 package 기준을 도입한다.

결정:

```text
package manager: npm
lockfile: package-lock.json 커밋
Playwright: devDependency로 1.61.1 고정
browser policy: 기존 runner의 PLAYWRIGHT_BROWSER_CHANNEL -> msedge -> chrome -> bundled fallback 유지
bundled Chromium: npm run setup:browsers로 설치 가능
npm scripts: smoke/core/ui/mobile/calibration/full/render audit 진입점 제공
```

분류:

```text
v8.0.1 hotfix 아님
v8.1 전 개발환경 재현성 개선
앱 동작 변경 없음
release tag 대상 아님
```

## 2. 현재 확인한 상태

| 항목 | 확인 결과 |
| --- | --- |
| 기준 브랜치 | `codex/v8.1-package-reproducibility` |
| 시작 기준 | `ae404d0 v8.1 개발환경 재현성 결정 문서화` |
| Node.js | 현재 환경에서 `v24.15.0` |
| npm | 현재 환경에서 `11.12.1` |
| `package.json` | 추가 |
| `package-lock.json` | 추가 |
| Playwright | `1.61.1` |
| repo-local `require("playwright")` | `node_modules/playwright/index.js`로 성공 |
| `node_modules` | local install용 ignored directory. 커밋하지 않음 |
| generated render audit 산출물 | `tools/render_audit/manifest.json` 등 ignored 경로에 존재 가능 |

Playwright `1.61.1`은 기존 Codex runtime cache의 Playwright 버전과 npm registry 최신 확인값이 일치해서 선택했다.

## 3. 설치 절차

새 컴퓨터에서는 먼저 dependency를 설치한다.

```powershell
npm ci
```

Edge/Chrome channel이 없거나 bundled Chromium을 명시적으로 준비하려면 다음을 실행한다.

```powershell
npm run setup:browsers
```

주의:

- `node_modules/`는 `.gitignore` 대상이며 커밋하지 않는다.
- Playwright browser binary는 사용자 cache에 설치된다.
- runner는 여전히 `PLAYWRIGHT_BROWSER_CHANNEL`, `msedge`, `chrome`, bundled Playwright 순으로 시도한다.
- 특정 channel을 강제하고 싶으면 PowerShell에서 `$env:PLAYWRIGHT_BROWSER_CHANNEL='chrome'`처럼 설정한다.

## 6. 권장 QA 명령

문서만 바꾼 경우:

```powershell
git diff --check
git status -sb
git diff --name-only
npm run test:smoke
```

문구/도움말 inventory 또는 UI copy 작업:

```powershell
npm run test:smoke
npm run test:ui
npm run test:mobile
```

산식/매크로 작업:

```powershell
npm run test:core
npm run test:calibration
npm run test:macro-policy
```

release 후보:

```powershell
npm run test:smoke
npm run test:core
npm run test:ui
npm run test:mobile
npm run test:calibration
npm run test:full
npm run audit:render:capture
npm run audit:render:analyze
```

주의:

- render audit analyze는 같은 source 시점에서 생성된 capture/manifest만 UI 증거로 인정한다.
- source 변경 후에는 `npm run audit:render:capture`부터 다시 실행한다.
- generated output은 커밋하지 않는다.
- `tools/render_audit/manifest.json`, `screenshots/`, `pages/`, `_debug/`, `v8_full_cartesian_shards/`, `v8_full_cartesian_campaigns/`는 ignored 산출물이다.

## 7. 남은 판단

이번 작업은 local QA reproducibility를 닫는다. CI 도입, GitHub Actions, full release gate 자동화는 아직 별도 작업이다.

다음 단계 후보:

1. 설명문/도움말 inventory 착수.
2. CI가 필요해지면 package 기준을 바탕으로 별도 workflow 설계.
3. release 후보에서 full exported와 render audit capture/analyze를 다시 실행.
