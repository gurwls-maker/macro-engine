## 4. WATCH별 decision

### W01. TDEE 2.0에서 finalScore 0

Audit result:
- case: E03 totalKcal/TDEE 2.0
- percent: 0
- rawScore: -16
- targetEnergyDeviationPenalty: 58
- tdeeOverloadPenalty: 58

Decision:
- classification: curve-steepness
- decision: keep
- implementation needed: no
- test needed: already partially covered, add later only if tuning opens
- UI/storage needed: no

Reason:
- TDEE 2.0은 목표와 총소비량 양쪽에서 모두 매우 큰 과다 섭취다.
- finalScore 0은 강하지만, rawScore가 -16이고 더 큰 E04에서는 -122.41로 계속 내려간다.
- 따라서 hard cap이나 bucket plateau가 아니다.
- targetEnergy와 TDEE overload가 둘 다 반영되는 것은 이중처벌이 아니라 앱 목표 축과 생리학적 overload 축의 분리다.

Next action:
- 지금 anchor를 바꾸지 않는다.
- 실제 사용자 샘플에서 1.5~2.0 구간이 지나치게 빨리 무너진다는 증거가 생기면 `targetEnergyOverAnchors`와 `tdeeOverloadAnchors`의 합산 강도를 함께 본다.

### W02. fat upper 3.0x에서 finalScore 0 근처

Audit result:
- case: F04 fat upper 3.0x
- percent: 0
- rawScore: -0.98
- targetEnergyDeviationPenalty: 38.42
- tdeeOverloadPenalty: 18.56
- fatRangePenalty: 44

Decision:
- classification: curve-steepness
- decision: tune-later
- implementation needed: no, not now
- test needed: later if tuning opens
- UI/storage needed: no

Reason:
- 이 케이스는 fatRange 단독이 아니라 targetEnergy와 TDEE overload가 함께 붙은 결과다.
- fat upper 1.2x와 2.0x는 각각 91.80, 59.76으로 자연스럽게 내려간다.
- fat 1000g은 rawScore -396.50까지 내려가므로 극단 collapse는 잘 작동한다.
- 다만 3.0x에서 이미 finalScore가 0 근처인 것은 product tone상 다시 볼 여지가 있다.

Next action:
- 지금은 WATCH로 유지한다.
- 다음에 score tuning을 연다면 `fatHighAnchors`의 2.0~3.0 구간과 target/TDEE 합산 강도를 함께 본다.
- fat 1000g collapse는 유지해야 한다.

### W03. alcohol 112g / 140g product tone

Audit result:
- alcohol 112g: percent 25.43
- alcohol 140g: percent 0
- alcohol physiology risk는 계속 증가.
- alcohol kcal 축도 반영.

Decision:
- classification: product-tone + copy-policy
- decision: keep
- implementation needed: no
- test needed: already covered for monotonicity, copy tests later if copy changes
- UI/storage needed: no

Reason:
- NIAAA anchor는 hard cutoff가 아니라 curve anchor로 쓰이고 있다.

- 사용자에게는 숫자보다 copy/action tone이 더 중요할 수 있지만, 지금 점수 산식을 바로 낮추거나 올릴 근거는 부족하다.

Next action:
- 점수값은 유지한다.
- DailyCoach/copy tone을 손볼 때 alcohol 112g/140g 메시지가 충분히 강한지 따로 본다.

### W04. UNKNOWN training context에서 high carb가 관대함

Audit result:
- case: C08 UNKNOWN + 450g carb
- percent: 98.54
- carbExerciseContextPenalty: 1.47

Decision:
- classification: context-policy
- decision: tune-later
- implementation needed: no, not now
- test needed: yes, if training context policy changes
- UI/storage needed: no

Reason:
- UNKNOWN은 REST와 NORMAL_RESISTANCE 사이의 보수적 중간값이어야 한다.
- 현재 450g carb, 75kg 기준 6g/kg는 unknown highGkg 5.5를 조금 넘는 수준이라 penalty가 작다.
- 이 자체가 버그라고 단정하긴 어렵지만, 운동 입력이 불확실한 상태에서 고탄수가 너무 쉽게 설명되는 인상을 줄 수 있다.
- 특히 사용자가 운동을 입력하지 않았는데 UNKNOWN이 사실상 training-friendly로 작동하면 current truth의 "운동은 carb context modifier" 기준이 흐려질 수 있다.

Next action:
- 지금은 anchor를 바꾸지 않는다.
- 다음 tuning 구현이 열리면 `trainingCarbContexts.unknown.highGkg`를 REST와 NORMAL 사이에서 더 보수적으로 둘지 검토한다.
- permanent regression test 후보로 남긴다.
