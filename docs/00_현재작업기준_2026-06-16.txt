탄단지 다이어리 현재 작업 기준
작성일: 2026-06-16

이 문서는 Codex가 docs 폴더를 읽을 때 가장 먼저 봐야 하는 현재 기준이다.
기존 v8 단계 문서들은 근거와 과거 판단을 확인하기 위한 참고자료이며, 이 문서와 충돌하면 이 문서와 최신 대화, 실제 앱 상태를 우선한다.

1. 최상위 원칙

- 기준문서는 정답지가 아니다.
- 실제 앱 상태, 사용자의 최신 의도, 실제 렌더 화면, 산식 결과가 기준문서보다 우선한다.
- 문서에 적혀 있다는 이유만으로 과거 단계(CH, CG, CD 등)를 계속 이어가면 안 된다.
- 문제가 보이면 문서에 맞추려고 하지 말고, 앱 완성도 기준으로 다시 판단한다.
- 감사 단계에서 사용자 결정이 필요한 선택지가 생기면 스스로 확정하지 말고, 코드 모르는 사용자도 이해할 수 있게 풀어서 질문한다.

2. 방금 정정된 방향

잘못 진행하면 안 되는 방향:

- v8.0-CH time ownership external-evidence scenario validation을 다음 작업으로 계속 진행하지 않는다.
- 업무/생활활동/운동/수면 시간소유권 TDEE 후보 검증을 최신 작업 우선순위로 보지 않는다.
- 가이드 기준/활동량 기준을 억지로 시간소유권 산식 단계에 다시 연결하지 않는다.

현재 우선 방향:

- "운동을 하는 사람인지 / 운동을 하지 않는 사람인지"를 더 상위 개념으로 둔다.
- 온보딩이 구현되기 전에는 Settings 탭에서만 이 값을 바꿀 수 있게 한다.
- 운동하는 사람은 운동 프로필, 루틴, 세션, 웨이트/유산소, 가이드 기준/활동량 기준을 모두 사용할 수 있다.
- 운동하지 않는 사람은 운동 관련 입력과 해석을 화면에서 숨긴다.
- 운동하지 않는 사람의 저장 데이터는 같은 백업/내보내기 형식을 쓰되, 읽지 않는 운동 값은 계산과 렌더에서 0 또는 fallback으로 처리한다.
- 단, 운동하지 않는 사람이 나중에 운동하는 사람으로 전환했을 때 과거 운동 데이터가 0으로 덮어써져 사라지면 안 된다.
- 반대로 운동하지 않던 기간에 갑자기 의미 있는 운동 데이터가 생긴 것처럼 보이는 것도 막아야 한다.

3. 가이드 기준 / 활동량 기준의 현재 위치

- 가이드 기준과 활동량 기준은 당장 삭제하지 않는다.
- 다만 이 둘은 최상위 사용자 모드가 아니다.
- 최상위 모드는 운동 여부다.
- 운동하는 사람 모드 안에서 가이드 기준/활동량 기준이 의미를 갖는다.
- 운동하지 않는 사람 모드에서는 운동 관련 기준과 문구를 숨기거나 일반 식단 관리에 맞게 축소한다.

4. Coach / 최근 흐름 문구 원칙

다음과 같은 정책 설명형 문구는 Coach나 최근 흐름 본문에 직접 노출하지 않는다.

- "운동 기록은 현재 관리 기준에 포함하지 않습니다."
- "단백질은 일반 권장 범위 안에서 관리합니다."
- "체성분 변화와 식단 기록을 함께 봅니다."

이런 문구는 사용자의 현재 행동을 돕는 말이 아니라 앱 정책 설명에 가깝다.
정책 설명이 필요하면 Settings의 도움말, 온보딩, 기준문서에 둔다.
Coach와 최근 흐름은 사용자가 지금 볼 상태와 다음 행동을 말해야 한다.

5. 활동/업무/TDEE 관련 미해결 판단

업무유형, 생활활동, 수면, 운동을 어떻게 TDEE에 반영할지는 아직 최종 구현 방향이 아니다.
현재 대화에서 나온 더 나은 방향은 다음과 같다.

- 중복 계산을 막으려면 시간 소유권 개념이 필요하다.
- 하지만 모든 사람에게 수면시간, 업무시간, 생활활동 시간을 매번 입력시키면 입력 피로도가 커진다.
- 생활활동은 "업무, 운동, 수면을 뺀 나머지 시간의 평균 활동량"으로 보는 방향이 자연스럽다.
- 별도 추가활동 카드는 지금 당장 기본 UI로 만들지 않는다.
- 예외적으로 이삿짐, 대청소, 긴 야외활동 같은 날을 더 정확히 처리하는 고급/예외 UI는 후속 후보로 둔다.
- 매우 높음/높음 같은 생활활동 선택지는 하루 남은 시간 전체 평균이라는 점을 사용자에게 물음표 도움말로 알려야 한다.

6. 다음 작업자가 착각하면 안 되는 점

- "다음단계 진행"이라는 지시는 오래된 v8 단계 목록을 기계적으로 이어가라는 뜻이 아니다.
- 최신 대화에서 확정된 현재 작업축을 기준으로 다음 단계를 골라야 한다.
- 현재 축은 "운동 여부 상위 모드"와 그 영향 범위 정리다.
- 기존 v8_CC 이후 TDEE 시간소유권 문서는 참고자료일 뿐, 최신 우선 작업 지시가 아니다.
- 기존 문서에 단계명이 많아도, 사용자가 최신 대화에서 방향을 바꾸면 최신 방향이 우선이다.

7. 권장 다음 단계

다음 단계는 아래 순서로 진행한다.

1. 운동 여부 상위 모드가 현재 코드 어디에 들어가야 하는지 감사한다.
2. Settings에서 운동 여부를 바꾸는 UI와 저장 위치를 확정한다.
3. Today에서 운동하지 않는 사람 모드일 때 숨겨야 할 입력, fallback 값, 계산 영향 범위를 정한다.
4. Records 저장/백업/복원에서 운동 값 보존과 fallback 처리를 정한다.
5. Coach, 최근 흐름, 점수 문구에서 운동 정책 설명문이 새지 않게 정리한다.
6. 산식은 운동하는 사람/운동하지 않는 사람 각각의 생리학적 타당성과 외부근거를 따로 확인한다.
7. 렌더 화면 기준으로 Today, Records, Recent, Settings가 자연스러운지 확인한다.

8. 문서 정리 방침

- 기존 문서를 즉시 전부 삭제하지 않는다.
- 삭제는 나중에 "현행 기준으로 흡수 완료"가 확인된 문서부터 한다.
- 지금은 이 문서를 현재 기준의 입구로 삼는다.
- 오래된 문서에서 이 문서와 충돌하는 내용이 발견되면, 오래된 문서를 고치는 것보다 이 문서의 기준과 실제 앱을 우선한다.
