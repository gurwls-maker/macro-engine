WHY APP-WIDE COPY INVENTORY IS STILL NEEDED
The user correctly pointed out that removing a few known phrases can still leave AI-like wording elsewhere.
The right next step is not another grep-only cleanup. It is an app-wide user-facing copy naturalness inventory
that reads copy line by line and classifies:

- screen / area
- exact current text
- whether it feels natural, technical-but-acceptable, awkward, AI-like, translated, over-explained, or stale
- why it feels that way: word choice, order, tone, grammar, metaphor, excessive hedging, or test fossilization
- proposed replacement
- whether an existing test should change with the copy
- whether the copy can be fixed immediately or needs a design/copy decision first

NEXT REQUIRED GATE

This next gate must be an inventory / decision pass first, not a blind rewrite. It should cover the whole app
copy surface and then split implementation batches by screen or responsibility. This prevents both extremes:
only fixing grep-visible phrases, or rewriting the app in one uncontrolled pass.
