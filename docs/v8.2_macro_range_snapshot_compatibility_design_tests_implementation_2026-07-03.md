상태:

```text
문서 성격: test implementation record
작업 범위: snapshot compatibility design tests 구현
```

이 문서는 `v8.2_macro_range_snapshot_compatibility_test_design_2026-07-03.md`의 test implementation 기록이다.

이번 단계의 목적은 macro range 저장 schema를 여는 것이 아니라, 현재 fixed target snapshot 계약과 future range field의 no-leak 방어선을 테스트로 고정하는 것이다.

## 1. 구현한 suite

추가 suite:

```text
runMacroRangeSnapshotCompatibilityDesignTests
```

등록 위치:

```text
index.html window export
tools/render_audit/run_internal_tests.cjs core profile
package.json test:macro-policy
```

이 suite는 production 계산 결과를 바꾸지 않고, 현재 `goalSnapshot`, backup/restore, Recent, score version 계약을 확인한다.

## 2. fixture 원칙

fixture는 old fixed snapshot을 baseline으로 둔다.

```text
old fixed snapshot:
- targetCal
- protein
- carbs
- fat
- goal
- exerciseManagementMode
- body/profile basis
- activity/work/sleep basis
- routine/cardio basis
```

future range field 후보는 일부러 unknown field로만 넣는다.

```text
macroRangeContract
rangeScoreCandidate
unknownFutureField
candidateScorePreview
scoreDeltaPreview
```

현재 계약에서는 이 field들이 저장 schema가 아니므로 normalize/backup/restore/Recent에 남아 있으면 실패다.

## 3. 추가 테스트

### 3-1. old fixed snapshot validity

확인:

```text
normalizeGoalSnapshot(oldSnapshot) !== null
targetCal / protein / carbs / fat 보존
macroRangeContract 없음
old fixed snapshot signature 생성 가능
```

의미:

```text
v8.0/v8.1 fixed target snapshot은 계속 source of truth다.
```

### 3-2. current unknown range field pruning

확인:

```text
unknown macroRangeContract prune
unknown rangeScoreCandidate prune
unknownFutureField prune
fixed signature unchanged
```

의미:

```text
unknown field pruning은 현재 정상 계약이다.
이 테스트는 future range field 보존을 요구하지 않는다.
```

### 3-3. explicit normalizer absence

확인:

```text
normalizeMacroRangeContractForSnapshot 없음
invalid macroRangeContract가 fixed snapshot 전체를 null로 만들지 않음
invalid macroRangeContract는 pass-through 되지 않음
```

의미:

```text
future known macroRangeContract는 explicit normalizer 없이 도입할 수 없다.
```

### 3-4. signature behavior explicit

확인:

```text
GOAL_SNAPSHOT_SIGNATURE_KEYS에 macroRangeContract 없음
GOAL_SNAPSHOT_SIGNATURE_KEYS에 rangeScoreCandidate 없음
GOAL_SNAPSHOT_SIGNATURE_KEYS에 candidateScorePreview 없음
unknown range field 추가 전후 signature 동일
```

의미:

```text
현재 signature는 fixed-only 기준이다.
range basis를 signature에 넣을지 말지는 future compatibility design에서 별도 결정해야 한다.
```

### 3-5. full backup no-leak

확인:

```text
buildFullBackupPayload 결과에 macroRangeContract 없음
buildFullBackupPayload 결과에 candidateScorePreview 없음
normalizeFullBackupPayload 후 fixed targetCal 유지
```

의미:

```text
backup payload는 아직 range field 보존 계약이 아니다.
```

### 3-6. records-only import pruning contract

확인:

```text
applyRecordsOnlyImportData 후 fixed snapshot 유지
unknown macroRangeContract prune
candidateScorePreview 미저장
```

의미:

```text
records-only import는 current fixed snapshot 계약을 유지한다.
```

### 3-7. smart restore meals-only local snapshot basis

확인:

```text
local record와 backup record가 같은 date를 공유
backup record에는 다른 fixed target과 unknown range field 포함
applySmartRestoreImportData(..., "meals")
backup meal은 merge
local goalSnapshot targetCal / protein 유지
backup macroRangeContract 미유입
```

의미:

```text
meals-only restore는 식사만 합치고 snapshot basis를 조용히 교체하지 않는다.
```

### 3-8. Recent fixed-basis aggregation explicit

확인:

```text
getRecentContextRecordSummary는 fixed targetFat을 읽음
summary에 macroRangeContract 없음
getRecentContextComponentCounts는 current fixed target 기준으로 fatLowDays 계산
range high/low basis를 invent하지 않음
```

의미:

```text
Recent는 아직 range-aware aggregator가 아니다.
fixed/range basis를 조용히 같은 의미로 섞지 않는다.
```

### 3-9. scoring version unchanged

확인:

```text
ADHERENCE_SCORING_VERSION === v6.1_alcohol_penalty_v2
stored adherence entry는 기존 scoring version 유지
range_score_candidate 미저장
candidateScorePreview 미저장
```

의미:

```text
range score production decision 전에는 score version을 바꾸지 않는다.
```
