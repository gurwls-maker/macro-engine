# v8.2 macro range score candidate report-only implementation

상태:

```text
작업 성격: report-only helper 구현
index.html 변경: 있음
production score 변경: 없음
getDailyAdherenceScore 변경: 없음
ADHERENCE_SCORING_VERSION 변경: 없음
goalSnapshot / backup / Recent / Today UI / DailyCoach 연결: 없음
Records 과거 점수 재해석: 없음
```

이 문서는 `v8.2_macro_range_score_candidate_report_only_design_2026-07-03.md`의 구현 기록이다.

## 1. 구현 목표

고정 매크로 목표 기반 기존 점수와 range candidate 기반 점수 후보를 같은 입력으로 나란히 비교할 수 있는 debug/report-only helper를 추가했다.

중요:

```text
이번 구현은 새 production score가 아니다.
이번 구현은 score version 변경이 아니다.
이번 구현은 Today/Records/Recent/backup/DailyCoach에 노출되지 않는다.
```

## 3. output 계약

`buildMacroRangeScoreCandidateReportOnly()`는 transient debug object만 반환한다.

핵심 meta:

```text
reportOnly: true
uiVisible: false
persisted: false
appliesToSavedRecord: false
productionScoreChanged: false
scoreVersionChanged: false
snapshotSchemaChanged: false
```

금지한 top-level output:

```text
score
adherenceScore
finalScore
macroRangeContract
```

`fixedScoreSummary.score`는 기존 production fixed score를 clone해서 비교 기준으로 둔 값이다. 새 점수가 아니다.

## 4. 숫자 점수 미확정

이번 구현은 range score의 총점 숫자를 만들지 않는다.

```text
candidateScorePreview: null
candidateScorePreviewStatus: not_computed_policy_unresolved
```

이유:

```text
range score weight
candidate score numeric policy
fat below/above recommended severity
carb high penalty policy
protein severity
data quality pending policy
```

위 항목이 아직 사용자/정책 판단 전이기 때문이다.

따라서 이번 helper는 점수 계산기가 아니라 component preview / comparison helper다.

## 5. component preview

생성하는 preview:

```text
kcal
protein
fat
carbExchange
dataQuality
```

각 preview는:

```text
status
pointsPreview: null
reasonIds
needsDecision
```

를 반환한다.

### kcal

`targetCal` 기준 mismatch 여부만 report-only로 본다.

```text
matched / high / low / pending_data_quality / unresolved
```

### protein

protein anchor를 만족하는지 본다.

```text
satisfied / low / pending_data_quality / unresolved
```

protein low는 range candidate에서도 중요한 신호로 유지한다.

### fat

기존 report-only helper의 fat status를 그대로 사용한다.

```text
below_action_floor
below_recommended
inside_recommended
above_recommended
above_flexible
unresolved
```

### carbExchange

fixed target 기준 carb/fat 차이가 range candidate에서 설명되는지 본다.

```text
exchange_explained
inside_candidate_range
above_candidate_range
below_training_support
pending_data_quality
unresolved
```

### dataQuality

기록 품질이 부족하면 강한 verdict를 내지 않는다.

```text
one_meal_pending
insufficient_record
snapshotless_debug_only
blocked_by_impossible_state
full_day_candidate
```

## 8. 테스트

추가 suite:

```text
runMacroRangeScoreCandidateReportOnlyTests
```

테스트 항목:

1. transient report-only metadata를 반환하고 top-level `score` / `finalScore` / `macroRangeContract`를 만들지 않는다.
2. fixed score를 clone하되 대체하지 않는다.
3. fat/carb exchange를 설명하되 fixed score는 유지한다.
4. kcal matched라도 protein low는 계속 flag한다.
5. one-meal pending에서는 강한 verdict와 numeric score를 만들지 않는다.
6. production, UI, backup, Recent, DailyCoach, Records side effect가 없다.
