## Scope

This document closes QA for `c451b51 v8.3.1 탄수 지방 공동 배분 모델 구현` and the master merge commit `177a195 v8.3.1 탄수 지방 공동 배분 모델 구현 병합`.

This closeout does not change score formula, anchors, UI layout, storage/schema, scoreDeltaPreview, old records cleanup/reset/fallback, or card help/copy. It checks whether the implementation actually fixed the carb/fat display-allocation issue instead of hiding it with text.

## Anti-inertia judgment

The prompt could have been too narrow if it only checked "tests pass" or "GPT audit says merge". The actual root issue was different:

- Today card carb/fat ranges must not imply impossible independent upper-corner combinations.
- Adaptive Today targets must not follow intake so far that overeating is excused.
- Score axes must not be over-mutated just to make the card text look consistent.
- High-volume training must not become a flat high-carb no-penalty permission.

The implementation passes this root-problem check. It uses the joint allocation model for adaptive target/card display evidence, while keeping the existing continuous score axes as the production score penalty source.

## Conditional carb/fat feasibility

Observed:

- if current carbs are 360g, conditional fat max becomes 66.6g
- 360g carbs plus 66.6g fat equals the available carb/fat kcal budget within tolerance
- if current fat is 90g, conditional carb max becomes 307.35g

Judgment:

- The model is not doing gram-for-gram exchange.
- It is calorie-equivalent exchange inside a fixed kcal budget after reserving protein.
- The user's concern that carb and fat cannot move as a simple 1:1 symmetric pair is correctly handled by the implementation.
