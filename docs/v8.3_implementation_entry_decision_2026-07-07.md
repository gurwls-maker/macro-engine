# v8.3 implementation entry decision

작성일: 2026-07-07

## 1. 문서 목적

이 문서는 v8.3 산식 구현 커밋이 아니다.
이 문서는 테스트 구현도 아니다.
이 문서는 `ADHERENCE_SCORING_VERSION` 변경도 아니다.
이 문서는 다음 implementation branch에서 무엇을 바꾸고 무엇을 바꾸지 않을지 닫는 entry decision이다.

이 문서가 필요한 이유:
- `docs/v8.3_fixture_direction_table_2026-07-07.md`는 fixture 방향을 닫았다.
- `docs/v8.3_continuous_scoring_test_design_2026-07-07.md`는 그 fixture를 어떤 테스트로 볼지 닫았다.
- 하지만 실제 repo에는 v8.2 exact score/contribution assertion이 이미 들어 있다.
- 따라서 다음 구현은 "새 suite만 추가"가 아니라 기존 suite 기대값 갱신과 production formula 교체를 같은 gate에서 처리해야 한다.

내 판단:
- 바로 구현으로 들어가지 않고 entry decision을 먼저 닫는 것은 보수적 최소구현이 아니다.
- 구현에서 빠뜨리면 안 되는 축과 일부러 열지 말아야 할 surface를 구분하는 작업이다.
- 다음 구현 브랜치에서는 테스트 기대값 갱신과 v8.3 formula implementation을 같은 gate에서 다루어야 한다.

`penaltyBreakdown` 필수 keys:
- `targetEnergyDeviationPenalty`
- `tdeeOverloadPenalty`
- `proteinShortagePenalty`
- `proteinExcessEfficiencyPenalty`
- `fatRangePenalty`
- `carbExerciseContextPenalty`
- `carbFatExchangeFailurePenalty`

### 4-6. Curve/coefficient decision boundary

결정:
- use additive continuous penalty model.
- use named policy constants in `getV83ContinuousScoringPolicy()`.
- use anchor arrays and monotonic curve helpers.
- use final clamp only.
- tests assert relations/directions, not exact public scores.

추천 curve family:
- piecewise anchor interpolation with smoothstep easing.
- monotonic helper such as `interpolatePenaltyFromAnchors(value, anchors, options)`.
- beyond-final-anchor handling must keep increasing or approach final display clamp through raw score, not create a wide plateau.

external anchor vs app policy:
- external anchor: NIAAA standard drink/binge/high-intensity, ISSN protein/carb ranges, AMDR/DRI range concepts.
- app policy constant: penalty weight, curve steepness, tolerance.
- tests must not pretend external evidence directly says "minus N points".

아직 닫지 않는 것:
- alcohol 56g exact point.
- fat upper 2.0x exact point.
- protein excess exact point.
- carb 450g exact point.

implementation branch에서 닫을 것:
- initial constants.
- anchor arrays.
- relation tests.
- no plateau tests.
- near-zero tolerance tests.

금지:
- score cap 정책 도입 금지.
- hard zero threshold primary policy 금지.

- 운동은 final score bonus가 아니라 carb context로만 표현되어야 한다.
- high-volume training day의 high carb는 설명 가능할 수 있지만, totalKcal/TDEE overload가 있으면 낮은 점수 가능.
- user-facing copy에는 curve/anchor/penalty/internal coefficient 용어를 노출하지 않는다.

이번 entry decision에서 하지 않는 것:
- copy 구현.
- UI layout 변경.
- 새로운 score breakdown UI.
