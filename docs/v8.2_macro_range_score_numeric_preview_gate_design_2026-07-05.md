# v8.2 macro range score numeric preview gate design

상태:

```text
문서 성격: score numeric preview gate docs-only design
작업 범위: candidateScorePreview 숫자화 전 gate / blocker / acceptance 조건 정리
index.html 변경: 없음
테스트 코드 변경: 없음
package.json 변경: 없음
production score 변경: 없음
getDailyAdherenceScore 변경: 없음
ADHERENCE_SCORING_VERSION 변경: 없음
candidateScorePreview 숫자화: 없음
scoreDeltaPreview 숫자화: 없음
pointsPreview 숫자화: 없음
storage/schema 구현: 없음
backup / Recent / UI / DailyCoach 연결: 없음
production macro range 구현: 없음
```

이 문서는 `v8.2_macro_range_score_severity_policy_tests_implementation_2026-07-05.md` 이후 단계다.

목적은 `candidateScorePreview`를 숫자로 만드는 것이 아니라, 숫자 preview를 열기 전에 필요한 gate를 정리하는 것이다.

중요:

```text
numeric preview gate design은 production score 구현 지시서가 아니다.
candidateScorePreview를 숫자로 만들지 않는다.
scoreDeltaPreview를 숫자로 만들지 않는다.
component pointsPreview를 숫자로 만들지 않는다.
fixed score는 계속 source of truth다.
```

## 1. 현재 기준

현재 통과한 guard:

```text
runMacroRangeScoreCandidateReportOnlyTests
runMacroRangeScoreSeverityPolicyDesignTests
runMacroRangeSnapshotCompatibilityDesignTests
runMacroRangeExplicitNormalizerSimulationDesignTests
runMacroRangeContractNormalizerDesignTests
runMacroRangeSignatureDecisionDesignTests
runMacroRangeBackupRestoreFixtureDesignTests
runMacroRangeRecentMixedBasisDesignTests
```

현재 보장:

```text
candidateScorePreview null
scoreDeltaPreview null
pointsPreview null
old fixed Records stored-value no-mutation
old fixed Records latest-policy preview eligibility depends on data sufficiency
snapshot / backup no-leak
Recent production counts unchanged
DailyCoach output unchanged
ADHERENCE_SCORING_VERSION unchanged
```

이 문서는 위 기준을 전제로 한다.

## 2. numeric preview를 아직 열면 안 되는 이유

숫자 preview는 단순 debug 숫자가 아니다.

위험:

```text
fixed score와 candidate score의 우열처럼 읽힐 수 있다.
사용자가 production score가 바뀐 것으로 오해할 수 있다.
old fixed Records의 saved score를 덮어쓰는 것처럼 오해될 수 있다.
Recent / backup / DailyCoach가 해당 숫자를 읽는 순간 product contract가 된다.
scoreDeltaPreview는 candidateScorePreview보다 더 강한 비교 의미를 만든다.
```

따라서 숫자 preview는 아래 gate가 닫히기 전까지 열지 않는다.

## 3. gate 분리

### Gate A. numeric preview scope

결정해야 할 것:

```text
숫자 preview가 test-local only인지
report-only object에 포함될 수 있는지
hidden debug output인지
사용자-facing UI에 절대 노출하지 않을지
saved record에 절대 저장하지 않을지
```

현재 기본값:

```text
candidateScorePreview: null
pointsPreview: null
```

보수적 판단:

```text
첫 숫자화는 production helper가 아니라 test-local helper 후보로만 검토한다.
```

### Gate B. component weight policy

결정해야 할 것:

```text
kcal / protein / fat / carbExchange / dataQuality 각각의 weight
fixed score component와 같은 weight를 쓸지
range score candidate 전용 weight를 둘지
kcal mismatch가 다른 macro range를 덮는지
protein low를 carb/fat exchange로 상쇄할 수 없는지
```

현재 기본값:

```text
range_score_weight_policy unresolved
candidate_score_numeric_policy unresolved
```

보수적 판단:

```text
kcal과 protein은 anchor로 유지한다.
fat/carb exchange는 설명 후보이지 보너스가 아니다.
```

### Gate C. component severity policy

결정해야 할 것:

```text
kcal high / low severity
protein low severity
fat below_action_floor severity
fat below_recommended severity
fat above_recommended severity
fat above_flexible severity
carb below_training_support severity
carb above_candidate_range severity
```

현재 기본값:

```text
각 component pointsPreview: null
```

보수적 판단:

```text
inside_recommended만으로 점수 상승을 만들지 않는다.
exchange_explained만으로 점수 상승을 만들지 않는다.
```

### Gate D. data quality policy

결정해야 할 것:

```text
full_day_candidate만 숫자화 대상으로 삼을지
one_meal_pending을 계속 null로 둘지
insufficient_record를 계속 null로 둘지
snapshotless debug fixture를 계속 null로 둘지
blocked_by_impossible_state를 계속 null로 둘지
```

현재 기본값:

```text
one_meal_pending -> null
insufficient_record -> null
snapshotless_debug_only -> null
blocked_by_impossible_state -> null
```

보수적 판단:

```text
숫자 preview 후보가 열려도 full-day test-local fixture 또는 충분한 detailed old fixed Record 외에는 null을 유지한다.
```

### Gate E. old Records policy

결정해야 할 것:

```text
old fixed Records의 saved score를 유지하면서 latest-policy preview를 계산할 기준
detailed macro data + fixed goalSnapshot sufficiency 기준
simple / weight_only / snapshotless / insufficient record unavailable 기준
new range-basis record에만 허용할지
snapshotless record는 계속 제외할지
```

현재 기본값:

```text
old fixed Records stored/raw score basis는 fixed score
old fixed Records are not automatically excluded
detailed macro data + fixed goalSnapshot sufficient -> latest-policy test-local preview candidate
simple / weight_only / snapshotless / insufficient -> unavailable or blocked
production-facing candidateScorePreview / pointsPreview / scoreDeltaPreview remain null
```

보수적 판단:

```text
numeric preview가 열려도 old fixed Records의 저장값을 바꾸지 않는다.
```

### Gate F. storage / backup / Recent / DailyCoach no-leak

결정해야 할 것:

```text
candidateScorePreview가 backup payload에 들어가지 않는지
scoreDeltaPreview가 backup payload에 들어가지 않는지
goalSnapshot에 score candidate field가 저장되지 않는지
Recent aggregation이 candidate score를 읽지 않는지
DailyCoach가 candidate score 또는 delta를 action으로 쓰지 않는지
```

현재 기본값:

```text
storage/schema readiness: 아직 아님
Recent range-aware score aggregation: 없음
DailyCoach action 변경: 없음
```

보수적 판단:

```text
numeric preview gate가 열려도 storage / backup / Recent / DailyCoach gate는 별도로 닫혀 있어야 한다.
```

## 4. candidateScorePreview gate

`candidateScorePreview` 숫자화를 열기 전 최소 조건:

```text
numeric preview scope decided
range_score_weight_policy decided
candidate_score_numeric_policy decided
kcal_candidate_policy decided
protein_severity_policy decided
fat low / recommended / high policy decided
carb high / training support policy decided
data quality pending policy decided
snapshot_candidate_scope decided
old Records stored-basis no-mutation policy decided
old detailed Records latest-policy preview eligibility policy decided
storage / backup / Recent / DailyCoach no-leak tests passing
```

gate가 닫히기 전:

```text
candidateScorePreview: null
```

gate가 닫힌 뒤에도 가능한 첫 단계:

```text
test-local helper only
not exported to production object
not persisted
not rendered
not used by DailyCoach
not used by Recent
```

## 5. pointsPreview gate

component별 `pointsPreview`는 `candidateScorePreview`보다 먼저 열릴 수도 있지만, 더 안전한 것은 아니다.

위험:

```text
component point가 있으면 사용자는 총점 산식을 추론한다.
fat/carb exchange가 보너스처럼 읽힐 수 있다.
protein low가 상쇄 가능하다고 오해될 수 있다.
```

따라서 component `pointsPreview`도 아래 조건 전에는 null이다.

```text
component severity decided
component weight decided
data quality handling decided
old Records policy decided
```

현재 결론:

```text
pointsPreview는 계속 null이다.
```

## 6. scoreDeltaPreview gate

`scoreDeltaPreview`는 마지막에 열어야 한다.

이유:

```text
fixed score와 candidate score를 직접 비교한다.
production score 개선 / 악화처럼 보일 수 있다.
old/new Records basis 비교를 흔든다.
UI copy와 DailyCoach action으로 번지기 쉽다.
```

필수 조건:

```text
candidateScorePreview numeric gate 통과
fixed score basis label 확정
candidate score basis label 확정
old Records delta no-persist / no-silent-comparison 정책 확정
UI 비노출 또는 표시 copy 정책 확정
DailyCoach no-action guard 확정
Recent no-aggregation guard 확정
backup no-leak guard 확정
```

현재 결론:

```text
scoreDeltaPreview는 계속 null이다.
```

## 8. production으로 가기 전 필요한 문서

numeric preview를 실제로 열기 전에 필요한 후속 문서:

```text
range score component weight decision
range score data quality decision
old/new Records score basis decision
candidate score version naming decision
candidate score storage non-goal decision
UI non-exposure or exposure copy decision
DailyCoach no-action or action policy decision
```

이 문서들은 구현 지시서가 아니라 gate decision 문서여야 한다.
