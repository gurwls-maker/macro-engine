## 2. 결론

v8.1 copy/help는 여기서 닫아도 된다.

이유:

- pass1~9에서 비위험 copy/help 축약과 도움말 이동을 작은 단위로 처리했다.
- pass10~10.1에서 누적 상태와 테스트 보호 후보를 정리했다.
- macro range rebalance와 onboarding start flow는 v8.1 copy/help에 섞지 않고 별도 후속 후보로 이관했다.
- pass11~14에서 위험 action / dynamic copy 영역을 audit했고, 모두 현재 copy 유지 / 구현 보류로 닫았다.
- pass15~15.1에서 종료 후보 판단과 문서 표현 보정을 마쳤다.

추가 copy 구현 blocker는 없다.

## 3. 닫힌 구현 축

v8.1 copy/help에서 실제로 정리한 표면:

- Today 기본 화면과 빠른 설정의 반복 설명.
- Today KPI 기준 설명의 `?` 도움말.
- Today 전체 입력 패널의 InBody load/save, Today 저장, routine/cardio hint.
- Settings 기본값 / 프로필 / 생활 패턴 설명과 Today 직접 수정값 우선순위.
- InBody hero copy와 원본 기록 / Today 적용 분리.
- Recent hero, 3일 / 7일 / fallback caption과 설정 자동 변경 방지 copy.
- Records 선택 날짜 작업 / empty 안내.
- Records 상세 quality state의 one meal, simple, weight-only 안내.

위 변경은 기존 의미를 보존하면서 기본 화면의 반복 설명을 줄이는 범위였다.

## 4. 현재 유지로 닫은 위험 축

다음 영역은 더 줄이지 않고 현재 유지한다.

| 영역 | 판단 | 이유 |
| --- | --- | --- |
| Records basis edit | 유지 / 구현 보류 | 목표, 생활활동, 업무·수면 변경 시 당시 목표 탄단지가 다시 계산된다는 의미가 핵심이다. |
| Records delete confirm | 유지 / 구현 보류 | 기록 전체 삭제는 즉시 원본 삭제, 식사 행 삭제는 draft 삭제 후 저장 반영이라 반영 시점이 다르다. |
| Backup/Restore warning | 유지 / 구현 보류 | full restore, records-only, smart restore conflict의 데이터 반영 범위가 다르다. |
| Coach/score dynamic copy | 유지 / 구현 보류 | action, reason, evidence, goal-aware copy, record shortage, InBody hint가 얽혀 있다. |

이 네 영역은 copy 길이가 아니라 데이터 안전, 해석 신뢰, 행동/근거 분리가 더 중요하다.
