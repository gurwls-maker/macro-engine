# v8.2 macro range score severity policy design

상태:

```text
문서 성격: range score severity / policy design docs-only
작업 범위: candidateScorePreview 숫자화 전 decision gate 정리
index.html 변경: 없음
테스트 코드 변경: 없음
production score 변경: 없음
getDailyAdherenceScore 변경: 없음
ADHERENCE_SCORING_VERSION 변경: 없음
candidateScorePreview 숫자화: 없음
scoreDeltaPreview 숫자화: 없음
goalSnapshot schema 변경: 없음
macroRangeContract 저장: 없음
macroRangeBasisSignature 저장: 없음
backup / restore schema 변경: 없음
Recent aggregation 변경: 없음
Today / Records / Recent UI 변경: 없음
DailyCoach action 변경: 없음
production macro range 구현: 금지 유지
```

이 문서는 `v8.2_macro_range_score_candidate_report_only_implementation_2026-07-03.md` 이후 단계다.

목적은 `candidateScorePreview: null`을 숫자로 바꾸는 것이 아니라, 숫자화 전에 닫아야 할 severity / weight / data quality / old-record policy decision을 분리하는 것이다.

중요:

```text
range score severity policy design은 production score 구현 지시서가 아니다.
candidateScorePreview를 숫자로 만들지 않는다.
scoreDeltaPreview를 숫자로 만들지 않는다.
getDailyAdherenceScore를 변경하지 않는다.
ADHERENCE_SCORING_VERSION을 변경하지 않는다.
fixed score는 계속 source of truth다.
```

## 1. 배경

현재 `buildMacroRangeScoreCandidateReportOnly()`는 range-aware 점수 계산기가 아니다.

현재 output:

```text
candidateScorePreview: null
candidateScorePreviewStatus: not_computed_policy_unresolved
scoreDeltaPreview: null
pointsPreview: null
```

현재 helper가 하는 일:

```text
fixed score를 clone해서 비교 기준으로 둔다.
range candidate component status를 preview로 나눈다.
reasonIds와 needsDecision을 모은다.
production score / UI / backup / Recent / DailyCoach / Records에는 연결하지 않는다.
```

현재 helper가 하지 않는 일:

```text
component point 계산
총점 preview 계산
score delta 계산
production score 대체
Records 과거 점수 재해석
DailyCoach action 변경
```

따라서 이번 문서는 "숫자를 계산하는 방법"이 아니라 "숫자를 계산하려면 어떤 결정을 먼저 해야 하는가"를 정리한다.

## 6. severity scale 후보

future range score가 열릴 경우 severity는 production DailyCoach severity와 섞으면 안 된다.

후보 scale:

```text
0 = no penalty / stable
1 = observation / light penalty candidate
2 = material penalty candidate
3 = strong penalty candidate
blocked = no numeric preview
```

주의:

```text
이 scale은 production score scale이 아니다.
DailyCoach signal severity와 동일하다고 가정하지 않는다.
ADHERENCE_SCORING_VERSION의 의미를 바꾸지 않는다.
```

severity를 숫자로 저장하지 않는다.

## 8. numeric preview gate

future에 `candidateScorePreview` 숫자화가 열리려면 최소한 아래 조건이 모두 닫혀야 한다.

```text
range_score_weight_policy decided
candidate_score_numeric_policy decided
kcal_candidate_policy decided
protein_severity_policy decided
fat low / recommended / high policy decided
carb high / training support policy decided
data quality pending policy decided
snapshot_candidate_scope decided
production score version policy decided
storage / backup / Recent no-leak policy still passing
UI / DailyCoach non-exposure or exposure policy decided
```

gate 전 상태:

```text
candidateScorePreview: null
scoreDeltaPreview: null
pointsPreview: null
```

gate 후에도 바로 production score가 되는 것은 아니다.

가능한 중간 단계:

```text
numeric candidate preview test-local only
numeric candidate preview report-only but not stored
numeric candidate preview hidden debug only
production score still fixed
```

## 9. scoreDeltaPreview gate

`scoreDeltaPreview`는 `candidateScorePreview`보다 더 위험하다.

이유:

```text
fixed score와 candidate score를 우열 관계처럼 보이게 만든다.
사용자-facing score 변경처럼 오해될 수 있다.
Records old/new basis 비교를 흔들 수 있다.
backup/restore diff와 Recent 해석에 영향을 줄 수 있다.
```

따라서 `scoreDeltaPreview` 숫자화는 별도 gate가 필요하다.

필수 조건:

```text
candidateScorePreview numeric gate 통과
fixed score basis와 candidate score basis label 확정
old Records에서 delta 계산 금지 또는 명시 정책 확정
UI 비노출 또는 표시 copy 정책 확정
DailyCoach가 delta를 action으로 쓰지 않는 guard 확정
```

현재 결론:

```text
scoreDeltaPreview는 계속 null이다.
```

## 10. old Records policy

old fixed Records는 range score로 조용히 재해석하지 않는다.

현재 정책:

```text
saved fixed record score basis는 fixed score다.
snapshotless record는 range candidate score 대상이 아니다.
macroRangeContract가 없는 record에 candidate score를 계산하지 않는다.
```

future 후보:

```text
Today/debug fixture only
new range-basis records only
explicit user-requested comparison only
never saved to record
```

이번 문서의 보수적 판단:

```text
old fixed Records에는 candidateScorePreview를 만들지 않는다.
```
