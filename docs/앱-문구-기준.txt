앱 전체 문구 기준

작성 목적

이 문서는 탄단지 다이어리 HTML 앱의 사용자-facing 문구 기준이다.
기존 v6 기준의 “짧고 바로 이해되는 코치 문구” 원칙은 유지하되, v6.2 이후 Coach가 다이어트 / 린매스업 / 벌크업 목표별로 다르게 판단하는 구조를 반영한다.

핵심은 다음이다.

- 단순 기록을 나열하거나, 아무 의미없는 기록을 보여주지 않는다. 
- 당연한 사실을 대단한 것처럼 말하지 않는다. 
- 사용자 입장에서 "그래서 뭐 어쩌라고?" 같은 반응이 나오면 안 된다.
- 문법이 옳고 어순이 맞더라도 사람이 느끼기에 어색하면 안 된다.
- 짧고 바로 이해되게 쓴다.
- 설명보다 결론을 먼저 쓴다.
- 사용자가 “그래서 지금 뭘 하면 되는데?”를 바로 알 수 있게 쓴다.
- 문구는 부드럽게 쓰되, 판단을 약하게 만들지 않는다.
- 좋지 않은 날을 괜찮은 날처럼 말하지 않는다.
- 판단은 흐리지 말고, 해야 할 말은 분명히 한다.
- 목표별 판단 차이를 문구에도 반영한다.
- 내부 로직 용어는 사용자에게 보이지 않는다.


────────────────────────
1. 말투 핵심 원칙
────────────────────────

기본 톤:
- 코치가 옆에서 짧게 말해주는 느낌
- 현실적이고 행동 중심
- 긴 설명보다 오늘 할 일을 먼저 제시

좋은 예:
- 단백질부터 챙기세요.
- 오늘은 여기서 마무리하세요.
- 기름진 음식과 간식은 끊으세요.
- 운동 전후로 탄수화물 챙기세요.
- 굶지 마세요. 단백질은 꼭 챙기세요.
- 밥이나 감자로 탄수화물을 조금 채우세요.

나쁜 예:
- 단백질 중심으로 가져가세요.
- 식단 흐름을 복구하세요.
- 좋은 방향으로 정리하세요.
- 가능하면 단백질을 챙겨보세요.
- 무조건 줄이세요.

최종 한 줄 기준:
설명하는 문장이 아니라, 지금 바로 어떻게 먹을지 말해주는 문장이어야 한다.


────────────────────────
2. 판단 정확성 원칙
────────────────────────

문구는 부드럽게 쓰되, 판단을 흐리면 안 된다.

좋지 않은 날을 괜찮은 날처럼 말하지 않는다.
술, 고지방, 단백질 부족, 기록 부족처럼 판단에 중요한 요소는 숨기지 않는다.
지금 할 행동을 말한다.

예:
❌ 오늘 식단은 나쁘지 않습니다. 가능하면 술은 줄여보세요.
⭕ 술 영향이 큽니다. 오늘 술은 여기서 멈추세요.

❌ 어제 많이 먹었으니 오늘 적게 드세요.
⭕ 어제 많이 먹었어도 오늘 먹는 양을 과하게 줄이지 마세요. 단백질은 꼭 챙기고 간식만 끊으세요.

❌ 탄단지는 맞췄으니 괜찮습니다.
⭕ 탄단지는 맞췄지만 술 영향이 큽니다. 술은 여기서 멈추세요.

문구와 점수의 역할을 구분한다.

문구:
- 사용자가 포기하지 않도록 코치답게 말한다.
- 행동으로 연결한다.

점수/판단:
- 사용자 기분을 위해 현실을 완화하지 않는다.
- 술, 회복, 식욕, 수면, 단백질 부족처럼 실제 영향이 큰 요소는 강하게 본다.


────────────────────────
3. 목표별 문구 원칙
────────────────────────

같은 탄단지 상태라도 목표에 따라 해석을 다르게 쓴다.

다이어트:
- 초과, 지방, 간식, 술을 먼저 막는다.
- 칼로리 부족 단독은 과하게 경고하지 않는다.
- 단백질 부족은 강하게 말한다.
- 탄수 부족은 휴식일에는 낮게 보고, 운동일에는 운동 전후 식사 중심으로 말한다.

예:
- 오늘은 여기서 마무리하세요.
- 단백질은 줄이지 마세요.
- 기름진 음식과 간식은 끊으세요.
- 운동 전후로 탄수화물 챙기세요.

린매스업:
- 먹은 양만 보지 말고 식사의 질을 같이 본다.
- 술, 튀김, 소스, 간식으로 칼로리를 채운 날은 좋게 말하지 않는다.
- 회복에는 단백질과 탄수화물이 필요하다.
- 부족하면 "운동한 날 먹은 양이 부족합니다"처럼 상태를 짧게 말하고, 행동은 "단백질과 탄수화물부터 챙기세요"처럼 분리한다.
- 만족했어도 술/지방이 높으면 좋은 식사로 말하지 않는다.

예:
- 먹은 양은 충분하지만 술·기름진 음식 비중이 높습니다.
- 단백질은 꼭 챙기고, 가능하면 탄수화물도 같이 드세요.
- 운동 전후로 탄수화물 챙기세요.

벌크업:
- 부족하면 채우라고 말한다.
- 하지만 술, 안주, 고지방 음식으로 채우라고 말하지 않는다.
- 칼로리 초과 단독은 과하게 경고하지 않는다.
- 술/고지방이 동반되면 벌크에 도움이 되는 식사가 아니라 살만 찌는 식사라고 말한다.

예:
- 단백질부터 챙기고, 밥이나 감자처럼 소화 쉬운 탄수화물도 채우세요.
- 밥, 감자, 오트밀처럼 소화 쉬운 탄수화물을 추가하세요.
- 이유: 술·안주 비중이 높습니다.
- 행동: 술·안주로 칼로리를 채우지 마세요.
- 보조 행동: 부족한 칼로리는 식사로 채우세요.


────────────────────────
4. Coach 문구 구조
────────────────────────

Coach 문구는 state / reason / action 역할을 구분한다.

state:
- 오늘 상태를 짧게 해석한다.
- 사용자가 현재 상황을 바로 알 수 있어야 한다.

reason:
- 왜 그런지 필요할 때만 붙인다.
- 너무 길게 설명하지 않는다.

action:
- 지금 할 행동을 말한다.
- “그래서 뭘 먹거나 멈춰야 하는지”가 보여야 한다.

좋은 구조:
상태: 먹은 양도 적고 단백질도 부족합니다.
이유: 다이어트 중에도 단백질은 줄이면 안 됩니다.
행동: 칼로리를 더 깎지 말고 단백질부터 챙기세요.

나쁜 구조:
오늘 기록을 보면 전체적으로 섭취량이 낮고 단백질도 충분하지 않은 편이기 때문에, 앞으로는 단백질 위주로 조금 보완해보는 것도 좋습니다.


────────────────────────
5. 카드별 문구 역할
────────────────────────

카드는 같은 말을 반복하지 않고 각자 역할을 가져야 한다.

총칼로리 카드:
- 방향 / 정리 / 마무리 역할
- 다이어트: 초과, 마무리, 간식 중단
- 린매스업: 필요한 식사인지, 술·지방으로 채운 초과인지 구분
- 벌크업: 부족분 보강 또는 과한 고지방/술 경고

예:
- 오늘은 여기서 마무리하세요.
- 이미 초과했습니다. 간식과 야식은 끊으세요.
- 벌크업 중이면 탄수화물과 단백질을 식사로 채우세요.

단백질 카드:
- 우선 행동 역할
- 모든 목표에서 단백질 부족은 먼저 챙긴다.
- 술/회복/전날 과식 문구가 단백질 부족을 덮으면 안 된다.
- 단백질 초과 단독은 강한 경고로 올리지 않는다.

예:
- 단백질부터 챙기세요.
- 다음 식사에 단백질부터 챙기세요.
- 술이 있어도 단백질은 줄이지 마세요.

탄수화물 카드:
- 타이밍 / 배치 역할
- 다이어트 휴식일: 과한 보완 강요 금지
- 다이어트 운동일: 운동 전후 위주로
- 린매스업: 운동 전후 위주 + 깔끔한 식사
- 벌크업: 소화 쉬운 탄수로 총량 보강

예:
- 운동 전후로 탄수화물 챙기세요.
- 밥이나 감자로 탄수화물을 조금 채우세요.

지방 카드:
- 제한 대상 역할
- 다이어트: 술, 튀김, 소스, 간식, 기름진 음식 제한
- 린매스업: 술, 튀김, 소스, 간식 제한
- 벌크업: 술, 안주, 고지방으로 채우지 않기
- 지방 부족 단독은 하루 단위 메인 경고로 올리지 않는다.

예:
- 기름진 음식과 간식은 끊으세요.
- 술·안주로 칼로리를 채우지 마세요.


────────────────────────
6. 술 / 회복 / 전날 과식 문구 기준
────────────────────────

술:
- alcoholKcal은 현재 프로젝트 기준에서 순수 에탄올 kcal로 해석한다.
- light는 과장하지 않는다.
- moderate 이상은 회복에 영향을 준다고 말한다.
- high 이상은 오늘 술을 멈추라고 말한다.
- high / very_high / extreme 이상은 탄단지를 맞춰도 좋은 식단일처럼 말하지 않는다.

예:
- 술 영향이 조금 있습니다. 오늘은 술을 더 마시지 마세요.
- 술 영향이 큽니다. 물과 단백질부터 챙기세요.
- 탄단지는 맞췄지만 술 영향이 큽니다. 오늘 식단은 좋지 않습니다.

전날 술:
- “회복데이”라는 내부 라벨은 사용자에게 보이지 않는다.
- 대신 "알코올 영향"을 표시한다.
- “어제 마신 술”처럼 원인을 짧게 말한다.
- 행동은 수분, 단백질, 수면을 우선한다.
- 오늘 술이 또 있으면 전날 회복보다 오늘 술이 우선이다.
- 오늘 단백질이 심하게 부족하면 전날 회복보다 단백질 보완이 우선이다.

예:
- 어제 술 기록이 있습니다. 수분과 단백질을 먼저 챙기세요.
- 연속으로 술 기록이 있습니다. 오늘 술은 여기서 멈추세요.

전날 비음주성 과식:
- 칼로리가 초과한 정도에 따라 "과식", “폭식”, “치팅” 같은 단어를 차등 사용한다.
- 오늘도 초과했다면 여기서 마무리하라고 말한다.

예:
- 어제 목표보다 조금 더 먹었습니다. 오늘은 간식과 야식만 끊으세요.
- 오늘도 이미 목표를 넘었습니다. 여기서 마무리하세요.


────────────────────────
7. 기록 품질 / 기타칼로리 문구 기준
────────────────────────

기록이 부족하면 식단을 단정하지 않는다.
먼저 식사 기록이 더 필요하다고 안내한다.

예:
- 식사 기록이 더 필요해요.
- 오늘 먹은 주요 식사부터 기록해 주세요.

식사 0~1끼:
- 칼로리/단백질/탄수/지방 부족을 강하게 단정하지 않는다.
- 먹은 주요 식사부터 남기도록 말한다.

식사 2끼:
- 강한 차단은 하지 않는다.
- 필요하면 “오늘 입력한 식사 기준”이라고 짧게 붙인다.

기타칼로리:
- 식단 실패가 아니라 기록 품질 문제로 말한다.
- 다음 기록에서 무엇을 해야 하는지 말한다.

예:
- 다음 기록부터 탄수화물·단백질·지방으로 나눠 입력하세요.
- 그래야 오늘 조언이 더 정확해집니다.

8. 표현 규칙
────────────────────────

1) GPT/번역체 제거

제거 대상:
- 중심으로 가져가세요
- 방향으로 정리하세요
- 흐름을 복구하세요
- 리듬을 관리하세요
- 밸런스를 맞춰가세요

대체:
- 단백질부터 챙기세요.
- 오늘은 여기서 마무리하세요.
- 운동 전후로 탄수화물 챙기세요.

2) 회피형 제거

메인 행동에서 제거 대상:
- 가능하면
- 해도 좋습니다
- 해보는 것도 좋습니다
- 권장됩니다

단, "가능하면"은 보조 행동을 말할 때만 쓸 수 있다.
메인 행동을 약하게 만들면 금지다.

대체:
- 하세요
- 멈추세요
- 챙기세요
- 끊으세요
- 남겨주세요

3) 보상 개념 금지

❌ 어제 많이 먹었으니 오늘 줄이세요.
⭕ 단백질은 꼭 챙기고 간식만 끊으세요.

❌ 점수를 만회하세요.
⭕ 오늘은 술과 간식을 더 먹지 마세요.

문서체 단어 주의:
- 섭취 / 보완 / 관리 / 기준 / 흐름 / 판단은 문서 내부 기준어로는 쓸 수 있다.
- 사용자 화면에서는 너무 자주 쓰지 않는다.
- 먹을 것, 멈출 것, 줄일 것, 기록할 것을 일상어로 먼저 말한다.

예:
❌ 흑자 품질이 낮습니다.
⭕ 먹은 양은 충분하지만 술·기름진 음식 비중이 높습니다.

❌ 벌크 칼로리로 보기 어렵습니다.
⭕ 벌크업에 도움이 되는 식사로 보기 어렵습니다.

전문 모드 / 상세 설명 / 테스트 로그 / 개발 문서에서는 내부 용어를 사용할 수 있다.
기본 화면에서는 쉬운 말로 먼저 보여준다.

목표 칼로리와 탄단지로 배분한 칼로리가 다를 때:
- A5.1 설명층은 유지한다.
- 기본 화면에서는 내부 산식 용어 대신 목표 칼로리, 탄단지로 배분한 칼로리, 목표와 차이, 현실적인 범위처럼 말한다.
- "탄단지는 현실적인 범위까지만 배분했습니다"는 계산 근거 상세나 도움말처럼 설명이 필요한 위치에서만 쓴다.
- Coach 메인 행동에서는 이 설명을 길게 풀지 말고, 오늘 먹을 탄수화물 / 단백질 / 지방 g과 바로 할 행동을 우선한다.
- 탄단지 g을 우선 기준으로 보라는 안내는 숫자 차이가 사용자를 헷갈리게 할 때만 보조 문구로 쓴다.
- macro gap, cap, hard cap, soft range, floor, policy, targetBurnBase, reflection, helper, confidence는 기본 사용자 화면에 노출하지 않는다.

운동 여부 상위 모드와 기존 dual basis 문구 기준:
- 최신 기준에서는 가이드 기준 / 활동량 기준 mini toggle을 사용자-facing Today 선택지로 유지하지 않는다.
- 사용자가 직접 고르는 최상위 기준은 "운동을 관리 기준에 포함할지"다.
- 기존 가이드 기준의 운동생리학 / 운동인 기준 성격은 운동함 모드로 흡수한다.
- 기존 활동량 기준의 일반 식단 / 체중 관리 성격은 운동 안 함 모드로 흡수한다.
- 기본 화면에서는 "가이드 기준", "활동량 기준"을 기준 선택 버튼이나 상단 칩처럼 노출하지 않는다.
- 내부 계산이나 개발 문서에서 dual basis라는 말을 쓸 수는 있지만, 사용자-facing 문구에서는 운동 여부와 현재 행동 중심으로 말한다.
- 운동함 모드에서는 운동 프로필, 루틴, 세션, 웨이트, 유산소, 체성분 해석이 자연스럽게 드러날 수 있다.
- 운동 안 함 모드에서는 운동 관련 입력과 해석을 숨기고, 식사 / 체중 / 체성분 기록 중심으로 말한다.
- 단, Coach / 최근 흐름 본문에 "운동 기록은 현재 관리 기준에 포함하지 않습니다" 같은 정책 설명을 쓰지 않는다.
- 계산 정확성이 필요한 도움말은 Settings 도움말, ? 툴팁, 온보딩, 기준문서 쪽으로 분리한다.

C4-0 Today 중심 입력 문구 기준:
- Today는 오늘 계산값과 오늘 기록의 중심이다.
- Settings는 나의 기준 / 기본값 / 장기 기준 / 데이터 관리로 말한다.
- 기존 "빠른 수정은 shortcut" 문구는 과도기 기준이다.
- 최종 문구는 "Today에서 오늘값을 확인하고 바꾼다"는 구조를 우선한다.
- 기본 화면에서는 "source of truth", "draft", "schema", "selectedBasis", "helper", "policy"를 쓰지 않는다.

C4.8 Today 중심 작업공간 문구 기준:
- Today는 단순 계산 탭이 아니라 오늘 작업공간이다.
- Today는 오늘 기록을 시작하고 갱신하는 흐름으로 말한다.
- 저장된 결과는 Records의 날짜별 기록으로 남는다고 말한다.
- Records는 확정 기록을 확인하고 수정하는 보관함으로 말한다.
- "입력탭 / 기록탭"처럼 과거 탭 구조를 중심으로 한 표현은 피한다.
- Today에서 오늘값을 바꾸고, Records에서 저장된 그날 기록과 그날 계산 기준을 확인한다는 관계를 유지한다.

생활활동 / 업무 유형 문구:
- Settings에서는 "기본 생활활동", "기본 업무 유형"을 쓴다.
- Today에서는 "생활활동", "업무 유형"을 쓴다.
- 시간 입력은 "생활활동 시간", "업무 시간"처럼 직접 말한다.
- 긴 helper를 반복하지 않는다.
- 생활활동은 사용자가 입력한 생활활동 시간 동안의 평균 강도다.
- 입력하지 않은 나머지 시간은 "자동 휴식시간"으로 말한다.
- 생활활동 설명은 카드 본문에 노출하지 않고 ? 도움말로 둔다.
- ? 도움말에는 수면·업무·운동과 겹치지 않게 입력한다는 점, 일부 시간만 힘든 활동이면 그 시간만 생활활동 시간에 넣는다는 점, 웨이트·유산소로 입력한 운동을 생활활동에서 다시 높게 잡으면 중복 계산된다는 점을 넣는다.
- 업무 유형 설명도 Today와 Settings 모두 선택칸 아래 고정 문구가 아니라 ? 도움말로 둔다.
- 업무 유형은 회사 일 여부보다 "일정 시간 동안 잡아둔 작업/근무의 평균 강도"로 설명한다. 다만 화면 기본 명칭은 현재 앱 흐름에 맞춰 "업무 유형"을 유지한다.
- 수면 시간, 업무 시간, 생활활동 시간 설명도 기본 화면을 길게 만들지 말고 ? 도움말로 둔다.
- Today 계산 요약에는 "시간 배분", "자동 휴식"처럼 짧은 라벨로 실제 적용된 시간을 보여줄 수 있다.
- "시간 배분"은 결과 설명이지 새 선택지가 아니다. 가이드 기준 / 활동량 기준처럼 독립 토글로 보이면 안 된다.
- 숨겨진 ? 도움말은 가로폭을 밀지 않아야 한다. 숨김 상태에서는 화면 공간을 차지하지 않게 처리한다.
- 설명이 길어지면 본문을 늘리지 말고 ? 도움말을 보강한다.
- "오늘 결과에 바로 반영되는 핵심값만 바꿉니다", "자세한 설정에서 관리합니다" 같은 반복 설명은 삭제 / 축약 / ? 이동 후보로 둔다.

유산소 문구:
- Today에서는 "오늘 유산소"를 쓴다.
- Settings에서는 "기본 유산소" 또는 "기본 유산소 프리셋"을 후보로 둔다.
- 프리셋이 있어도 Today 값이 오늘 계산 기준임을 필요할 때만 설명한다.
- "기본값 불러오기"는 버튼 또는 작은 action 문구로 쓸 수 있다.

InBody → Today 문구:
- 최신 InBody를 Today 몸 상태 값으로 채운 경우 기본 화면에서는 "출처: 최신 InBody"처럼 짧게 쓴다.
- 자세한 설명은 ? / 접힘으로 보낸다.
- InBody 측정 체중을 오늘 공복 체중처럼 말하지 않는다.
- 새 InBody 저장 후 Today 반영 선택 후보는 "오늘 계산에도 반영할까요?"처럼 말한다.

적용 패턴 문구 기준:
- 프리셋 적용형은 "프리셋 적용", "기본값 불러오기"처럼 말한다.
- 프리셋은 계속 따라가는 값이 아니라 Today 값에 한 번 적용하는 값으로 말한다.
- 기록 적용형은 "최신 InBody 적용", "InBody 기록 선택"처럼 말한다.
- 기록 적용형은 저장된 원본 기록을 Today 값에 복사하는 흐름이며, 원본 기록이 바뀐 것처럼 말하지 않는다.
- 기준 유지형은 "자동 기준 사용", "직접 입력 유지"처럼 말한다.
- 기준 유지 상태에서는 기준값 변경이 반영될 수 있고, 직접 입력 상태에서는 현재 입력값을 유지한다는 차이를 필요할 때만 설명한다.
- 일회성 반영형은 "오늘 계산에도 반영"처럼 말한다.
- 일회성 반영형은 계속 따라가는 구조가 아니라 한 번 복사하는 흐름으로 설명한다.
- 기본값 시작형은 "기본값으로 시작", "나의 기준에서 불러오기"처럼 말한다.
- 새 날짜 / 첫 생성 시 기본값으로 시작하고, 해당 날짜에서 직접 수정하면 그 날짜 값은 유지한다고 설명한다.

기록 시작 메모 문구:
- 상세 기록 식사 입력용 메모는 "식사 메모"라고 말한다.
- 기록 전체에 남기는 메모는 "오늘 메모"라고 말한다.
- 오늘 공복 체중은 Records 기록값이다.
- 체크 ON일 때만 계산용 체중에도 반영한다고 말한다.
- 식단 건너뛰기는 식단 점수 / 탄단지 판단을 만들지 않는 흐름으로 말한다.

설명문 최소화 기준:
- 기본 화면은 라벨 + 값 + 버튼 중심이다.
- 설명은 기본 숨김을 원칙으로 한다.

10. 길이 기준
────────────────────────

기본:
- state: 짧게
- reason: 필요할 때만 1문장
- action: 짧고 행동 중심

권장:

- 최대 2문장
- 너무 길면 UX가 깨진다.

예외:
- 문장이 너무 짧아져서 어색하면 자연스러운 한국어를 우선한다.
- 계산 기준, 기록 저장 조건, 기록 방식 기준, InBody 주의사항은 정확성이 우선이다.
- 도움말/툴팁은 설명 말투가 자연스러울 수 있다.

10-1. UI 위치별 문구 기준

같은 의미라도 UI 위치에 따라 길이와 말투가 달라야 한다.
하나의 문장을 chip, button, card, Coach, helper, tooltip, empty state, confirm에 공통으로 돌려 쓰지 않는다.

chip / badge:
- 역할: 상태를 아주 짧게 표시한다.
- 문장형 금지, 마침표 금지.
- 2~8자 권장.
- 예: 단백질 부족 / 탄수 부족 / 고지방 / 술 영향 / 기록 부족 / 기록 보완 / InBody 있음 / 식단 판단 없음
- "기록 보완"은 chip / badge에서는 가능하지만 문장형 helper나 Coach 문구로는 딱딱하다.

button:
- 역할: 누르면 무엇을 하는지 바로 보여준다.
- 문장형보다 짧은 동작어를 우선한다.
- 예: 기록하기 / 수정하기 / 삭제 / 열기 / 접기 / 저장 / 불러오기 / 문제 날짜 보기
- 예: 빠른 수정 / 전체 설정 / 데이터 관리 / 기록 내보내기 / 불러오기
- "관리 열기"는 기계적으로 느껴질 수 있다. UI 위치에 따라 열기 / 펼치기 / 수동 입력 열기 중 고른다.

icon / action label:
- 신규 주요 action과 navigation은 SVG icon + 풀네임 라벨을 우선한다.
- 기존 emoji / 장식 요소는 후속 UI 감사에서 정리한다.
- 중요한 action은 icon 단독으로 전달하지 않는다.
- badge / chip / status / action의 역할을 구분한다.
- 장식성 badge 남발을 피한다.

card title:
- 역할: 카드가 무엇인지 말한다.
- 판단 문구보다 영역 이름에 가깝게 쓴다.
- 예: 기록 보관함 / 오늘 식단 점수 / 오늘 먹은 양 / 그날 목표 / 그날 계산 기준
- 예: 오늘 계산 기준 / 현재 계산 기준 / 데이터 관리

short helper / subtitle:
- 역할: 카드 아래에 붙는 짧은 설명이다.
- 너무 긴 문장을 피한다.
- 예: 날짜별 식사와 체중을 확인합니다.
- 예: 필요할 때만 펼쳐서 입력하세요.
- 예: 식사 기록이 더 필요해요.
- 예: 기록이 부족하면 식단 원인은 말하지 않습니다.
- 예: 오늘 결과에 바로 반영되는 핵심값만 바꿉니다.
- 예: 자세한 계산 기준은 전체 설정에서 관리합니다.

Coach state:
- 역할: 지금 상태를 짧게 말한다.
- 예: 단백질이 부족해요.
- 예: 술 영향이 큽니다.
- 예: 오늘은 이미 많이 먹었습니다.
- 예: 아직은 식단 원인을 말하기 어렵습니다.

Coach reason:
- 역할: 왜 그런지 필요할 때만 짧게 말한다.
- 예: 최근 상세 기록에서 단백질 부족이 반복됩니다.
- 예: 탄단지 기록이 부족해 식단 원인은 아직 말하기 어렵습니다.
- 예: 오늘은 술 칼로리 비중이 큽니다.

Coach action:
- 역할: 사용자가 지금 무엇을 해야 하는지 말한다.
- 가장 중요한 문구다.
- 예: 다음 식사에 단백질부터 챙기세요.
- 예: 오늘 술은 여기서 멈추세요.
- 예: 기름진 음식과 간식은 끊으세요.
- 예: 운동 전후로 탄수화물 챙기세요.
- 예: 오늘 먹은 주요 식사부터 남겨주세요.

tooltip / 자세히 보기:
- 역할: 짧은 화면 문구보다 길어도 되며 정확성을 우선한다.
- 예: 최근 상세 기록이 적어서, 어떤 영양소가 자주 부족한지는 아직 구분하기 어렵습니다.
- 예: 식단 판단 없음 기록은 체중 변화는 볼 수 있지만, 식단 원인은 말하지 않습니다.

empty state:
- 역할: 비어 있는 이유와 다음 행동을 말한다.
- 예: 아직 기록이 없습니다. 오늘 식사부터 남겨주세요.
- 예: InBody 기록이 없습니다. 측정값을 저장하면 여기서 볼 수 있습니다.

confirm / warning:
- 역할: 데이터 손실이나 중요한 제한을 분명히 말한다.
- 예: 기록 방식을 바꾸면 식사 내용이 사라질 수 있습니다. 계속할까요?
- 예: 식단 판단 없음으로 바꾸면 식사와 식단 점수는 저장하지 않습니다.

10-2. 영양소별 문구 분리 기준

부족한 항목이 다르면 행동 문구도 달라야 한다.
"부족한 양은 밥과 단백질로 채우세요"처럼 모든 상황에 쓰는 공통 문구는 쓰지 않는다.

단백질 부족:
- 단백질부터 챙기세요.
- 다음 식사에 단백질부터 챙기세요.
- 단백질은 줄이지 마세요.
- 단백질은 꼭 챙기세요.

탄수 부족:
- 운동 전후로 탄수화물 챙기세요.
- 운동한 날엔 탄수화물을 너무 빼지 마세요.
- 벌크업 중이면 밥이나 감자 같은 탄수화물을 더 챙기세요.
- 린매스업 중이면 단백질은 꼭 챙기고, 탄수화물도 식사로 같이 드세요.
- 다이어트 휴식일에는 탄수화물 부족을 강하게 채우라고 말하지 않는다.

지방 부족:
- 하루만 보고 강하게 말하지 않는다.
- 반복될 때만 말한다.
- 며칠째 지방이 낮습니다. 견과류나 달걀처럼 부담이 적은 지방을 조금 추가하세요.

고지방:
- 기름진 음식과 간식은 끊으세요.
- 오늘은 튀김과 소스류를 더 먹지 마세요.

술:
- 오늘 술은 여기서 멈추세요.
- 술 영향이 큽니다. 물과 단백질부터 챙기세요.

전체 섭취 부족:
- 단백질은 꼭 챙기고, 가능하면 탄수화물도 같이 드세요.
- 벌크업 중이면 탄수화물과 단백질을 식사로 채우세요.
- 린매스업 중이면 단백질과 탄수화물을 식사로 채우세요.

"가능하면" 기준:
- 메인 행동을 약하게 만들 때는 쓰지 않는다.
- 보조 행동을 말할 때는 쓸 수 있다.
- 나쁨: 가능하면 단백질을 챙겨보세요.
- 가능: 단백질은 꼭 챙기고, 가능하면 탄수화물도 같이 드세요.

10-3. 동사 사용 기준

챙기세요:
- 단백질, 탄수화물, 물처럼 먹거나 준비할 것을 말할 때 쓴다.
- 자연스러운 코치 문구에 적합하다.

드세요:
- 실제 먹는 행동을 말할 때 쓴다.
- "같이 드세요"처럼 보조 행동에 자연스럽다.

넣으세요:
- 식단 구성에 무언가를 추가한다는 뜻일 때만 쓴다.
- 먹는 행동에는 "챙기세요" 또는 "드세요"를 우선한다.
- "식단에 단백질을 넣으세요"처럼 식단 구성 자체를 말할 때는 가능하다.
- "다음 식사에 단백질부터 넣으세요"는 "다음 식사에 단백질부터 챙기세요"로 쓴다.
- "단백질 챙기고 탄수화물도 넣으세요"는 "단백질은 꼭 챙기고, 탄수화물도 같이 드세요"로 쓴다.

채우세요:
- 부족한 양을 구체적 음식으로 채울 때 쓴다.
- "밥이나 감자로 탄수화물을 채우세요"는 가능하다.
- "부족한 영양을 채우세요"처럼 뭉뚱그리면 약하다.

멈추세요 / 끊으세요:
- 술, 간식, 야식, 기름진 음식처럼 멈출 행동에 쓴다.

남겨주세요 / 기록해 주세요:
- 기록 행동에 쓴다.
- 딱딱한 안내보다 자연스럽다.

반복이 허용되는 경우:

- 테스트에서 필요한 의미 키워드

12. 예외 기준
────────────────────────

1. 정확한 설명이 필요한 경우
- 짧은 문장보다 정확성이 우선이다.
- 계산 기준, 기록 저장 조건, 기록 방식 기준, InBody 주의사항이 여기에 해당한다.

2. 오해 가능성이 큰 경우
- 행동 문구만 쓰지 말고 이유를 짧게 붙인다.

예:
- 굶어서 만회하지 마세요. 피로와 재과식으로 이어질 수 있습니다.

3. 위험하거나 중요한 제한이 있는 경우
- 부드럽게 돌려 말하지 않는다.

예:
- 오늘 술은 더 마시지 마세요.
- 단백질은 줄이지 마세요.

4. 전문 모드 / 상세 설명 영역
- 전문용어를 완전히 금지하지 않는다.
- 기본 화면에는 쉬운 말로 먼저 보여준다.
- 자세한 설명에서만 전문용어를 쓴다.

5. 안내 / 도움말 / 툴팁
- 코치 말투보다 설명 말투가 자연스러울 수 있다.
- 도움말은 설명 정확성이 우선이지만, 기본 화면 문구를 대신 길게 만들면 안 된다.
- 계산 근거 / 탄단지 범위 / dual basis 설명은 후속 도움말 체계에서 위치, 접힘 구조, 모바일 표시, 테스트 기준까지 함께 설계한다.

예:
- 이 값은 오늘 소비 칼로리에 운동량을 더해 계산합니다.

6. 버튼 / 라벨
- 짧고 명확한 이름이 우선이다.
- 문장형 코치 톤을 억지로 넣지 않는다.

예:
- 저장
- 불러오기
- 최근 InBody 값 불러오기

7. 기능명 반복
- 반복 방지보다 일관성이 중요한 기능명은 반복해도 된다.

13. 문구 테스트 기준
────────────────────────

문구 테스트는 나쁜 사용자-facing 표현을 고정하는 장치가 되면 안 된다.

아래 같은 표현을 테스트 기대값으로 고정하지 않는다.

- 흑자 품질
- 벌크 칼로리
- signal
- severity
- penalty
- score cap
- decision
- decisionType
- 회복데이

문구 테스트는 정확한 문장 전체보다 핵심 의미와 행동을 확인하는 방식을 우선한다.

예:
- “술·기름진 음식 비중이 높다”는 의미
- “단백질부터 챙긴다”는 행동
- “오늘은 여기서 마무리한다”는 행동
- “운동 전후로 탄수화물을 챙긴다”는 행동

단, 아래는 정확한 표현을 확인할 수 있다.

- 위험 제한 문구
- 핵심 행동 문구
- 내부 용어 노출 금지
- 기록 품질 안내
- alcoholKcal처럼 오해 가능성이 큰 입력 의미 안내

테스트가 현재 소스의 낡은 문구를 보존하는 장치가 되면 안 된다.
문구를 바꿨는데 테스트가 깨진 경우, 먼저 테스트가 앱 문구 기준과 맞는지 확인한다.

────────────────────────
14. InBody / 체중 기록 / Today 입력값 근거 분리
────────────────────────

InBody 분석 결과는 최종적으로 Today Coach에 적극 연결한다.
다만 InBody 단독으로 성급한 행동 조언을 만들지 않도록, 기록 수 / 측정 간격 / 측정 조건 / suspicious 여부 / 최근 식단 기록 / 체중 기록과의 정합성에 따라 행동 권한을 나눈다.

사용자-facing 문구에서는 아래 세 근거를 반드시 분리한다.

1. Records의 아침 공복 체중 기록
2. InBody의 체중 / 골격근량 / 체지방량 / 체지방률 기록
3. Today 계산에 쓰인 오늘 입력값

표현 기준:
- InBody 기록을 근거로 한 문구는 “인바디 분석 결과” 또는 “인바디 기록”이라고 밝힌다.
- Records 체중 기록을 근거로 한 문구는 “체중 기록” 이라고 밝힌다.
- Today 계산 입력을 근거로 한 문구는 “오늘 입력값” 이라고 밝힌다.
- 인바디 체중값을 Records 체중 기록처럼 말하지 않는다.
- Records 체중 추세를 InBody 분석처럼 말하지 않는다.
- 체중 기록과 인바디 기록이 모두 있을 때는 두 근거를 분리해서 말한다.

v6.6-C1 이후 체중 라벨 기준:
- 계산용 체중: 오늘 목표 칼로리와 탄단지 계산에 쓰는 값이다. Settings와 Today 빠른 수정 후보에서 사용한다.
- 오늘 공복 체중: Records에 남기는 기록값이다. 최근 체중 흐름과 목표 진행 판단에 쓰는 값이다.
- InBody 측정 체중: InBody 측정 결과에 포함된 값이다. InBody 탭에서 중립 관찰로 다룬다.
- 입력폼, 카드 제목, helper에서 세 값을 모두 “체중”으로만 쓰지 않는다.
- 화면 맥락만으로 의미가 분명하지 않으면 “계산용 체중”, “오늘 공복 체중”, “InBody 측정 체중”처럼 역할을 붙인다.
- “오늘 입력값”은 넓은 표현이므로 체중 단독 입력에는 “계산용 체중”을 우선한다.

좋은 예:
- 다이어트: 최근 14일 체중은 목표대로 줄고 있습니다.
- 린매스업: 최근 14일 체중은 목표대로 천천히 늘고 있습니다.
- 벌크업: 최근 14일 체중은 목표대로 늘고 있습니다.
- 인바디 분석 결과, 체지방은 줄고 골격근량은 유지되고 있습니다.
- 체중 기록은 줄고 있습니다. 다만 인바디 기록이 아직 1회라 변화를 보기 어렵습니다.
- 오늘 입력 기준으로는 단백질이 부족합니다. 다음 식사에 단백질부터 챙기세요.
- 계산용 체중을 바꾸면 오늘 목표 칼로리와 탄단지가 다시 계산됩니다.
- 오늘 공복 체중은 기록에 남기고, 계산용 체중은 전체 설정에서 관리합니다.
- InBody 측정 체중은 인바디 기록 안에서만 확인합니다.

나쁜 예:
- 체중이 줄고 있습니다.
- 체성분 흐름이 보입니다.
- 최근 변화가 있습니다.
- 목표와 맞습니다.
- 인바디도 참고됩니다.
- 체성분 변화도 보조로만 보입니다.

InBody 탭 역할:
- InBody 탭은 중립 관찰 탭이다.
- InBody 탭에서는 목표별 행동 조언을 하지 않는다.
- InBody 탭은 기록 수, 변화 방향, 측정 조건, 의심스러운 값 여부, 변화 판단 가능 여부를 말한다.
- 행동 조언은 Today Coach에서 한다.
- InBody 탭 문구는 “다이어트니까 이렇게 하세요 / 벌크업이니까 이렇게 하세요”로 가지 않는다.

Today Coach 역할:
- Today Coach는 InBody 분석 결과를 적극 활용할 수 있다.
- InBody 분석 결과가 오늘 행동을 바꾸려면 기록 신뢰도 조건이 필요하다.
- Today Coach에서 InBody를 말할 때는 “인바디 분석 결과”라고 밝힌다.
- 체중 기록을 말할 때는 “체중 기록”이라고 밝힌다.
- 두 근거를 함께 쓰는 경우 반드시 분리해서 말한다.

InBody 연결 권한:

Blocked / Low:
- InBody 분석 결과를 Today Coach 행동에 쓰지 않는다.
- InBody 탭에서는 왜 경향 판단이 어려운지와 다음 측정 행동을 말한다.
- Today Coach에는 상시 노출하지 않는다.
- 노출이 필요하면 이유와 다음 측정 행동을 함께 말한다.

Medium:
- InBody 분석 결과를 Today Coach의 reason에만 사용한다.
- 행동은 식단 기록, 최근 흐름, 목표 상태가 만든다.
- “인바디 분석 결과, 무엇이 보이는지”를 명확히 말한다.

High:
- InBody 분석 결과를 Today Coach의 action 우선순위에 반영할 수 있다.
- 단, 최근 식단 기록 / 체중 기록 / 목표 맥락과 방향이 맞아야 한다.
- 목표별 행동 문구로 번역한다.

사용자-facing 화면에는 confidence / blocked / low / medium / high 같은 내부 용어를 쓰지 않는다.
사용자에게는 “인바디 기록이 1회뿐이라”, “측정 간격이 짧아”, “수분 상태 영향일 수 있어”처럼 이유를 말한다.

낮은 신뢰도 InBody 문구:
- 인바디 기록이 아직 1회라 변화를 보기 어렵습니다.
- 인바디는 수분과 측정 조건 영향을 많이 받습니다. 다음 측정은 같은 시간대와 비슷한 식사 상태에서 기록하세요.
- 측정 간격이 짧아 수분 변화 영향이 클 수 있습니다. 2~4주 간격으로 같은 조건에서 한 번 더 기록하세요.
- 이번 인바디 변화가 너무 큽니다. 수분 상태나 측정 조건 영향일 수 있습니다. 값을 다시 확인하고 다음 측정 조건을 맞춰 기록하세요.
- 인바디 변화는 아직 보기 어렵습니다. 운동 및 식사 직후, 수분 상태 차이가 크면 결과가 흔들릴 수 있습니다. 다음 측정은 같은 시간대와 비슷한 식사 상태에서 기록하세요.

중간 신뢰도 InBody 문구:
- 인바디 분석 결과, 골격근량이 줄고 있습니다. 최근 단백질 부족도 반복됩니다.
- 인바디 분석 결과, 체지방이 늘고 있습니다. 최근에는 술과 기타 칼로리가 많습니다.
- 다이어트: 최근 14일 체중은 목표대로 줄고 있습니다. 다만 인바디 기록이 아직 적어 장기 흐름은 보기 어렵습니다.
- 린매스업: 최근 14일 체중은 목표대로 천천히 늘고 있습니다. 다만 인바디 기록이 아직 적어 장기 흐름은 보기 어렵습니다.
- 벌크업: 최근 14일 체중은 목표대로 늘고 있습니다. 다만 인바디 기록이 아직 적어 장기 흐름은 보기 어렵습니다.

높은 신뢰도 InBody 문구:

다이어트:

- 인바디 분석 결과, 체중은 줄었지만 골격근량도 같이 줄고 있습니다. 칼로리를 더 줄이지 말고 단백질부터 챙기세요.
- 인바디 분석 결과, 체지방 변화가 거의 없습니다. 술·기름진 음식과 간식·소스·음료부터 줄이세요.

린매스업:
- 인바디 분석 결과, 체중과 골격근량이 같이 늘고 있습니다. 현재 식사량을 유지하세요.
- 인바디 분석 결과, 체중은 늘었지만 체지방 증가가 더 큽니다. 칼로리를 더 올리지 말고 술·기름진 음식은 줄이세요.
- 인바디 분석 결과, 골격근량 증가는 아직 뚜렷하지 않습니다. 단백질과 탄수화물부터 챙기세요.

벌크업:
- 인바디 분석 결과, 체중과 골격근량이 같이 늘고 있습니다. 현재 식사량을 유지하세요.
- 인바디 분석 결과, 체지방만 빠르게 늘고 있습니다. 술·안주·튀김으로 더 채우지 마세요.
- 인바디 분석 결과, 체중 증가에 비해 골격근량은 크게 늘지 않았습니다. 단백질은 꼭 챙기고, 탄수화물은 식사에서 채우세요.

금지 표현:

- 보조 참고입니다.

식단 점수 자동 계산 토글은 사용자-facing 문구에서 다루지 않는다.
recordMode는 저장/백업/최근 흐름 판정용 내부 값이다. Records 화면에서는 사용자가 "기록 방식을 고른다"는 느낌보다, 이 날짜에 무엇이 남아 있는지를 먼저 말한다.

간단 식단 평가는 자동 점수가 아니다.
간단 평가는 과거 기록 또는 빠른 저장 흐름의 낮은 해상도 식단 근거다.
간단 평가는 탄수화물 / 단백질 / 지방 / alcoholKcal / otherKcal 세부 반복 판단에 쓰지 않는다.

- InBody 기록 목록을 기본 화면에서 숨길 경우 "기록 보기"는 InBody 내부 modal / bottom sheet를 여는 action으로 말한다.

최근 흐름 문구 기준:
- v7.0 이후 최근 흐름 탭의 기간 선택은 3 / 7 / 14 / 28일 기준으로 말한다.
- 1일은 Today / Records 당일 문맥으로 두고, 최근 흐름 탭에서 별도 참고 분석으로 만들지 않는다.
- 상세 기록일과 간단 기록일을 분리해서 말한다.
- 간단 평가를 자동 점수 평균처럼 말하지 않는다.
- 상·중·하 기록이 많으면 식단 원인을 단정하지 않고, 상세 기록을 남기도록 안내한다.
- 짧은 helper에서는 "탄단지 기록이 부족해요"를 쓰고, 설명이 필요한 위치에서는 "탄수화물·단백질·지방 기록이 부족해요"처럼 풀어 쓴다.
- 상세 기록으로 전환하면 식사별 탄단지 분석을 사용할 수 있음을 자연스럽게 알려준다.
- detailed 기록만 단백질 부족, 탄수 부족, 고지방, 술 영향, 기타 칼로리 같은 원인 날짜로 말할 수 있다.
- simple / weight_only 기록에서는 탄단지 원인을 단정하지 않고 "기록 부족" 또는 "식사 기록이 더 필요해요"처럼 위치에 맞게 말한다.

좋은 예:
- 최근 7일 중 상세 기록 4일, 간단 기록 2일입니다.
- 단백질 부족이 반복되는지 보려면 상세 기록이 필요합니다.
- 탄단지 기록이 부족해 식단 원인은 아직 말하기 어렵습니다.
- 상세 기록이 부족해서 식단 원인은 아직 말하기 어렵습니다.
- 탄단지 기록이 부족해요. 오늘 먹은 주요 식사부터 상세 기록으로 남겨주세요.
- 체중 기록은 있지만 식사 기록이 부족합니다. 오늘 먹은 주요 식사부터 상세 기록으로 남겨주세요.

나쁜 예:
- 기록이 있습니다.
- 현재 기록만으로 볼 수 있습니다.
- 최근 식단 점수 82점입니다.

- 자동 점수 계산을 끄면 상세 기록은 저장되지만 분석은 일부 가능합니다.

선택형 식사 상황 태그 문구 기준:
- 식사 상황 태그가 도입되더라도 필수 입력처럼 말하지 않는다.
- 태그가 없다는 이유만으로 기록이 부족하다고 단정하지 않는다.
- 태그는 Coach 문구와 최근 흐름 해석을 보조하는 힌트이지 점수 산식 직접 근거처럼 말하지 않는다.
- 태그 문구는 술자리, 외식, 기름진 식사, 간식/디저트, 야식처럼 쉬운 단어를 우선한다.
- 고염분, 가공식품, 소스 많음, 클린푸드, 치팅처럼 처음 보는 사용자가 부담스럽거나 판단적으로 느낄 수 있는 말은 초기 후보에서 피한다.

좋은 방향:
- 식사별 탄단지 기록이 쌓이면 원인 분석이 더 정확해집니다.
- 오늘 먹은 주요 식사부터 탄수화물·단백질·지방 기준으로 남겨 주세요.
- 체중·메모만 남아 있는 날은 체중 변화는 볼 수 있지만 식단 원인은 말하지 않습니다.

Today Coach 기준:
- 최근 상세 기록이 충분하면 단백질 / 탄수 / 지방 / 술 / 기타 칼로리 반복 문제를 말할 수 있다.
- 최근 간단 기록만 많으면 강한 탄단지 반복 조언을 하지 않는다.
- 최근 체중만 기록이 많으면 식단 원인을 단정하지 않는다.
- 오늘 상세 식사 기록이 있다면 오늘 입력 기준 행동은 말할 수 있다.

운동 여부 모드 문구 기준:
- 운동 여부 모드는 앱이 사용자를 어떤 기준으로 해석할지 정하는 상위 기준이다. 다만 이 사실을 Coach / 최근 흐름의 본문 문구에 그대로 설명하지 않는다.
- Coach / 최근 흐름은 앱 내부 정책을 설명하지 말고, 사용자가 지금 볼 수 있는 상태와 다음 행동을 말한다.
- 운동 안 함 모드에서는 운동 관련 판단 재료를 계산과 해석에서 제외하되, 사용자-facing 문구로 "운동 기록은 현재 관리 기준에 포함하지 않습니다", "일반 권장 범위 안에서 관리합니다"처럼 정책을 설명하지 않는다.
- 운동함 모드에서도 "체성분 변화와 식단 기록을 함께 봅니다"처럼 앱이 무엇을 함께 보는지 설명하는 문구를 중심 문구로 쓰지 않는다.
- 정책 / 기준 설명은 Settings 도움말, ? 툴팁, 온보딩, 기준문서 쪽으로 분리한다.
- Coach / 최근 흐름의 허용 문구는 행동, 해석, 주의점 중심이어야 한다.

피해야 할 문구 성격:
- 현재 관리 기준에 포함하지 않습니다.
- 일반 권장 범위 안에서 관리합니다.
- 체성분 변화와 식단 기록을 함께 봅니다.
- 이 앱은 운동 여부에 따라 다르게 해석합니다.
- 운동 관련 값은 계산에서 제외됩니다.

좋은 방향:
- 오늘은 식사 기록부터 채우세요.
- 최근 섭취 변동이 큽니다.
- 단백질이 며칠째 낮습니다.
- 운동 기록이 없는 날은 식사와 체중 흐름부터 확인하세요.
- 식사 기록이 부족해 원인 판단은 아직 이릅니다.

좋은 예:
- 이유: 탄단지 기록이 부족해 식단 원인은 아직 말하기 어렵습니다.
- 행동: 오늘 먹은 주요 식사부터 상세 기록으로 남겨주세요.
- 단백질 부족이 반복되는지 보려면 상세 기록이 더 필요합니다.
- 상태: 인바디 분석 결과, 체지방이 늘고 있습니다.
- 이유: 탄단지 기록이 부족해 식단 원인은 아직 말하기 어렵습니다.
- 행동: 오늘 먹은 주요 식사부터 상세 기록으로 남겨주세요.

금지:
- 자동 평가 ON/OFF 또는 식단 점수 자동 계산 토글 문구를 사용자-facing에 노출하기
- "자동 평가 OFF는 분석 OFF가 아니다" 같은 과도기 설명을 사용자-facing 기준으로 쓰기
- 간단 기록을 상세 탄단지 분석 근거처럼 말하기
- 간단 평가 90 / 75 / 60을 자동 adherencePercent와 같은 평균처럼 표시하기
- InBody high만으로 simple-only 최근 식단의 원인을 단정하기

테스트 기준:
- 문구 테스트는 simple / detailed / weight_only의 판단 권한을 분리해서 확인한다.
- 문구 테스트는 “기록이 있습니다” 같은 빈 문구를 고정하지 않는다.
- Today Coach 테스트는 simple-only 최근 흐름에서 macro 반복 원인을 단정하지 않는지 확인한다.

C4.8-M / CAL / UI5 이후 사용자-facing 문구는 아래 기준을 우선한다.

- 일반인 / 운동인 / 전문가 같은 사용자 등급 선택 문구는 기본 화면에 다시 노출하지 않는다.

- 내부적으로 남은 mode, legacy, compatibility 같은 단어를 사용자-facing에 쓰지 않는다.
- targetCal은 “목표 칼로리” 또는 “오늘 목표 칼로리”로 말하고, guide/activity basis는 목표 칼로리를 바꾸는 선택처럼 말하지 않는다.
- guide/activity basis는 사용자-facing 문구가 아니라 내부 기준명이다. 화면 문구는 "운동 기준 사용" 또는 "일반 식단 관리" 같은 관리 방식으로 말하고, 두 내부 기준이 항상 같은 목표 안에 있다고 설명하지 않는다.
- 탄수화물이 가이드 하한보다 낮게 조정된 경우는 “목표 안에서 조정”처럼 말하되, 기본 화면에 긴 계산 설명을 깔지 않는다.
- 기록 부족 Coach는 macro 진단을 여러 장 쌓지 않는다. 첫 행동과 기록 부족 이유만 말한다.
- Stitch UI 이후 문구는 카드/칩/아이콘이 이미 말하는 내용을 반복하지 않는다.
- 버튼, 칩, 카드 제목, 빈 상태, Coach 문구는 각각 다른 길이와 역할을 가져야 한다.
- UI가 연결감 있게 바뀌더라도 Records 공복 체중, Today 계산용 체중, InBody 측정값처럼 오해하면 안 되는 값은 필요한 위치에서 역할을 붙여 말한다.

v7.0 Stitch 반영 문구:

- “정책”, “fallback”, “legacy”, “schema”, “payload”, “draft”, “source of truth”, “selectedBasis”, “localStorage” 같은 내부 용어는 계속 금지한다.

16. 문구 QA 체크리스트

문구를 추가하거나 수정할 때 아래를 확인한다.

1. 결론이 먼저 나오는가?
2. 사용자가 지금 할 행동을 바로 알 수 있는가?
3. 목표별 판단 차이가 문구에 반영됐는가?
4. 판단을 지나치게 완화하지 않았는가?
5. 당연한 기록, 사실을 코치문구에 나열하지 않았는가?
6. 사용자가 느끼기에 "그래서 어쩌라고?" 같은 반응이 나오지 않는가?
7. 술/회복/전날 과식의 원인과 행동이 분명한가?
8. 기록 부족 상태에서 식단을 단정하지 않는가?
9. 내부 용어가 기본 화면에 노출되지 않는가?
10. 같은 의미가 여러 카드에서 반복되지 않는가?

17. 최종 기준
────────────────────────

좋은 문구는 다음 네 가지를 만족한다.

1. 짧다.
2. 바로 이해된다.
3. 목표별 판단이 맞다.
4. 지금 할 행동이 보인다.

최종 한 줄 기준:

“좋게 들리는 문장”이 아니라,
“지금 바로 어떻게 먹을지 말해주는 문장”이어야 한다.
