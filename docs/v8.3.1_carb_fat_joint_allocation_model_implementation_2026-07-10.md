# v8.3.1 carb-fat joint allocation model implementation

Date: 2026-07-10

## Scope

This branch implements the v8.3.1 carb/fat joint allocation model that was selected in `docs/v8.3.1_carb_fat_exchange_joint_allocation_model_decision_2026-07-10.md`.

Root problem:
- The app had moved toward calorie-equivalent carb/fat exchange, but the Today card could still expose carb and fat as independent ranges.
- That made impossible corner combinations look available, for example "carb upper" and "fat upper" both appearing acceptable under one fixed kcal target.
- The right fix is not card help text or a tooltip. The app should calculate the feasible carb/fat pair correctly and show compact numbers.

## Implemented

- Added `getV831CarbFatJointAllocationModel`.
- The model reserves protein first, then computes the carb/fat budget:
  - `availableCarbFatKcal = target.kcal - proteinReservedG * 4`
- It exposes:
  - raw carb/fat ranges
  - joint feasible carb/fat ranges
  - conditional display ranges based on current counterpart intake
  - base target pair
  - adaptive target pair
  - macro kcal gap
  - 3-day / 7-day trend limiter evidence
- `getV83AdaptiveTodayMacroTarget` now uses the joint model for the adaptive carb/fat target pair.
- `getV83ContinuousMacroScoreSummary` exposes `jointAllocationModel` inside `scoringContext`.
- `getTodayMacroCardRange` reads `scoringContext.jointAllocationModel.conditionalDisplayRanges` before falling back to older range context.

## Important Boundary

The first draft tried to replace the scoring penalty `carbRange` / `fatRange` with the joint display ranges. That was too broad.

Final implementation:
- Keeps existing continuous score penalty axes intact.
- Uses the joint allocation model for adaptive Today target and card conditional feasible ranges.
- Exposes the model in scoring context so card display and QA can verify the same evidence.

Reason:
- Score penalties already include energy/TDEE pressure and carb/fat exchange failure logic.
- Replacing the penalty range itself made a high-volume synthetic fixture worse than rest because its fixed target macros had a small macro-kcal mismatch.
- That would have changed score semantics while fixing a display/allocation problem.

## Regression Coverage

Updated `runMacroRangeContinuousScoringTests`:
- Added one v8.3.1 joint allocation model case.
- Guards protein reserve plus carb/fat budget equality.
- Guards that raw independent carb/fat upper corners are recognized as infeasible.
- Guards joint carb upper with fat lower and joint fat upper with carb lower.
- Guards current high-carb intake lowering conditional fat upper.
- Guards current high-fat intake lowering conditional carb upper.
- Guards Today macro card carb/fat ranges reading the joint conditional model.
- Extended adaptive Today target test to require the v8.3.1 joint model and joint-range bounds.
