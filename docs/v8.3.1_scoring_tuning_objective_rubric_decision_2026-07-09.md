## 4. External anchor vs point value

External references can say:
- carbohydrate needs rise with training load
- protein sufficiency has a practical range
- dietary fat has a general range and a floor
- alcohol has standard-drink and high-risk patterns
- total calorie surplus/deficit matters for body composition goals

External references usually do not say:
- TDEE 1.2x equals 82 points
- fat upper 2.0x equals 49 points
- alcohol 112g equals 9 points
- high-volume 700g carbohydrate equals 0 points

Therefore:

```text
external anchor -> physiological zone
objective rubric -> app score band target
curve candidate -> coefficients that hit the band
regression tests -> guard against side effects
```

## 5. Score-band language

The app should use score bands as translation targets, not exact user-picked values.

| band | score range | product meaning |
| --- | ---: | --- |
| aligned | 95-100 | 목표/근거와 잘 맞는 날 |
| acceptable | 85-94 | 작은 이탈은 있지만 좋은 날 |
| warning | 70-84 | 명확히 주의할 점이 있으나 하루 전체가 망가진 것은 아님 |
| problem | 50-69 | 뚜렷한 식단 문제 |
| serious | 30-49 | 목표 달성 관점에서 심각한 문제 |
| severe | 1-29 | 생리학적/목표 관점에서 거의 실패에 가까운 날 |
| display collapse | 0 | 최종 표시 점수 하한. raw score는 계속 더 나빠질 수 있음 |

Rules:
- These are objective target bands for scenario design.
- They are not new hard caps.
- A future curve can output any exact number inside a band.
- The final 0 clamp is display boundary, not a hidden hard threshold.
- Raw score must still be allowed to worsen past 0 for regression evidence.
