## 2. 승인 가능한 것

현재까지 승인 가능한 것:

```text
numeric preview는 제품 문제로 추적할 가치가 있음
숫자 후보는 production-facing field가 아니라 testLocalNumericPreview nested object에만 둘 수 있음
full_day_candidate와 sufficient detailed old fixed Records만 test-local numeric candidate 대상이 될 수 있음
simple / weight_only / snapshotless / insufficient Records는 null/blocked로 유지해야 함
kcal/protein anchor lane과 fat/carbExchange range evidence lane을 분리해야 함
inside_recommended / exchange_explained는 bonus가 아님
candidate score version naming은 ADHERENCE_SCORING_VERSION과 분리된 test-local preview 이름이어야 함
consumer no-leak는 normalizer / backup / restore / Recent / score / DailyCoach / UI / Records state 기준으로 테스트할 수 있음
macro-policy suite와 core profile registration은 coverage 차이가 없어야 함
```

승인 가능한 작업 성격:

