# v8.3.1 adaptive-target stable help implementation

## 비개발자용 설명

### 무엇을 바꿨는지

`오늘 목표 자동 조정`을 Settings와 Today가 서로 다르게 설명하던 문제를 정리했다.

- Settings에는 짧은 한 줄 설명과 `?` 버튼을 함께 둔다.
- Settings의 상세 도움말은 자동조정 기능의 전체 의미를 설명한다.
- Today의 `?` 버튼은 같은 설명 원본에서 가져온 짧은 참조만 보여 준다.
- 모바일에서는 반복되는 `오늘 목표 섭취량` 제목을 계속 숨기되, 도움말 버튼은 칼로리 카드 오른쪽 위에 남긴다.
- Today와 Settings 도움말은 모두 실제 버튼이며, 어떤 설명과 연결됐는지 화면 읽기 도구가 알 수 있게 했다.
- 키보드 초점, Enter/Space, Escape, 바깥 클릭 또는 탭으로 열고 닫을 수 있다.

### 왜 바꿨는지

이전 Settings 문구는 기록이 쌓이면 목표가 바뀐다고 설명했고, Today 문구는 먹은 기록에 맞춰 목표가 바뀐다고 설명했다. 둘 다 총칼로리를 늘리는 기능인지, 단백질도 바뀌는지, 많이 먹은 양을 새 권장량으로 인정하는지 분명하지 않았다.

더 큰 문제는 모바일에서 도움말이 제목과 함께 완전히 사라졌다는 점이다. 데스크톱에서도 Today 도움말은 겉으로 버튼처럼 보이는 `span`이었고, 실제 설명과 접근성 연결이 없었다.

이번 변경은 기능의 변하지 않는 뜻을 Settings가 책임지고, Today는 그 뜻을 짧게 확인하는 역할만 맡게 한다. 오늘 실제로 탄수화물이나 지방이 몇 g 움직였는지와 그 이유는 아직 증거가 부족하므로 정적 도움말에 넣지 않았다.

### 실제 사용자 화면이나 계산 결과가 어떻게 달라지는지

Settings에는 다음 짧은 뜻이 먼저 보인다.

```text
총칼로리와 단백질 기준은 유지하고, 탄수화물·지방 배분만 조정합니다.
```

`?`를 열면 총 목표 칼로리를 늘리지 않고 단백질을 먼저 확보한 뒤 남은 칼로리 안에서 탄수화물과 지방만 조정한다는 전체 설명이 나온다. 한쪽으로 치우친 기록이 반복되면 같은 방향의 이동 폭이 줄 수 있고, 많이 먹은 만큼을 새 권장량으로 인정하는 기능은 아니라는 경계도 포함한다.

Today에서는 긴 설명을 카드 본문에 추가하지 않았다. 데스크톱에서는 `오늘 목표 섭취량` 제목 옆에 `?`가 있고, 390px 모바일에서는 제목 대신 `?`만 칼로리 카드 오른쪽 위에 보인다. 실제 브라우저 확인에서 모바일 칼로리 카드 높이는 약 176px로 기존 수준을 유지했고 가로 넘침도 생기지 않았다.

### 정책, 산식, 데이터 해석이 바뀌었는지

변경 없음.

예를 들어 자동조정 전후의 총 목표가 2,000kcal이고 단백질 기준이 120g이라면 이번 작업은 그 숫자를 바꾸지 않는다. 남은 칼로리 안에서 탄수화물 25g이 줄면 지방이 약 11g 늘 수 있다는 기존 같은-칼로리 배분 구조도 그대로다. 이번 작업은 그 기능의 안정적인 뜻만 설명한다.

- `adaptiveMacroTargetsEnabled` 기본값, ON/OFF, 저장과 복원은 그대로다.
- protein reserve, carb/fat 같은-칼로리 joint allocation, 최근 기록 pressure, limiter와 boundary 계산은 그대로다.
- 목표 kcal, 단백질·탄수화물·지방 수치와 Records snapshot은 그대로다.
- scoring version, percent, rawScore, 9개 penalty breakdown은 그대로다.
- 운동은 계속 점수 보너스가 아니라 탄수화물 곡선의 맥락으로만 남는다.

### 바꾸지 않은 범위와 보류한 내용

- 오늘 실제 자동조정 방향, 변화량, 최근 기록 신뢰도, 제한 원인은 설명하지 않았다.
- `3일 연속`, `7일 연속`, `더 이상 조절할 수 없음` 같은 문구를 만들지 않았다.
- DailyCoach semantic v2, Hero/Coach 중복 정리, Coach 말투 설정을 구현하지 않았다.
- broad tooltip/glossary와 app-wide copy batch를 열지 않았다.
- score formula, anchor, curve, adaptive limiter, storage/schema/backup/Records migration을 바꾸지 않았다.
- v8.3 title, version, stage와 tag를 바꾸지 않았다.

사용되지 않던 과거 점수 문구 함수는 실제 호출이 없음을 확인한 뒤 제거했다. report-only 설명에 남아 있던 제거된 함수명도 현재 점수 근거 모델과 렌더러 이름으로 정리했다. 이 정리는 점수 계산이나 사용자 화면의 점수값을 바꾸지 않는다.

### 사용자가 큰 틀에서 확인해야 할 판단점

추가 판단 필요 없음.

이번 도움말은 “이 기능이 항상 무엇을 하는가”만 말한다. “오늘 왜 탄수화물이 몇 g 움직였는가”는 다음 DailyCoach semantic v2가 실제 계산 증거를 전달받은 뒤 설명하는 것이 맞다. 이 책임 분리를 유지하는지가 다음 작업의 핵심 판단점이다.

### 테스트에서 무엇을 검증했는지

- Settings와 Today가 하나의 공용 설명 source를 사용하는지 확인했다.
- Settings는 전체 정의, Today는 짧은 참조만 표시하는지 확인했다.
- 390px 모바일에서 Today 도움말 버튼이 실제로 보이고 카드 안에 남는지 확인했다.
- 두 trigger가 native button이고 `aria-describedby`가 실제 `role="tooltip"` 설명을 가리키는지 확인했다.
- 클릭/탭, 키보드 초점, Enter, Escape, 바깥 클릭 닫기가 동작하는지 확인했다.
- 도움말을 열고 닫아도 자동조정 목표쌍, 목표 kcal, 단백질 목표, 최종 점수, raw score, 9개 감점 내역이 달라지지 않는지 확인했다.
- 기존 ON/OFF 설정과 저장값이 계속 동작하는지 확인했다.
- 데스크톱과 모바일 실제 화면에서 tooltip이 화면 밖으로 넘치지 않는지 확인했다.
