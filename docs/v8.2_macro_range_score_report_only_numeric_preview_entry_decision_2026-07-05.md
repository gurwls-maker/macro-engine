## 2. entry decision

결정:

```text
report-only object에 test-local numeric output을 넣는 것은 다음 단계에서 검토 가능하다.
허용되는 위치는 nested testLocalNumericPreview 뿐이다.
허용되는 이름은 testLocal prefix 뿐이다.
candidateScorePreview / pointsPreview / scoreDeltaPreview 이름은 계속 production-facing closed field다.
```

허용 후보 shape:

```text
testLocalNumericPreview: {
  testLocalOnly: true,
  persisted: false,
  uiVisible: false,
  testLocalCandidateScorePreview: number | null,
  testLocalComponentPoints: object | null,
  testLocalCandidateScorePreviewVersion: "range_score_candidate_preview_test_local_v0",
  testLocalFixedScoreBasisLabel: "fixed_adherence:v6.1_alcohol_penalty_v2",
  testLocalCandidateScoreBasisLabel: "macro_range_candidate_preview_test_local",
  testLocalReasons: string[],
  testLocalBlockedReason: string | null
}
```

금지 shape:


## 5. scoreDeltaPreview remains closed

`scoreDeltaPreview`는 이번 decision으로 열리지 않는다.

이유:

```text
fixed score와 candidate score를 직접 비교한다.
사용자가 production score 개선 / 악화로 읽을 수 있다.
old fixed Records의 saved score 의미를 흔드는 것처럼 보일 수 있다.
UI copy / Recent / DailyCoach로 번지기 쉽다.
```

따라서 다음 구현이 report-only `testLocalNumericPreview`를 열더라도:

```text
comparison.scoreDeltaPreview: null
top-level scoreDeltaPreview: absent
```
