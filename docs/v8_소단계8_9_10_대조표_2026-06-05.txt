v8 소단계8-10 대조표
작성일: 2026-06-05
목적: 소단계8 전체 앱 잔존 감사와 소단계9 정리 결과를 대조하고, 소단계11에서 닫아야 할 항목을 100% 완료 기준에 가깝게 명확히 고정한다.

이 문서는 "고친 것처럼 보이는가"가 아니라 "사용자 의도대로 충분히 닫혔는가"를 판단하기 위한 대조표다.
기준문서는 정답지가 아니며, 실제 앱 렌더 / 실제 사용자 흐름 / 실제 코드 경로 / 테스트 / 최신 사용자 의도가 문서보다 우선한다.

────────────────────────

현재 브랜치:
- codex/v8-formula-audit

- 전체 앱 잔존 버그 감사.
- 코드 수정 없음.

소단계10:

- 코드 기능 수정이 아니라, 소단계11 작업 범위와 완료 기준을 고정한다.

소단계11 목표:

- "기능이 없다"와 "기능이 숨겨져 있다"를 구분한다.
- "legacy compatibility로 유지"와 "사용자 신규 입력 경로로 살아 있음"을 구분한다.

1. 완료 판정 언어
────────────────────────

닫힘:
- 사용자 경로에서 보이지 않는다.
- 코드 경로가 의도와 맞다.
- 테스트가 새 의도를 잠근다.
- 문서가 새 의도와 충돌하지 않는다.
- 렌더 감사 또는 실제 뷰포트 확인이 있다.

부분 닫힘:
- 기본 사용자 경로는 닫혔지만 DOM, 직접 호출 함수, 오래된 테스트, 문서, 렌더 감사 중 일부가 남아 있다.

레거시 유지:
- 과거 기록/백업/최근 흐름 해석을 위해 내부 값으로 남기는 것이 합리적인 경우다.
- 이 경우 사용자에게 "새 기록 방식 선택지"처럼 보이면 안 된다.
- 남기는 이유가 주석/문서/테스트 중 하나에 기록되어 있어야 한다.

미닫힘:
- 사용자가 볼 수 있거나, 직접 호출/이벤트/테스트/문서가 이전 의도를 정상으로 잠그고 있다.

검증 필요:
- 현재 코드상 추정은 가능하지만 실제 렌더/시나리오로 아직 확인하지 못했다.

3. 탭별 소단계11 체크리스트
────────────────────────

3-1. Today

소단계11에서 닫아야 할 것:
- Today `기록 시작하기`는 상세 식사 입력만 연다.
- old choice grid는 사용자 신규 경로에서 삭제한다.
- `간단 기록`, `식단 건너뛰기`, `기록 방식 다시 선택`은 Today modal visible text에 나오지 않는다.
- 직접 호출 가능한 `openTodayRecordStartDialog("simple")`, `openTodayRecordStartDialog("skip")`의 존재 이유를 결정한다.
  - 제거하면 가장 명확하다.
  - 남길 경우 테스트 이름과 문서가 "legacy-only"라고 말해야 하며, 사용자 버튼/이벤트에서 접근 불가해야 한다.
- `[data-open-today-record-start]` fallback `choices`는 상세 전용으로 바꾸거나 제거한다.
- 상세 식사 저장 시 goalSnapshot, basis signature, current record confirmation이 정상이다.
- 체중만 입력하는 shortcut은 식사 시작 모달이 아니라 체중·메모 경로에 있어야 한다.

필수 검증:
- Today 빈 식사 카드에서 `기록 시작하기` 클릭.
- Today 이미 상세 기록 있음 상태에서 `식사 추가하기` 클릭.
- Today meal start에서 저장 후 기록탭 당시 목표/당시 계산 기준 확인.
- 식사 0건, 식사 1건, 체중만 있는 기록, 기존 simple legacy record가 있을 때 Today 상태창 확인.

3-2. Records

소단계11에서 닫아야 할 것:
- 새 기록을 만드는 사용자 선택지는 `식사 추가`, `체중·메모 수정`으로 충분한지 확인한다.
- 빈 날짜에는 날짜 선택 / 식사 추가 / 체중·메모 수정이 자연스럽게 보인다.
- 기존 기록 상세는 `기록 방식`보다 `식사`, `체중`, `메모`, `InBody`, `당시 목표/당시 계산 기준`의 정보 구조로 보인다.
- legacy simple/weight-only 기록은 보관/해석되지만 신규 선택지처럼 보이지 않는다.
- 수정하기 전/후 레이아웃, 열린 섹션 유지, 버튼 폰트/라운딩/밀도 통일을 렌더로 확인한다.

필수 검증:
- 기록 없음 날짜 선택.
- 식사 1건만 있는 날짜.
- 식사 여러 건 있는 날짜.
- 체중·메모만 있는 날짜.
- legacy simple 기록 날짜.
- 삭제 후 빈 날짜.
- 모바일 Records, desktop Records 모두 확인.

3-3. Recent

소단계11에서 닫아야 할 것:
- low-data는 중립 안내로만 보인다.

4. 소단계11의 "100%에 가깝다" 기준
────────────────────────

소단계11에서 "완전 닫힘"이라고 말하려면 아래 조건을 모두 만족해야 한다.

1. 사용자-facing 잔존 제거
- Today 신규 기록 시작 경로에 `간단 기록`, `식단 건너뛰기`, `체중만 기록`, `기록 방식 다시 선택`이 보이지 않는다.
- Records 신규 작업공간에 `체중만 기록 / 간단 기록 / 상세 기록`이 선택지처럼 보이지 않는다.
- Recent/Coach/InBody에서 low-resolution 기록을 상세 탄단지 원인처럼 말하지 않는다.

2. 코드 경로 정리
- simple/skip 저장 함수는 제거하거나 legacy-only로 격리한다.
- `openTodayRecordStartDialog("simple")`, `openTodayRecordStartDialog("skip")`이 신규 Today 경로로 쓰이지 않는다.
- `data-open-today-record-start` fallback이 old choices를 열 수 없다.
- recordMode는 internal persistence/backup/recent inference 용도로만 유지된다.

3. 테스트 기준 개정
- old simple/skip Today modal 저장 성공을 정상으로 잠그는 테스트를 그대로 두지 않는다.
- 신규 테스트는 다음을 잠근다.
  - Today 기록 시작은 상세 식사 폼만 연다.
  - Records 작업공간은 식사 추가 / 체중·메모 수정만 노출한다.
  - legacy simple/weight-only 기록은 분석 권한을 제한한다.
  - 부족 데이터는 중립 톤이다.
  - InBody analysis/change 렌더가 실제로 존재한다.

4. 문서 기준 개정
- `docs/앱-문구-기준.txt`의 UX2 Today 기록 시작 기준에서 old 3분기 문구를 현재 기준으로 고친다.
- `docs/캔버스_updated.txt`의 old Today record start 설명은 historical로 표시하거나 현재 기준과 충돌하지 않게 정리한다.
- 소단계11이 끝난 뒤 이 대조표의 미닫힘/부분 닫힘 항목을 닫힘/레거시 유지/후속 후보 중 하나로 갱신한다.

- Today 계산용 체중 반영 shortcut은 "오늘 계산에도 반영"처럼 명시적으로만 작동해야 한다.
- 문서가 old 3분기를 말하고 있으면 다음 코덱스가 old UI를 다시 살릴 수 있다. 문서 충돌은 코드 버그만큼 중요하다.
