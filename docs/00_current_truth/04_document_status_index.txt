상태 정의
- KEEP: 최신 기준에서도 유지할 원칙 또는 guard.
- SUPERSEDE: v8.3에서 대체된 판단. 직접 구현 기준으로 쓰면 안 됨.
- HISTORICAL: 구현 기록 / report-only 기록 / 과거 decision. 최신 지시로 쓰지 않음.
- REVIEW: current truth와 대조해 추가 정합성 확인 필요.
- ARCHIVE: root docs에서 archive 이동 완료 또는 archive 유지 대상.
- KEEP_CURRENT_REFERENCE: 최신 작업에서도 참고 가치가 있으나 current truth보다 우선하지 않음.
- ABSORBED_INTO_CURRENT_TRUTH: 핵심 원칙은 current truth에 이미 흡수됐고, 원문은 근거/맥락으로만 읽음.
- EXTERNAL_REFERENCE_ONLY: 외부근거/해설 reference이며 구현 지시서가 아님.
- COPY_REFERENCE_ONLY: 사용자-facing copy 기준으로만 읽고 score formula 지시서로 쓰지 않음.
- HISTORICAL_REFERENCE: historical/report-only map 또는 과거 감사 문맥. 최신 지시로 쓰지 않음.
- ROUTED_REFERENCE: root docs에서 reference folder로 이동 완료. current truth보다 우선하지 않음.
- REVIEW_BEFORE_MOVE: 과거 role review 당시 실제 rename/move/archive 전 확인 대상이었음을 뜻하는 historical 상태.
- CANDIDATE_ARCHIVE_AFTER_CONSOLIDATION: 과거 role review 당시 통폐합 decision 후 archive/reference 폴더 이동 후보였음을 뜻하는 historical 상태.

target/scoring alignment release-blocker 기준
===========================================

- 앱 target macros 그대로 섭취했을 때 valid full-day/no alcohol/no outlier 조건에서 점수가 낮게 떨어지면 release blocker다.
- target carbs/protein/fat은 displayed scoring range 안에 있거나, 범위 밖일 경우 명시적인 제품 의미와 테스트가 있어야 한다.
- macro card range는 scoring boundary와 일치해야 하며 UI-only display cap으로 숨기지 않는다.
- 8~12g/kg carbohydrate external anchor는 high-volume 무감점 허가증이 아니라 carb curve anchor다.
- exercise context는 final score bonus도 coarse bucket 면죄부도 아니며, continuous factor/shared policy로 재정렬되어야 한다.
- automatic exercise context label은 설명용일 수 있지만, actual carb context는 `continuous_training_load_interpolation`으로 보간되어야 한다.
- 3일/7일 limiter는 count bucket이 아니라 continuous recency-weighted excess pressure 모델이어야 한다.
- `오늘 목표 자동 조정`은 persistent setting surface이며, 대규모 migration은 아니지만 저장되는 설정 field 추가 사실을 숨기지 않는다.
- Today macro card의 carb/fat range chip은 marginal range이며, 모든 chip corner combination을 동시에 가능한 식단 조합으로 암시하면 안 된다.
- Carb/fat exchange는 gram 1:1이 아니라 calorie-equivalent exchange다. 다음 작업은 사용자 설명이 아니라 joint allocation model을 먼저 닫아야 하며, UI-only display cap으로 숨기면 안 된다.
