- current truth와 다르게 가야 한다면 이유를 결과로그에 남긴다.
- "문서에 있으므로 맞다"는 통과 사유가 아니다.
- "테스트가 통과했으므로 생리학적으로 맞다"도 통과 사유가 아니다.

- 이미 검증된 docs-only/readiness/copy branch의 merge/publish는 대형 독립 작업으로 쪼개지 않는다. 짧은 HEAD/clean/test/ref 체크 후 병합/push하고, 실제 열린 다음 작업이 있으면 같은 흐름에서 이어간다.
- 단, scoring formula/range calculation/storage/schema/tag movement처럼 publish 경계 자체가 위험한 경우에는 별도 audit/closeout으로 분리한다.

macro scoring 금지선
- v8.2 fixed penalty table을 production 최종 산식으로 쓰지 않는다.
- high = -10, severe = -16 같은 상태별 고정 감점표를 v8.3 production body로 되살리지 않는다.
- 중간 score cap을 정책으로 쓰지 않는다.
- 마지막 0~100 clamp는 표시 범위로만 허용한다.
- exercise bonus를 주지 않는다.
- 운동은 finalScore 보너스가 아니라 carb curve context로만 사용한다.
- v6.1 alcoholImpactPenalty 후처리를 되살리지 않는다.
- alcohol은 kcal 축과 physiology risk 축으로 분리한다.
- scoreDeltaPreview product path는 폐기한다. repo 밖 일회성 forensic 분석이 필요하면 별도 명시 지시로만 다룬다.
- old fixed records는 제품 보존 요구가 아니다. dev/local legacy records는 explicit cleanup/reset/fallback decision으로 다룬다.
