충돌 처리
- current truth와 오래된 v8.2 문서가 충돌하면 current truth가 우선이다.
- current truth와 최신 사용자 의도가 충돌하면 최신 사용자 의도가 우선이다.
- current truth와 다르게 가야 한다면 이유를 결과로그에 남긴다.
- "문서에 있으므로 맞다"는 통과 사유가 아니다.
- "테스트가 통과했으므로 생리학적으로 맞다"도 통과 사유가 아니다.
- REQUIRED_NEXT_GATES는 증명이 아니라 가설이다. 실제 문제를 평문으로 다시 말하고, 선택한 작업이 그 문제를 푸는지 확인해야 한다.
- 제품/수학/UI 문제는 문서 라우팅만 보지 말고 관련 코드 경로, 화면 동작, 데이터 shape, fixture logic 중 필요한 증거를 확인한다.
- 이미 검증된 docs-only/readiness/copy branch의 merge/publish는 대형 독립 작업으로 쪼개지 않는다. 짧은 HEAD/clean/test/ref 체크 후 병합/push하고, 실제 열린 다음 작업이 있으면 같은 흐름에서 이어간다.
- 단, scoring formula/range calculation/storage/schema/tag movement처럼 publish 경계 자체가 위험한 경우에는 별도 audit/closeout으로 분리한다.
- publish가 끝났다는 이유만으로 다음 구현을 지어내지 않는다. 실제 이슈 또는 current gate가 있으면 진행하고, 없으면 monitor로 닫는다.

macro scoring 금지선
- v8.2 fixed penalty table을 production 최종 산식으로 쓰지 않는다.
- high = -10, severe = -16 같은 상태별 고정 감점표를 v8.3 production body로 되살리지 않는다.
- 중간 score cap을 정책으로 쓰지 않는다.
- 마지막 0~100 clamp는 표시 범위로만 허용한다.
- exercise bonus를 주지 않는다.
- 운동은 finalScore 보너스가 아니라 carb curve context로만 사용한다.
- v6.1 alcoholImpactPenalty 후처리를 되살리지 않는다.
- alcohol은 kcal 축과 physiology risk 축으로 분리한다.
- scoreDeltaPreview product path는 폐기한다. repo 밖 일회성 forensic 분석이 필요하면 별도 명시 지시로만 다룬다.
- old fixed records는 제품 보존 요구가 아니다. dev/local legacy records는 explicit cleanup/reset/fallback decision으로 다룬다.

target/scoring alignment release blocker
- 앱이 제시한 target macros를 그대로 섭취했을 때 valid full-day/no alcohol/no outlier 조건에서 낮은 점수로 떨어지면 실패로 본다.
- target macro generator와 v8.3 scoring range는 weight/TDEE/training/fat range source를 공유하거나, 서로 다를 경우 명시적인 제품 의미가 있어야 한다.
- macro card 오늘 범위는 scoring boundary와 일치해야 하며 UI-only display cap으로 mismatch를 숨기지 않는다.
- 8~12g/kg carbohydrate external anchor는 high-volume 무감점 허가증이 아니라 carb curve anchor다.
- 운동 context는 final score bonus도 coarse bucket 면죄부도 아니며, target/scoring shared policy의 continuous factor로만 반영한다.
- 자동 운동 context는 `continuous_training_load_interpolation`으로 rest -> normal -> high anchor를 보간하며 label threshold로 carb boundary가 뛰면 안 된다.
- Today adaptive target이 켜져 있으면 탄수화물과 지방 목표는 같은 kcal 안에서 교환될 수 있고, 반복 고탄수/고지방 흐름은 3일/7일 continuous pressure limiter로 이동폭을 줄인다.
- `오늘 목표 자동 조정`은 persistent setting surface다. 대규모 migration은 아니지만 저장되는 설정 field가 추가됐음을 결과로그에서 명시한다.
- Today macro card의 carb/fat range chip은 각각의 marginal range다. 사용자가 carb upper와 fat upper/lower를 동시에 모두 허용된 조합으로 읽게 만들면 안 된다.
- Carb/fat exchange는 gram 1:1이 아니다. 같은 kcal 안에서 carb 1g은 약 fat 0.44g, fat 1g은 약 carb 2.25g에 대응한다.
- 사용자에게 탄수/지방 교환 계산식을 설명하는 것으로 해결하지 않는다. 앱 내부 모델이 맞게 계산하고 화면은 결과 숫자만 자연스럽게 보여준다.
