# v8.2 macro range score Records latest policy correction

상태:

```text
작업 성격: Records latest-policy correction docs-only
index.html 변경: 없음
package.json 변경: 없음
runner 변경: 없음
production score 변경: 없음
getDailyAdherenceScore 변경: 없음
ADHERENCE_SCORING_VERSION 변경: 없음
candidateScorePreview 숫자화: 없음
pointsPreview 숫자화: 없음
scoreDeltaPreview 숫자화: 없음
storage/schema 구현: 없음
backup / Recent / UI / DailyCoach 연결: 없음
production macro range 구현: 없음
```

이 문서는 v8.2 macro range score 문서들에서 `old fixed Records no-reinterpret`와 `old fixed Records excluded`가 같은 의미처럼 읽히는 부분을 보정한다.

current source of truth:

```text
저장값 변경 금지 != 재계산 금지
```

## 1. correction reason

이전 문서들은 old fixed Records의 stored score basis를 보호하기 위해 보수적으로 작성되었다.

보호해야 하는 것은 다음이다.

```text
원본 record
meal data
goalSnapshot
backup schema
saved adherencePercent
saved adherenceScoringVersion
production-facing score field
```

하지만 이 보호선은 latest-policy preview 계산 자체를 영구 금지한다는 뜻이 아니다.

## 2. corrected policy

결정:

```text
old fixed Records의 stored/raw score basis는 fixed로 유지한다.
saved adherencePercent를 덮어쓰지 않는다.
saved adherenceScoringVersion을 candidate version으로 바꾸지 않는다.
goalSnapshot / record / backup schema에 candidate score field를 저장하지 않는다.
```

동시에:

```text
old fixed Records는 자동 제외 대상이 아니다.
detailed macro data와 fixed goalSnapshot이 충분한 old fixed Record는 latest-policy candidate preview eligibility 후보가 될 수 있다.
latest-policy score는 표시 / 분석 / preview 계산값일 뿐 저장 마이그레이션이 아니다.
```

제외 또는 blocked:

```text
simple record
weight_only record
snapshotless record
macro data insufficient record
impossible state record
raw backup / raw import 상태 그대로의 record
```

위 record는 latest-policy numeric preview를 계산하지 않거나 `unavailable` / `blocked`로 둔다.

## 3. terminology

stored fixed basis:

```text
기록에 저장된 adherencePercent와 adherenceScoringVersion의 원래 의미
backup / restore / Recent production path에서 유지되는 raw score basis
```

latest-policy preview basis:

```text
현재 scoring policy로 상세 기록을 다시 평가하는 non-persisted 분석 기준
test-local numeric helper 또는 preview-only object에서만 의미를 가진다
```

production score basis:

```text
getDailyAdherenceScore와 ADHERENCE_SCORING_VERSION이 정의하는 현재 저장 점수 기준
```

## 4. document correction map

`old fixed Records fixed basis`는 이제 다음처럼 읽는다.

```text
stored/raw fixed basis preserved
not migrated to range-basis storage
not overwritten by latest-policy preview
```

`old fixed Records no-reinterpret`는 이제 다음처럼 읽는다.

```text
no silent storage mutation
no backup/schema migration
no production Recent / DailyCoach / UI reuse
no production-facing candidateScorePreview / pointsPreview / scoreDeltaPreview field
```

다음처럼 읽으면 안 된다.

```text
old fixed Records are permanently excluded from latest-policy preview
detailed old Records can never be recalculated for analysis
Records detail display path can never compute a non-persisted latest-policy preview
```

## 5. helper entry implication

향후 test-local numeric helper가 열릴 경우 첫 eligibility 후보는 다음을 포함할 수 있다.

```text

detailed old fixed Record with sufficient macro data
fixed goalSnapshot present
fixedScoreContext present
candidate version = range_score_candidate_preview_test_local_v0
```

단, output은 production-facing field가 아니다.

허용 후보:

```text

testLocalComponentPoints
testLocalCandidateScorePreviewVersion
testLocalFixedScoreBasisLabel
testLocalCandidateScoreBasisLabel
testLocalReasons
testLocalBlockedReason
```

계속 금지:

```text
candidateScorePreview
pointsPreview
scoreDeltaPreview
record-level score candidate field
goalSnapshot score candidate field
backup payload score candidate field
Recent aggregation input
DailyCoach action signal
UI production score display
```
