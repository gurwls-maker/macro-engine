목적은 `testLocalNumericPreview`가 report-only output에 실제로 존재하더라도 production 소비자가 그 값을 점수, 저장값, Recent, UI, DailyCoach, Records state로 읽지 않는다는 경계를 테스트로 고정하는 것이다.

## 1. 구현 요약

추가:

```text
runMacroRangeScoreReportOnlyNumericPreviewConsumerNoLeakTests
```

등록:

```text
window.runMacroRangeScoreReportOnlyNumericPreviewConsumerNoLeakTests
npm run test:macro-policy suite 목록
tools/render_audit/run_internal_tests.cjs core profile
```

`tools/render_audit/run_internal_tests.cjs`의 `core` profile에도 등록한다. macro range score guard suite들은 core profile에 수동 등록되어 있으므로, 새 consumer no-leak suite도 같은 묶음에 들어가야 한다.

등록 점검 보강:

```text
runAuthoritativeTargetCalTests
runTargetCalSemanticGuardTests
runModeFreeCalculationContextTests
```

위 3개는 이전부터 `test:macro-policy`에 포함되어 있었지만 `tools/render_audit` core profile에는 없었다. 이번 registration audit에서 core profile에도 포함해 macro-policy / core coverage 차이를 줄였다.

suite 성격:

```text

before/after side-effect comparison
```

## 4. no-leak key guard

consumer payload에서 잡는 forbidden key:

```text
testLocalNumericPreview
testLocalCandidateScorePreview
testLocalComponentPoints
testLocalCandidateScorePreviewVersion
testLocalFixedScoreBasisLabel
testLocalCandidateScoreBasisLabel
testLocalBlockedReason
candidateScorePreview
pointsPreview
scoreDeltaPreview
rangeScoreCandidate
macroRangeContract
rangeScoreCandidateSummary
```

consumer text에서 잡는 forbidden text:

```text
testLocalNumericPreview
testLocalCandidateScorePreview
testLocalComponentPoints
candidateScorePreview
pointsPreview
scoreDeltaPreview
macro_range_candidate_preview_test_local
range_score_candidate_preview_test_local_v0
```

## 5. normalizer guard

leaky raw record와 leaky raw goalSnapshot에 위 key들을 일부러 넣는다.

확인:

```text
normalizeGoalSnapshot 결과에 forbidden key가 없음
normalizeRecord 결과에 forbidden key가 없음
normalizeRecordsCollection 결과에 forbidden key가 없음
fixed snapshot targetCal / protein은 살아남음
stored adherencePercent / adherenceScoringVersion은 살아남음
```

즉 unknown range/numeric field pruning은 range 지원 실패가 아니라 현재 fixed snapshot 계약이다.

## 6. backup / restore guard

source candidate는 `testLocalNumericPreview`를 가질 수 있다.

하지만 확인:

```text
buildFullBackupPayload 결과에 testLocalNumericPreview 없음
normalizeFullBackupPayload 결과에 testLocalNumericPreview 없음
backup string에 testLocalCandidateScorePreview / candidateScorePreview 없음
restored records는 fixed adherencePercent / adherenceScoringVersion 유지
```

즉 report-only candidate object가 존재해도 backup payload source가 되지 않는다.

## 7. Recent guard

leaky raw record와 clean normalized record를 비교한다.

확인:

```text
getRecentContextRecordSummary 결과가 clean fixed record summary와 동일
getRecentContextComponentCounts 결과가 동일
proteinLowDays / fatLowDays / carbLowTrainingDays가 nested numeric preview에 영향받지 않음
```

즉 Recent는 fixed target과 consumed macros 기준을 유지한다.

## 8. fixed score / DailyCoach / UI / Records guard

candidate를 생성하기 전후로 비교한다.

확인:

```text
getDailyAdherenceScore percent unchanged
scoringVersion unchanged
buildFullBackupPayload unchanged
getRecentContextComponentCounts unchanged
rendered body text unchanged
generateDailyCoachOutput unchanged
recordsState.items unchanged
ADHERENCE_SCORING_VERSION unchanged
```

그리고 output text에 test-local preview 이름이 새지 않는지 확인한다.

## 9. promotion reject

suite 내부 promoted payload를 만든다.

포함:

```text

goalSnapshot.candidateScorePreview
consumerPreview.rangeScoreCandidateSummary

consumerPreview.scoreDeltaPreview
```

```text
promoted consumer payload는 no-leak guard가 violation으로 잡는다.
```
