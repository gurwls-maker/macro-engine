# v8.2 macro range production score transition tests implementation (2026-07-06)

## 1. 목적

이 문서는 `runMacroRangeProductionScoreTransitionDecisionTests` test-only suite 구현 기록이다.

이번 구현의 목적은 v8.2 macro range 본류가 `scoreDeltaPreview` 비교 확장이 아니라 `v8.2_macro_range_score_v1` range-aware production score 전환임을 테스트 목록에서 고정하는 것이다.

이번 suite는 현재 앱 상수의 즉시 bump를 요구하지 않는다. `ADHERENCE_SCORING_VERSION`은 현재 `v6.1_alcohol_penalty_v2`로 유지되어야 하며, `v8.2_macro_range_score_v1`은 future production implementation target으로만 고정한다.

## 5. production score formula guard

suite는 future range-aware score formula 후보를 다음처럼 고정한다.

```text
productionScore = clamp(100 + sum(kcal/protein/fat/carbExchange contribution), 0, 100)
```

합산 대상:

```text
kcal
protein
fat
carbExchange
```

합산 제외:

```text
dataQuality
```

`dataQuality`는 score component가 아니라 eligibility gate이므로 numeric contribution으로 합산하지 않는다.

## 6. component contribution table

```text
kcal:
- matched: 0
- high / low: -10
- severe_high / severe_low: -16

protein:
- satisfied: 0
- low: -16
- severe_low: -24

fat:
- inside_recommended: 0
- below_recommended / above_recommended: -5
- below_action_floor / above_flexible: -12

carbExchange:
- exchange_explained / inside_candidate_range: 0
- below_training_support: -8
- above_candidate_range: -6

dataQuality:
- full_day_candidate: null
- blocked / unresolved states: null
```

`0`은 eligible neutral / explanation contribution이고, `null`은 blocked / unresolved / non-score eligibility state이다.

## 7. implemented test cases

이번 suite는 12 cases를 추가한다.

```text
1. production score transition inventory stays test-only
2. exact score fixtures match component formula
3. product behavior fixtures avoid fixed-target double punishment
4. blocked and unavailable records return null without sentinel score
5. invalid numeric inputs are rejected
6. future version bump contract stays explicit
7. getDailyAdherenceScore replacement remains future-only
8. new Records use forward-only v8.2 scoring contract
9. old fixed Records stay non-blocking without silent mutation
10. backup and restore payloads reject preview persistence
11. Today Records Recent and DailyCoach ignore preview fields
12. scoreDeltaPreview remains optional audit-only and non-blocking
```

## 8. exact score fixtures

```text
eligible_full_day_neutral:
[0, 0, 0, 0] -> raw 100 -> percent 100

kcal_high_fat_observation:
[-10, 0, -5, 0] -> raw 85 -> percent 85

all_major_penalties:
[-16, -24, -12, -8] -> raw 40 -> percent 40

protein_anchor_carb_high:
[-10, -16, 0, -6] -> raw 68 -> percent 68

severe_kcal_fat_carb:
[-16, 0, -5, -6] -> raw 73 -> percent 73

lower_clamp_defensive:
[-80, -40, -30, -10] -> raw -60 -> percent 0
```

These fixtures intentionally validate exact component values instead of global severity-label ordering.

## 9. product behavior fixtures

이번 suite는 v8.2의 실제 제품 방향에 맞는 behavior fixture도 포함한다.

```text
fat below_recommended + carbExchange exchange_explained:
fat -5, carbExchange 0 -> score 95
목적: carb/fat double punishment 방지

protein low:
protein -16 -> score 84
목적: protein anchor penalty 유지

fat below_action_floor + carbExchange exchange_explained:
fat -12, carbExchange 0 -> score 88
목적: action floor breach는 강한 range evidence penalty로 유지
```

이 case들은 fixed target g 비교에 끌려가던 과거 산식 문제를 반복하지 않도록 고정한다.

## 10. blocked/null policy

아래 상태는 production score를 만들지 않는다.

```text
one_meal_pending
insufficient_record
snapshotless_old_record
impossible_state
unsupported_range_basis
invalid_range_basis
range_policy_missing
```

expected output:

```text
percent: null
scoringVersion: null
sentinel 0: 금지
```

0점은 가능한 numeric score이고, 계산 불가 상태는 0점이 아니다.

## 11. invalid numeric input guard

다음 input은 invalid로 거부한다.

```text
positive component point
fractional component point
NaN
Infinity
string numeric value
dataQuality sentinel 0
missing source component
unknown component
```

positive input을 upper clamp로 흡수하거나, fractional input을 rounding으로 복구하지 않는다.

## 12. version / replacement guard

suite는 future version target을 고정하지만 current production version을 바꾸지 않는다.

```text
future target: v8.2_macro_range_score_v1
current code: v6.1_alcohol_penalty_v2
```

`getDailyAdherenceScore`도 아직 교체하지 않는다. suite 내부 helper는 future fixture score를 계산하지만, current `getDailyAdherenceScore` output과 version에는 side effect를 만들지 않아야 한다.

## 13. Records policy guard

new Records:

```text
adherencePercent: future range-aware percent
adherenceScoringVersion: v8.2_macro_range_score_v1
```

blocked new Records:

```text
adherencePercent: null
adherenceScoringVersion: null
```

old fixed Records:

```text
no silent mutation
old adherencePercent 유지
old adherenceScoringVersion 유지
production transition blocker로 부풀리지 않음
```

이는 실사용 데이터 보존 요구가 낮은 현재 제품 배경을 반영한다. old fixed score와의 장기 병행이 본류가 아니며, 새 Records forward-only 전환이 우선이다.

## 14. preview field no-leak guard

다음 field는 backup / Recent / UI / DailyCoach / Records production payload로 새면 안 된다.

```text
candidateScorePreview
pointsPreview
scoreDeltaPreview
rangeScoreCandidateSummary
testLocalCandidateScorePreview
testLocalComponentPoints
```

이번 suite는 promoted preview payload를 forbidden persistence로 감지한다.

## 15. scoreDeltaPreview position

`scoreDeltaPreview`는 이번 suite에서도 production prerequisite가 아니다.

```text
scoreDeltaPreviewRequired: false
scoreDeltaPreviewBlocking: false
```

production transition score는 fixed score context 없이 계산되어야 한다. fixed score context가 있어도 future range-aware score fixture 결과는 바뀌지 않는다.
