# v8.3 fixture direction table

작성일: 2026-07-07
변경 성격: docs-only fixture direction table
구현 여부: code implementation 없음
점수 숫자 확정 여부: 없음
용도: v8.3 continuous scoring test design의 선행 입력

## 1. 문서 목적

이 문서는 구현 지시서가 아니다.
이 문서는 점수 숫자표도 아니다.
이 문서는 v8.3 anchor-based continuous macro scoring을 구현하기 전에 대표 fixture별 점수 방향과 QA 판정 기준을 닫는 문서다.

닫는 것:
- 어떤 케이스가 거의 변화 없어야 하는지.
- 어떤 케이스가 더 낮아져야 하는지.
- 어떤 케이스가 null/blocked여야 하는지.
- 어떤 케이스가 hard zero면 실패인지.
- 어떤 케이스가 curve-mediated collapse로 낮아질 수 있는지.
- v8.2 fixed penalty table과 충돌하는 exact-score fixture를 어떻게 supersede할지.

닫지 않는 것:
- alcohol 56g은 몇 점 감점인지.
- fat upper 2배는 몇 점 감점인지.
- carb 450g은 몇 점인지.
- final penalty curve shape.
- ADHERENCE_SCORING_VERSION 변경.
- index.html scoring formula 구현.
- UI/storage/schema/backup/Recent/DailyCoach 변경.

v8.3 implementation은 이 문서와 v8.3 continuous scoring test design이 둘 다 닫힌 뒤에만 가능하다.

## 2. 공통 QA 원칙

### 2-1. monotonicity

같은 조건에서 섭취량 또는 리스크가 커졌는데 점수가 같거나 높아지면 실패다.

예:
- alcohol 80g은 alcohol 56g보다 같은 조건에서 더 낮거나 같아야 하며, 실제 fixture에서는 더 낮은 방향이어야 한다.
- fat upper 3.0x는 fat upper 2.0x보다 더 낮은 방향이어야 한다.
- totalKcal/TDEE 2.0+는 totalKcal/TDEE 1.5보다 더 낮은 방향이어야 한다.

단, monotonicity는 같은 축과 같은 맥락 안에서만 비교한다.
예를 들어 training context가 바뀐 carb fixture는 운동 맥락으로 curve anchor가 이동할 수 있으므로 REST fixture와 단순 동일축 비교만으로 판정하지 않는다.

### 2-2. near-zero tolerance

target +1kcal, fat upper +1g 같은 미세 이탈이 큰 감점이면 실패다.
v8.3의 목적은 "목표값을 1 단위라도 벗어나면 벌점"이 아니다.
목표와 범위는 anchor이며, 작은 이탈은 작은 변화로 처리되어야 한다.

### 2-3. no bucket plateau

56g, 80g, 112g, 140g 알코올이 같은 점수면 실패다.
fat upper 1.2x, 2.0x, 3.0x가 같은 점수면 실패다.
high/severe 같은 상태 라벨 안에서 점수가 멈추는 plateau도 실패다.

### 2-4. no hard cap

중간 score cap으로 점수를 자르지 않는다.
예를 들어 "알코올이 일정량 이상이면 최대 60점" 같은 정책은 또 다른 계단식이므로 실패다.
마지막 0~100 clamp만 표시 범위로 허용한다.

### 2-5. hard-collapse 금지, curve-mediated collapse 허용

특정 입력 하나가 threshold를 넘었다는 이유만으로 즉시 0점 처리하면 실패다.
하지만 여러 연속 penalty가 함께 커져 finalRawScore가 자연스럽게 0점 근처로 내려가는 것은 허용한다.

예:
- high training context에서 carb가 높다는 이유만으로 즉시 0점이면 실패.
- 같은 carb가 totalKcal/TDEE extreme overload와 결합되어 곡선상 낮은 점수가 되는 것은 허용.

### 2-6. null gate

insufficient, no meals, snapshotless, range missing은 0점이 아니라 null/blocked다.
sentinel 0은 금지한다.
0점은 계산 가능한 최저 점수이고, null은 계산할 수 없는 상태다.

### 2-7. no double-count confusion

alcohol kcal와 alcohol physiology risk는 다른 축이다.
alcohol kcal는 totalKcal와 TDEE overload에 반영된다.
alcoholG, standard drinks, bodyWeightKg, drinking intensity는 recovery/sleep/metabolic/behavior risk 축에 반영된다.
이것은 같은 현상을 두 번 벌하는 것이 아니라 서로 다른 risk channel을 분리하는 것이다.

### 2-8. no exercise bonus

운동은 finalScore bonus가 아니다.
운동 소모는 target/TDEE에 이미 반영된다.
운동 맥락은 carb curve anchor를 이동시키는 context일 뿐이다.

### 2-9. exact score 금지

이 문서의 fixture는 "몇 점이어야 한다"가 아니라 "어느 방향이어야 한다"를 닫는다.
exact numeric score는 다음 test design 또는 implementation decision에서도 외부근거 anchor와 curve shape가 닫힌 뒤에만 제한적으로 다룬다.

## 3. Fixture table

각 fixture는 다음 항목을 가진다.

- 입력 조건
- 기대 방향
- 실패 조건
- 관련 penalty 후보
- 외부 anchor 여부

### A. Energy fixtures

#### E01 target + 1kcal

입력 조건:
- totalKcal이 targetKcal보다 1kcal 높다.
- TDEE/maintenance 대비 overload는 없다.
- protein/fat/carb/alcohol은 모두 설명 가능한 범위다.

기대 방향:
- 점수 변화는 near-zero여야 한다.
- targetEnergyDeviationPenalty는 존재하더라도 극히 작아야 한다.

실패 조건:
- 큰 감점.
- high bucket 진입만으로 고정 감점.
- target 초과 1kcal를 policy violation처럼 취급.

관련 penalty 후보:
- targetEnergyDeviationPenalty.

외부 anchor 여부:
- 외부근거 직접 anchor 없음.
- 앱 목표 adherence anchor.

#### E02 target 대비 10% 초과, TDEE 이하

입력 조건:
- totalKcal이 targetKcal보다 약 10% 높다.
- totalKcal은 TDEE 이하 또는 근처다.
- alcohol 없음 또는 낮음.

기대 방향:
- E01보다 낮아야 한다.
- 하지만 catastrophic이면 안 된다.
- 목표 초과와 생리학적 과부하를 분리해야 한다.

실패 조건:
- TDEE 이하인데 extreme overload처럼 처리.
- target 초과만으로 hard collapse.

관련 penalty 후보:
- targetEnergyDeviationPenalty.
- tdeeOverloadPenalty는 없거나 낮아야 한다.

외부 anchor 여부:
- 앱 target adherence anchor.
- TDEE/maintenance 생산값 anchor.

#### E03 target 대비 30% 초과, TDEE 근처

입력 조건:
- totalKcal이 targetKcal보다 약 30% 높다.
- totalKcal은 TDEE 근처다.

기대 방향:
- E02보다 낮아야 한다.
- 그래도 TDEE extreme overload와 같은 취급은 하지 않는다.
- 다이어트 목표일수록 targetEnergyDeviationPenalty가 더 명확해질 수 있다.

실패 조건:
- E02와 같은 점수 plateau.
- TDEE overload fixture와 구분하지 못함.

관련 penalty 후보:
- targetEnergyDeviationPenalty.
- goalMode-sensitive interpretation.

외부 anchor 여부:
- 앱 target/TDEE anchor.

#### E04 totalKcal / TDEE 1.5

입력 조건:
- totalKcal이 TDEE의 약 1.5배다.
- 목표 대비 초과도 크다.

기대 방향:
- E03보다 낮아야 한다.
- tdeeOverloadPenalty가 본격적으로 커져야 한다.
- 그러나 중간 score cap으로 자르면 안 된다.

실패 조건:
- E03과 비슷한 점수.
- "TDEE 1.5배 이상이면 최대 N점" 같은 hard cap.

관련 penalty 후보:
- targetEnergyDeviationPenalty.
- tdeeOverloadPenalty.
- dataOutlierPenalty는 일반적으로 아직 primary가 아니다.

외부 anchor 여부:
- TDEE/maintenance anchor.

#### E05 totalKcal / TDEE 2.0+

입력 조건:
- totalKcal이 TDEE의 2배 이상이다.
- macro distribution이 무엇이든 energy overload가 매우 크다.

기대 방향:
- E04보다 낮아야 한다.
- curve-mediated collapse로 0점 근처에 갈 수 있다.
- 즉시 hard zero가 아니라 연속 penalty 누적으로 낮아져야 한다.

실패 조건:
- E04와 같은 점수 plateau.
- hard zero threshold primary policy.
- 총열량 과부하를 macro exchange로 면제.

관련 penalty 후보:
- tdeeOverloadPenalty.
- targetEnergyDeviationPenalty.
- dataOutlierPenalty 후보.

외부 anchor 여부:
- TDEE/maintenance anchor.

#### E06 target보다 낮지만 protein도 낮은 감량일

입력 조건:
- totalKcal은 target 이하 또는 근처다.
- protein이 shortage 상태다.
- alcohol 없음.

기대 방향:
- 낮은 칼로리만으로 좋은 점수가 되면 안 된다.
- proteinShortagePenalty가 강하게 반영되어야 한다.

실패 조건:
- target 이하라는 이유로 단백질 부족을 무시.
- 단백질 부족인데 neutral 처리.

관련 penalty 후보:
- proteinShortagePenalty.
- targetEnergyDeviationPenalty는 낮거나 goalMode에 따라 다름.

외부 anchor 여부:
- ISSN protein anchor.

### B. Alcohol fixtures

#### A01 alcohol 0g

입력 조건:
- alcoholG 0.
- totalKcal/TDEE 정상.
- macro distribution 정상.

기대 방향:
- alcoholPhysiologyPenalty는 0 또는 없음.
- alcohol 관련 warning/action은 없어야 한다.

실패 조건:
- alcohol이 없는데 alcohol physiology risk 부여.

관련 penalty 후보:
- alcoholPhysiologyPenalty 없음.

외부 anchor 여부:
- NIAAA zero baseline.

#### A02 alcohol 14g

입력 조건:
- pure alcohol 약 14g.
- totalKcal/TDEE 정상.
- 다른 macro 정상.

기대 방향:
- A01보다 약간 낮아질 수 있다.
- 큰 붕괴는 실패다.
- standard drink anchor로 곡선 시작점을 확인한다.

실패 조건:
- 14g을 폭음과 같은 취급.
- alcohol kcal만 반영하고 physiology risk 축을 완전히 무시.

관련 penalty 후보:
- alcoholPhysiologyPenalty 낮음.
- alcohol kcal가 totalKcal에 포함되므로 targetEnergyDeviationPenalty 또는 tdeeOverloadPenalty에 아주 작게 영향 가능.

외부 anchor 여부:
- NIAAA standard drink 14g.

#### A03 alcohol 56g

입력 조건:
- pure alcohol 약 56g.
- bodyWeightKg와 drinking duration 정보가 있으면 standard drinks/intensity 계산 가능.
- totalKcal/TDEE 정상 또는 약간 초과.

기대 방향:
- A02보다 낮아야 한다.
- binge-risk anchor 근처로 곡선 기울기가 커질 수 있다.
- hard cutoff가 아니라 anchor다.

실패 조건:
- A02와 같은 점수.
- 즉시 score cap 또는 hard zero.
- v6.1 후처리 penalty를 그대로 부활.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- alcohol kcal로 인한 energy penalties.

외부 anchor 여부:
- NIAAA binge drinking anchor.

#### A04 alcohol 80g

입력 조건:
- pure alcohol 약 80g.
- totalKcal/TDEE는 fixture별로 통제.

기대 방향:
- A03보다 낮아야 한다.
- A03과 같은 bucket이면 실패다.

실패 조건:
- 56g, 80g을 같은 점수로 처리.
- score cap으로 차이를 지움.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- tdeeOverloadPenalty if alcohol kcal pushes totalKcal up.

외부 anchor 여부:
- NIAAA binge/high-intensity 사이의 curve point.

#### A05 alcohol 112g

입력 조건:
- pure alcohol 약 112g.
- binge threshold 약 2배에 가까운 high-intensity anchor 후보.

기대 방향:
- A04보다 낮아야 한다.
- physiology risk가 명확히 커져야 한다.
- totalKcal/TDEE도 함께 초과하면 더 낮아질 수 있다.

실패 조건:
- A04와 같은 점수.
- hard zero threshold.
- alcohol kcal만 보고 physiology risk를 무시.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- tdeeOverloadPenalty.
- dataOutlierPenalty는 입력 맥락에 따라 후보.

외부 anchor 여부:
- NIAAA high-intensity drinking anchor.

#### A06 alcohol 140g

입력 조건:
- pure alcohol 약 140g.
- 매우 높은 alcohol intake.

기대 방향:
- A05보다 낮아야 한다.
- curve-mediated collapse 가능성이 커진다.
- 하지만 "140g이면 무조건 0" 같은 hard-collapse는 피한다.

실패 조건:
- A03/A04/A05/A06 모두 같은 점수.
- score cap 때문에 더 마셔도 점수가 내려가지 않음.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- tdeeOverloadPenalty.
- dataOutlierPenalty 후보.

외부 anchor 여부:
- NIAAA high-intensity drinking anchor beyond 2x binge.

#### A07 alcohol kcal 때문에 totalKcal도 TDEE 초과

입력 조건:
- alcoholG가 높다.
- alcohol kcal를 포함한 totalKcal/TDEE가 overload다.

기대 방향:
- alcohol physiology와 energy overload가 둘 다 반영되어야 한다.
- 이것은 double-count가 아니라 energy axis + physiology risk axis 분리다.

실패 조건:
- alcohol kcal만 반영하고 alcohol physiology risk 무시.
- alcohol physiology만 반영하고 totalKcal overload 무시.
- 두 축이 있다는 이유로 어느 한 축을 강제로 꺼버림.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- tdeeOverloadPenalty.
- targetEnergyDeviationPenalty.

외부 anchor 여부:
- NIAAA alcohol anchor.
- TDEE/maintenance anchor.

#### A08 alcohol 극단 입력 / outlier 후보

입력 조건:
- alcoholG가 현실적으로 매우 의심스러운 수준이다.
- 예: 입력 실수 가능성이 큰 숫자.

기대 방향:
- 계산 가능한 극단과 데이터 오류 후보를 구분해야 한다.
- 계산 가능하면 curve-mediated low score.
- 데이터 오류급이면 dataOutlier/invalid guard로 점수 산출 자체를 막을 수 있다.

실패 조건:
- 모든 극단 입력을 score cap으로 처리.
- invalid guard를 score policy처럼 사용.

관련 penalty 후보:
- alcoholPhysiologyPenalty.
- tdeeOverloadPenalty.
- dataOutlierPenalty.

외부 anchor 여부:
- NIAAA anchor + app invalid/outlier policy.

### C. Fat fixtures

#### F01 fat upper + 1g

입력 조건:
- fatG가 targetFatUpper보다 1g 높다.
- totalKcal/TDEE 정상.
- protein/carb/alcohol 정상.

기대 방향:
- near-zero change.
- 큰 감점이면 실패.

실패 조건:
- upper를 1g 넘었다는 이유로 큰 penalty.
- above bucket 고정 감점.

관련 penalty 후보:
- fatRangePenalty 아주 낮음.

외부 anchor 여부:
- AMDR/DRI range anchor.

#### F02 fat upper 1.2x

입력 조건:
- fatG가 targetFatUpper의 약 1.2배다.
- totalKcal/TDEE는 정상 또는 약간 초과.

기대 방향:
- F01보다 낮아야 한다.
- 아직 catastrophic은 아니다.

실패 조건:
- F01과 같은 점수.
- F03/F04와 같은 plateau.

관련 penalty 후보:
- fatRangePenalty.
- targetEnergyDeviationPenalty if totalKcal moves.

외부 anchor 여부:
- AMDR/DRI.

#### F03 fat upper 2.0x

입력 조건:
- fatG가 targetFatUpper의 약 2배다.
- totalKcal/TDEE도 올라갈 가능성이 크다.

기대 방향:
- F02보다 낮아야 한다.
- fatRangePenalty와 tdeeOverloadPenalty가 모두 커질 수 있다.

실패 조건:
- F02와 같은 점수.
- fat above_flexible 고정값에서 멈춤.

관련 penalty 후보:
- fatRangePenalty.
- tdeeOverloadPenalty.

외부 anchor 여부:
- AMDR/DRI + TDEE.

#### F04 fat upper 3.0x

입력 조건:
- fatG가 targetFatUpper의 약 3배다.
- totalKcal/TDEE overload 가능성이 크다.

기대 방향:
- F03보다 낮아야 한다.
- energy overload와 결합되면 매우 낮은 방향 가능.

실패 조건:
- F02/F03/F04 같은 plateau.
- 중간 score cap.

관련 penalty 후보:
- fatRangePenalty.
- tdeeOverloadPenalty.
- dataOutlierPenalty 후보.

외부 anchor 여부:
- AMDR/DRI + TDEE.

#### F05 fat 1000g

입력 조건:
- fatG 1000g.
- fat kcal만으로도 매우 큰 totalKcal가 된다.

## 4. Superseded exact-score fixture map

이 섹션은 v8.2 exact-score fixture를 v8.3에서 어떻게 대체하는지 정리한다.
아래 항목은 모두 SUPERSEDE 또는 폐기 맥락이다.

- SUPERSEDE: high = -10 fixed penalty table as production body.
- SUPERSEDE: severe = -16 fixed penalty table as production body.
- SUPERSEDE: candidateScorePreview = clamp(100 + fixed component point sum, 0, 100) as final policy.
- SUPERSEDE: component pointsPreview exact integer table as production score body.
- SUPERSEDE: scoreDeltaPreview comparison line as mainline scoring direction.

v8.3 대체 기준:
- exact score fixture가 아니라 direction fixture로 대체한다.
- monotonicity를 본다.
- near-zero tolerance를 본다.
- no bucket plateau를 본다.
