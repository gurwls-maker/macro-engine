# v8.3 continuous scoring implementation

작성일: 2026-07-07
변경 성격: production scoring implementation
구현 범위: minimal surface, complete model

## 1. 구현 요약

이번 구현은 v8.3 macro scoring을 실제 production score path로 전환한다.

변경된 production identity:
- `MACRO_RANGE_PRODUCTION_SCORING_VERSION = "v8.3_anchor_continuous_macro_score_v1"`
- `ADHERENCE_SCORING_VERSION = MACRO_RANGE_PRODUCTION_SCORING_VERSION`
- `MACRO_RANGE_SCORE_FORMULA_NAME = "anchor_continuous_macro_score_v1"`

유지한 public/storage surface:
- `getDailyAdherenceScore(...)` entry 유지.
- `getMacroRangeProductionScoreSummary(...)` entry 유지.
- Records 저장 계약은 `adherencePercent` / `adherenceScoringVersion` 유지.
- blocked/null record는 `adherencePercent: null`, `adherenceScoringVersion: null` 유지.
- current v8.3 auto score가 저장된 기록은 조회 시 저장 `adherencePercent`를 우선한다.
- 식사 변경, Today live score, 수동 기준 갱신처럼 명시적 재계산 이벤트가 있을 때만 v8.3 formula를 다시 실행한다.
- old fixed Records no silent mutation 유지.

새로 추가한 runtime detail:
- `detail.penaltyBreakdown`
- `detail.scoringContext`

이 detail은 formula QA와 transient runtime detail이다.
record/backup schema 확장이 아니다.

## 2. 산식 본체

v8.3 production score는 아래 구조를 따른다.

```text
rawScore =
  100
  - targetEnergyDeviationPenalty
  - tdeeOverloadPenalty
  - proteinShortagePenalty
  - proteinExcessEfficiencyPenalty
  - fatRangePenalty
  - carbExerciseContextPenalty
  - carbFatExchangeFailurePenalty
  - alcoholPhysiologyPenalty
  - dataOutlierPenalty

percent = clamp(rawScore, 0, 100)
```

중요:
- 각 penalty는 상태별 고정 감점표가 아니다.
- 마지막 clamp만 표시 범위로 남긴다.
- rawScore와 penaltyBreakdown은 final clamp 이후 plateau 여부를 검사할 수 있게 남긴다.
- `componentContributions`는 source of truth가 아니라 compatibility aggregate다.

## 3. 축별 구현

### Energy

구현:
- `targetEnergyDeviationPenalty`
- `tdeeOverloadPenalty`

의미:
- target 대비 1kcal 같은 미세 이탈은 near-zero다.
- target 초과와 TDEE 초과는 분리한다.
- TDEE 1.5배, 2.0배, fat 1000g 같은 상황은 TDEE 축에서 연속적으로 더 낮아진다.

### Protein

구현:
- `proteinShortagePenalty`
- `proteinExcessEfficiencyPenalty`

의미:
- 부족은 강하게 반영한다.
- 초과는 효율 저하/배분 문제로 완만하게 반영한다.
- 단백질 목표 달성을 보너스로 더하지 않는다.

### Fat

구현:
- `fatRangePenalty`

의미:
- fat upper +1g은 near-zero다.
- fat upper 1.2x, 2.0x, 3.0x, 1000g은 같은 bucket에서 멈추지 않는다.
- display percent가 0에서 같아질 수는 있지만, rawScore와 fatRangePenalty는 계속 더 나빠진다.

### Carb / exercise

구현:
- `carbExerciseContextPenalty`
- `carbFatExchangeFailurePenalty`

의미:
- 운동은 final score bonus가 아니다.
- 운동 맥락은 carb curve anchor를 이동시키는 context다.
- 같은 450g carb라도 rest / normal resistance / high-volume context의 carb penalty가 달라질 수 있다.
- high-volume context라도 TDEE overload가 있으면 낮은 점수가 가능하다.

### Alcohol

구현:
- alcohol kcal는 totalKcal / target energy / TDEE overload 축에 반영.
- `alcoholG = alcoholKcal / 7`
- `standardDrinks = alcoholG / 14`
- `alcoholPhysiologyPenalty`는 standard drinks와 body-weight pressure로 계산.

의미:
- alcohol을 kcal만으로 보지 않는다.
- alcohol physiology risk를 무시하지 않는다.
- v6.1 alcoholImpactPenalty 후처리 감산은 되살리지 않는다.
- `analyzeAlcoholImpact(...)`의 legacy `penaltyPoints`는 DailyCoach signal용으로 남지만 production score body가 아니다.

## 5. 테스트 갱신

추가:

- `package.json` `test:macro-policy` 등록.
- `tools/render_audit/run_internal_tests.cjs` core profile 등록.

갱신:
- `runMacroRangeProductionScoreImplementationDecisionTests`
- `runMacroRangeProductionScoreTransitionDecisionTests`
- `runAdherenceScoringTests`
- mode golden test는 v8.3에서 의도적으로 바뀐 `adherencePercent`만 exact golden 비교에서 제외하고, BMR/TDEE/target/macro/snapshot fields는 계속 고정한다.

새 suite가 검증하는 것:
- v8.3 version / formula identity.
- penaltyBreakdown source field.
- target +1kcal near-zero.
- target deviation / TDEE overload monotonicity.
- alcohol kcal + physiology risk axis separation.
- alcohol no bucket plateau.
- fat upper +1g near-zero.
- fat excess no fixed plateau.
- fat 1000g curve-mediated collapse.
- carb exercise context modifier.
- no exercise bonus.
- invalid/missing scoring context null, not sentinel 0.

추가 회귀 보정:
- `test:full`에서 Today live basis 변경 후 저장 기록 점수가 drift되는 케이스가 발견됐다.
- 원인은 v8.3 formula가 saved `goalSnapshot` target은 우선했지만, `totalBurn` / exercise context 일부를 live `currentResult`에서 다시 읽을 수 있었기 때문이다.
- 이번 구현은 저장 기록 조회와 명시적 재계산을 분리한다.
- 옵션 없이 저장 기록을 조회하고, record가 current `ADHERENCE_SCORING_VERSION` auto score를 갖고 있으면 stored `adherencePercent`를 반환한다.
- `options.goalSnapshot`을 넘기거나 `preferSnapshot:false`인 Today live score는 계속 v8.3 formula로 재계산한다.
