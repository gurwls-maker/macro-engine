## 2. 구현 범위

추가한 production 비연결 helper:

```text
buildMacroRangeReportOnlyInput()
buildMacroRangeReportOnlyHelper()
getMacroRangeReportOnlyFixedTargetSummary()
buildMacroRangeReportOnlyProteinAnchor()
buildMacroRangeReportOnlyFatRange()
buildMacroRangeReportOnlyCarbRange()
buildMacroRangeReportOnlyImpossibleState()
buildMacroRangeReportOnlyExchangeValidity()
buildMacroRangeReportOnlyComparison()
```

이 helper들은 current fixed target과 range candidate를 같은 입력으로 비교하는 transient debug/report-only object를 만든다.

중요:

```text
helper는 calculate() 결과를 바꾸지 않는다.
helper는 getDailyAdherenceScore() 결과를 바꾸지 않는다.
helper output은 Records / goalSnapshot / backup / Recent / UI에 연결되지 않는다.
```

## 3. output 계약

helper output은 다음 meta를 가진다.

```text
reportOnly: true
uiVisible: false
persisted: false
appliesToSavedRecord: false
productionFormulaChanged: false
scoreVersionChanged: false
snapshotSchemaChanged: false
model: macro_range_report_only_helper_v0
generatedAtStage: v8_2_report_only_helper_v0
```

`macroRangeContract`는 아직 저장 schema가 아니다.

이번 구현은 `macroRangeContract` top-level field를 만들지 않는다. range 후보는 `rangeCandidateSummary` 안의 debug output으로만 존재한다.

## 3-1. 2026-07-03 sample matrix 보정

후속 감사에서 지적된 A- 항목을 반영했다.

보정:

```text
fat status에 below_recommended 추가
sample matrix를 최소 6행에서 10행으로 확대
낮은 targetCal 조건에서 fat floor / carb training minimum 충돌과 fat recommended max로 인한 negative carbMin 후보 테스트 추가
```

fat status 의미:

| status | 의미 |
| --- | --- |
| `below_action_floor` | action floor 아래라 하한 미달 상태다. |
| `below_recommended` | action floor는 넘었지만 recommendedMin 아래다. |
| `inside_recommended` | recommended range 안이다. |
| `above_recommended` | recommendedMax 위이지만 flexibleMax 이하다. |
| `above_flexible` | flexibleMax 위다. |
| `unresolved` | policy 숫자가 부족하거나 입력이 부족해 상태를 확정하지 않는다. |

중요:

```text
above_recommended는 fixed target의 남은 지방 압박을 줄인다는 뜻이 아니다.
fixed target remaining이 충분히 작으면 rangeReducesFatRemainingPressure는 false가 맞다.
```

## 4. 숫자 정책 취급

이번 구현은 앱 정책 숫자를 확정하지 않는다.

원칙:

```text
rangePolicyCandidate가 입력으로 준 숫자만 candidate 계산에 사용한다.
숫자가 없으면 null / unresolved / needsDecision으로 남긴다.
helper가 fat floor, carb minimum, protein preset을 임의로 만들지 않는다.
```

따라서 다음 값들은 아직 확정되지 않았다.

```text
protein mode
BW / FFM / adjusted weight 기준
fat physiological floor
fat action floor
fat recommended range
fat score threshold
fat coach threshold
carb training support minimum
carb penalty threshold
range-aware score weight
```

## 5. impossible state

이번 helper가 report-only로 구분하는 상태:

```text
protein_anchor_consumes_targetCal
fat_floor_breaks_carb_training_minimum
fat_recommended_max_makes_carb_min_negative
insufficient_policy_numbers
insufficient_data_quality
```

이 값들은 production warning이 아니다.

```text
테스트와 감사가 읽는 debug state
UI 노출 없음
Coach action 아님
Records 저장값 아님
```

## 6. 테스트

추가 suite:

```text
runMacroRangeReportOnlyHelperTests
```

검증 항목:

1. helper output이 report-only / non-persisted meta를 갖는다.
2. fixed fat remaining pressure와 range exchange candidate를 나란히 비교한다.
3. fat action floor / recommended range / flexible ceiling 상태를 분리한다.
4. policy 숫자가 없으면 unresolved / needsDecision으로 남긴다.
5. protein anchor가 targetCal을 거의 소진하는 impossible state를 보고한다.
6. fat floor와 carb training minimum 충돌, fat recommended max로 인한 negative carbMin 후보를 보고한다.
7. maintain / diet / cut / recomp / lean_bulk / bulk + one-meal pending / snapshotless / fat high + carb low / protein low + kcal matched sample matrix가 report-only로만 생성된다.
8. helper 호출 전후 production target, fixed score, goalSnapshot signature, backup payload, Recent component counts, DOM text, `ADHERENCE_SCORING_VERSION`이 변하지 않는다.

해석 주의:

```text
helper.guards는 output meta에 가까운 선언값이다.
실제 무영향 보증은 runMacroRangeReportOnlyHelperTests의 전후 비교 assertion이 담당한다.
```

등록:

```text

```
