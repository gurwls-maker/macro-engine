## 5. formula definition

후보 formula:

```text
scoreComponentPoints = [
  componentPreviews.kcal.pointsPreview,
  componentPreviews.protein.pointsPreview,
  componentPreviews.fat.pointsPreview,
  componentPreviews.carbExchange.pointsPreview
]

rawCandidateScorePreview = 100 + sum(scoreComponentPoints)
candidateScorePreview = clamp(rawCandidateScorePreview, 0, 100)
```

value domain:

```text
integer 0..100
null
```

예시:

```text
[0, 0, 0, 0] -> 100
[-10, 0, -5, 0] -> 85
[-16, -24, -12, -8] -> 40
[-80, -40, -30, -10] -> 0
```

중요:

```text
100은 positive bonus가 아니다.
100은 base score에서 penalty contribution이 없는 neutral candidate total이다.
component pointsPreview는 여전히 <= 0이어야 한다.
```

## 6. clamp decision

clamp 순서:

```text
1. component pointsPreview shape 검증
2. score component pointsPreview 합산
3. rawCandidateScorePreview 산출
4. clamp(rawCandidateScorePreview, 0, 100)
```

lower clamp:

```text
rawCandidateScorePreview < 0이면 0
```

upper clamp:

```text
rawCandidateScorePreview > 100이면 100
```

단:

```text
positive component pointsPreview가 있어 rawCandidateScorePreview > 100이 된 경우는 clamp로 조용히 흡수하지 않는다.
positive component pointsPreview는 invalid payload로 reject한다.
upper clamp는 future-proof defensive boundary일 뿐 positive bonus 허용이 아니다.
```

## 7. rounding decision

rounding policy:

```text
identity / no-op integer
```

이유:

```text
component pointsPreview formula v0는 integer-only다.
base score 100도 integer다.
integer sum 결과는 이미 integer다.
따라서 정상 입력에서는 Math.round / floor / ceil이 필요 없다.
```

fractional input 처리:

```text
fractional component pointsPreview는 invalid다.
fractional raw total은 정상 경로에서 발생하면 안 된다.
fractional candidateScorePreview를 rounding해서 저장하거나 표시하지 않는다.
```

결정:

```text
rounding은 invalid fractional input을 허용하기 위한 보정 장치가 아니다.
fractional input은 candidateScorePreview null / violation으로 막는다.
```
