v8 운동 프로필 / 목표 산식 / 시나리오 검증 정밀 설계

작성일: 2026-06-02
상태: v8 진입 기준 초안
적용 범위: v7.x UI 안정화 이후의 산식, 사용자 유형, 운동 프로필, 시나리오 검증

────────────────────────
1. v8에서 풀 문제
────────────────────────

v7.x까지는 Today / Records / 최근 흐름 / InBody / Settings의 UI와 데이터 책임을 정리했다.
v8부터는 앱이 “운동하는 사람에게도 믿을 만한 목표 칼로리와 탄단지”를 주는 쪽으로 산식과 프로필을 다시 정밀화한다.

현재 구조는 유지 칼로리를 대략 다음 축으로 만든다.

- 기초대사량
- 활동량
- 업무 강도 보정
- 주간 웨이트일수 / 오늘 웨이트 세션
- 유산소
- 목표별 감량 / 증량 방향
- InBody 또는 직접 입력 체성분
- 고급 설정

문제는 운동 수준과 사용자 유형을 주간 웨이트일수만으로 간접 판단하면 틀릴 수 있다는 점이다.
예를 들어 체성분이 아주 좋은 사람이 주 2회만 운동하면 초급자로 오판될 수 있고, 반대로 체성분은 낮지만 빈도만 높은 사람을 고급자로 볼 수도 있다.
v8은 운동 빈도, 체성분, 수행 능력, 운동 목적을 분리해서 본다.

────────────────────────
2. 최상위 원칙
────────────────────────

1. 유지 칼로리와 목표 칼로리를 분리한다.
   - 유지 칼로리는 몸 상태와 생활/운동 조건을 계산한 값이다.
   - 목표 칼로리는 유지 칼로리에 목표별 변화율을 적용한 값이다.

2. 목표별 변화율은 고정값 하나가 아니라 범위로 둔다.
   - 다이어트 / 커팅 / 린매스업 / 벌크업 / 유지 각각의 권장 속도 범위를 설계한다.
   - 범위 안에서 어디를 고를지는 체지방률, 근육량, 운동 수준, 최근 변화, 기록 신뢰도로 결정한다.

3. 자동 조정은 급하게 하지 않는다.
   - 14일은 관찰과 경고에 가깝다.
   - 28일은 조정 제안 기준으로 둔다.
   - 앱이 임의로 설정을 바꾸지 않고, “제안 → 사용자 확인 → 반영” 흐름을 유지한다.

4. 안전선은 실제 생리학적 값에 둔다.
   - Energy availability 같은 안전선은 실제 제지방량을 기준으로 한다.
   - 계산용 보정 LBM은 목표 보정에 쓸 수 있지만, 생리학적 안전선 자체를 낮추는 데 쓰지 않는다.

5. 사용자가 이해할 수 있어야 한다.
   - “전문가 보정”, “유효 반영률” 같은 내부식 표현은 기본 화면에 직접 노출하지 않는다.
   - 고급 설정은 열어두되, 왜 바꾸는지 / 바꾸면 무엇이 달라지는지 설명해야 한다.

────────────────────────
3. 사용자 유형 판단 축
────────────────────────

v8은 한 가지 축으로 사람을 분류하지 않는다.
아래 축을 각각 계산하고, 최종 산식에서는 조합해서 사용한다.

1. 체지방 상태
   - 저체지방
   - 보통
   - 과지방
   - 비만
   - 고도비만

2. 근육량 / 제지방 상태
   - 키, 체중, 골격근량, 체지방률을 함께 본다.
   - 필요하면 FFMI 또는 FFMI 유사 지표를 사용한다.
   - 골격근량만 보지 않고 체지방률과 함께 본다.

3. 운동 빈도
   - 주간 웨이트일수
   - 세션 길이
   - 오늘 훈련 여부
   - 유산소 빈도와 강도

4. 운동 수행 능력
   - 기본값은 보디빌딩 / 근비대형 운동 기준으로 둔다.
   - 필요하면 사용자가 운동 성향을 고른다.
   - 보디빌딩: 8~12회 반복 기준 주요 리프트 또는 대표 운동 수행 수준
   - 스트렝스 / 파워리프팅: 1RM 또는 추정 1RM 기준
   - 파워빌딩: 근비대와 중량 지표를 함께 사용
   - 러닝: 5km / 10km 기록 또는 주간 러닝량
   - 크로스핏 / 혼합 운동: 주간 세션, WOD 강도, 근력/유산소 혼합도를 보수적으로 반영

5. 기록 신뢰도
   - InBody 기록 수와 간격
   - 가정용 InBody 여부
   - 직접 입력 여부
   - 최근 체중 기록 수
   - 최근 식사 기록 수

────────────────────────
4. 운동 프로필 설계
────────────────────────

기본 프로필은 보디빌딩으로 둔다.
이 앱의 출발점이 탄단지, 린매스업, 벌크업, 커팅 같은 체성분 목표에 가깝기 때문이다.

다만 다른 운동 사용자가 앱을 열었을 때 “대충 만든 운동 분류”처럼 느끼면 안 된다.
따라서 각 프로필은 이름만 바꾸지 말고 세션 구조와 계산 책임도 바꿔야 한다.

프로필 후보:

1. 보디빌딩
   - 기본값.
   - PUSH / PULL / LEGS / UPPER / LOWER / REST 같은 현재 루틴 구조와 가장 잘 맞는다.
   - 목표는 근육량 유지/증가와 체지방 조절이다.

2. 파워빌딩
   - 근비대와 중량 향상을 함께 본다.
   - 주간 웨이트일수와 세션 강도 외에 주요 리프트 수행 능력을 더 중요하게 본다.

3. 스트렝스 / 파워리프팅
   - 1RM 또는 추정 1RM, 훈련 강도, 피로 관리가 중요하다.
   - 체중 증가/감량 목표가 있더라도 성능 저하 위험을 함께 봐야 한다.

4. 러닝 / 지구력
   - 웨이트 세션보다 러닝 거리, 페이스, 주간 총량, 회복이 중요하다.
   - 유산소 입력을 재사용하되, 단순 보조 유산소가 아니라 주 운동으로 취급해야 한다.

5. 크로스핏 / 혼합형
   - 고강도 혼합 운동이므로 단순 웨이트 MET나 단순 유산소 MET로만 보기 어렵다.
   - 초기에는 보수적 보정과 사용자 확인을 강하게 둔다.

────────────────────────
5. 목표별 산식 방향
────────────────────────

목표 후보:

- 유지
- 다이어트
- 커팅
- 린매스업
- 벌크업

다이어트와 커팅은 둘 다 감량이지만 성격이 다르다.

다이어트:
- 일반 감량.
- 체지방률이 높거나 운동 수준이 낮아도 쓸 수 있다.
- 보수적인 단백질 확보와 과도한 저칼로리 방지가 중요하다.

커팅:
- 운동 수행과 근육 보존을 전제로 한 감량.
- 체지방률이 이미 낮거나 운동 수행이 높은 사용자에게 더 적합하다.
- 감량 속도는 더 보수적으로 둔다.

린매스업:
- 체지방 증가를 억제하면서 느리게 증량한다.
- 체성분 기록과 주간 체중 변화가 중요하다.

벌크업:
- 증량을 허용하되 체지방 증가가 지나치면 조정한다.
- 운동 수행과 체중 변화가 함께 봐야 한다.

목표별 변화율은 고정값이 아니라 범위로 둔다.
예: 주당 체중 변화율 범위, 또는 월간 변화율 범위.
범위의 중앙값을 무조건 쓰지 않고, 아래 조건으로 위치를 고른다.

- 체지방률이 높으면 감량 목표에서 조금 더 공격적으로 갈 수 있다.
- 체지방률이 낮으면 감량 목표에서 보수적으로 간다.
- 근육량이 높고 운동 수행이 높으면 커팅/린매스업에서 보수적으로 간다.
- 기록 신뢰도가 낮으면 큰 조정을 하지 않는다.
- 최근 28일 변화가 목표와 어긋나면 조정 제안을 만든다.

────────────────────────
6. 진행 상황 반영
────────────────────────

진행 상황은 즉시 산식에 자동 반영하지 않는다.
기본 흐름은 다음과 같다.

1. 최근 14일
   - 데이터가 충분하면 흐름을 말한다.
   - 목표와 크게 어긋나는 경우 경고할 수 있다.
   - 기본 설정을 자동 변경하지 않는다.

2. 최근 28일
   - 데이터가 충분하면 목표 속도와 실제 변화 속도를 비교한다.
   - 목표 범위를 벗어나면 조정 제안을 만든다.
   - 제안은 사용자가 승인해야 반영된다.

3. 기록이 부족한 경우
   - 조정하지 않는다.
   - 기록 보완 행동을 우선 안내한다.

────────────────────────
7. 시나리오 검증 기준
────────────────────────

v8 산식 변경은 대표값 몇 개만 보고 통과시키지 않는다.
사용자가 말한 “모든 유형의 사람이 직접 쓴다고 생각하는 검증”에 맞춰 아래처럼 검증한다.

필수 시나리오 축:

- 성별
- 키
- 체중
- 체지방률
- 골격근량
- FFMI 또는 근육량 상태
- 목표: 유지 / 다이어트 / 커팅 / 린매스업 / 벌크업
- 활동량
- 업무 강도
- 주간 웨이트일수
- 운동 프로필
- 운동 수행 능력
- 오늘 세션
- 웨이트 시간
- 훈련 강도
- 유산소 시간 / 속도 / 경사
- InBody 신뢰도
- 최근 14일 / 28일 체중 변화
- 최근 식사 기록 품질

검증 출력은 반드시 아래를 포함한다.

1. 유지 칼로리 분해
   - BMR
   - 활동 보정
   - 업무 보정
   - 주간 운동 보정
   - 오늘 운동 보정
   - 유산소

2. 목표 칼로리 분해
   - 목표별 변화율 범위
   - 선택된 변화율
   - 선택 이유
   - 안전선 적용 여부

3. 탄단지 분해
   - 단백질 기준
   - 지방 하한/상한
   - 탄수 배정
   - 목표별 조정 이유

4. 경고 목록
   - 너무 낮은 칼로리
   - Energy availability 위험
   - 단백질 과소/과다
   - 지방 과소/과다
   - 탄수 과소
   - 운동량 대비 회복 부족
   - 기록 신뢰도 부족

5. 사람 눈으로 볼 대표 케이스
   - 마른데 근육 적은 사람
   - 마른데 근육 많은 사람
   - 일반 체형
   - 과체중이지만 운동하는 사람
   - 근육 많고 체지방도 높은 사람
   - 비만 / 고도비만
   - 여성 저체중 / 여성 일반 / 여성 운동인
   - 주 0회 / 2회 / 4회 / 6회 운동
   - 러닝 중심 사용자
   - 스트렝스 중심 사용자

통과 기준:
- 극단값이 안전선을 깨지 않는다.
- 사용자가 볼 때 목표와 산출값이 상식적으로 납득된다.
- 체성분이 좋은 사람이 단순 운동일수 때문에 초급자로 오판되지 않는다.
- 운동 빈도만 높은 사람이 고급 운동인으로 과대평가되지 않는다.
- 다이어트 / 커팅 / 린매스업 / 벌크업의 차이가 실제 목표 칼로리와 탄단지에 드러난다.
- 28일 조정 제안이 사용자의 목표와 기록 신뢰도를 함께 고려한다.

────────────────────────
8. v8 초반에 하지 않을 것
────────────────────────

v8 초반에는 UI를 다시 크게 갈아엎지 않는다.
이미 v7.x에서 UI를 충분히 많이 건드렸기 때문에, v8 초반은 산식과 프로필 신뢰도를 우선한다.

하지 않을 것:
- 전체 UI 리디자인
- Records / 최근 흐름 / InBody / Settings의 대형 화면 재배치
- 점수 산식 전면 재설계
- 백업 schema 대형 변경
- InBody 목표별 행동 조언을 InBody 탭 안에 직접 넣기

필요하면 하는 것:
- Settings의 운동 프로필 입력 구조 추가
- Today의 세션 / 운동 기준 선택 구조 보완
- 산식 검증용 내부 리포트 / 테스트 추가
- 고급 설정 설명 보강

────────────────────────
9. 문서와 구현 순서
────────────────────────

1. v7.x 안정 태그를 닫는다.
2. v8 기준문서를 확정한다.
3. 현재 산식의 입력값 / 출력값 / 보정값 흐름을 감사한다.
4. 운동 프로필과 사용자 유형 축을 설계한다.
5. 목표별 변화율 범위를 설계한다.
6. 시나리오 검증 러너를 만든다.
7. 현재 산식과 새 산식을 비교한다.
8. 위험한 차이가 있으면 산식이 아니라 설계부터 다시 조정한다.
9. 산식이 안정되면 UI 입력 구조를 최소 변경으로 반영한다.
10. 마지막에 전체 UI 정리를 다시 한다.
