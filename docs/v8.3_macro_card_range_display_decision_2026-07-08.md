## 1. 목적

`docs/v8.3_stabilization_tag_readiness_checkpoint_2026-07-08.md`는 v8.3 scoring implementation block을 `tag-ready after PR merge` 후보로 닫았다.

그 판단은 scoring engine, docs-policy, reference routing, app display label, regression test coverage 기준으로는 유지한다.

하지만 그 뒤 사용자와의 원문 대화에서 별도 문제가 확인됐다.

```text
v8.3 점수는 range-aware다.
그런데 사용자가 보는 macro card가 단일 목표 / 남은 g / 초과 g 중심이면
"목표를 넘었는데 왜 점수가 높은가"라는 혼선이 생길 수 있다.
```

따라서 이번 문서는 merge/tag instruction을 폐기하는 문서가 아니다.

이번 문서의 목적은 다음을 닫는 것이다.

```text
- tag 전에 macro card range display 결정을 닫아야 하는가
- 닫는다면 tag 전 narrow implementation까지 필요한가
- 구현한다면 어떤 문구와 범위까지 허용할 것인가
- 기존 copy test가 어색한 문구를 고정할 때 어떻게 처리할 것인가
```

## 2. 독립 판단

내 판단은 다음이다.

```text
merge/tag instruction은 폐기하지 않는다.
하지만 지금 바로 tag로 가는 것은 보류한다.
tag 전 macro card range display narrow implementation이 필요하다.
```

이유는 두 가지다.

1. v8.3 scoring body가 이미 단일 g target에서 벗어난 상태다.
2. Today macro card는 아직 단일 target과 remaining/exceeded copy 중심으로 렌더된다.

즉 engine은 v8.3인데, 화면의 가장 가까운 macro 설명은 v8.0/v8.1 계열 단일 목표 감각에 남아 있다.

이 문제는 score tuning이나 새 기능 욕심이 아니다.
점수 의미와 화면 의미의 정합성 문제다.

## 3. 원문 대화에서 확인한 사용자 의도

GPT 감사문과 Codex 로그만 보지 않고, 사용자가 직접 문제 제기한 원문 흐름도 함께 반영했다.

확인한 사용자 의도:

```text
- 고정 감점표나 계단식 판정으로 돌아가면 안 된다.
- 외부근거 anchor 기반의 자연스러운 curve가 본류다.
- 운동은 final score bonus가 아니라 carb context다.
- alcohol은 kcal 축과 physiology risk 축을 모두 표현해야 한다.
- 사용자-facing 문구가 AI스럽거나 번역투처럼 보이면 안 된다.
- copy test가 어색한 문구를 고정하고 있다면, 앱 문구를 테스트에 맞추기보다 테스트를 갱신할 수 있어야 한다.
- macro card는 크게 재설계하기보다 기존 구조를 유지하면서 v8.3 의미가 보이게 하는 쪽이 우선이다.
```

이번 문서는 이 의도를 수용한다.

## 7. 표시 문구 기준

용어는 `오늘 범위`로 통일한다.

권장 copy:

```text
오늘 범위 300~460g
오늘 범위 안
오늘 범위까지 25g
오늘 범위보다 12g 높음
```

피할 copy:

```text
- 오늘 해석 범위
- 회복재료
- 회복 재료
- 하한 / 상한을 primary card copy로 쓰는 방식
- 운동했으니 괜찮음
- 탄수만 오늘 기준, 단백질/지방은 권장처럼 섞는 방식
```

`하한`, `상한`은 정확하지만 card primary copy로는 딱딱하다.
필요하면 developer/debug assertion 또는 tooltip 설명 후보로만 둔다.

## 12. copy test 정책

이번 사용자 지적을 명시적으로 수용한다.

```text

무조건 기존 테스트 expectation에 문구를 맞추지 않는다.
```

판단 순서:

1. 테스트가 실제 사용자-facing 의미를 보호하는지 확인한다.

3. 테스트가 낡았으면, 앱 문구를 후퇴시키지 말고 테스트 expectation을 갱신한다.
4. 단, 의미 보호 테스트는 유지한다.

유지해야 할 의미:

```text
- 술 영향을 숨기지 않음.
- 운동 context를 무조건 승인/보너스로 쓰지 않음.
- 기록 부족 상태를 과하게 단정하지 않음.
- source/internal formula term을 사용자에게 노출하지 않음.
- Today range display와 score meaning이 충돌하지 않음.
```

폐기할 수 있는 테스트 관성:

```text
- "회복 재료" 같은 어색한 표현을 보호하는 expectation.
- 단일 target 기준 "남음/초과"만 맞으면 된다는 expectation.
```
